## How to verify port requirements

Identify if specific ports on a remote computer are open, closed, or filtered is a good starting point for troubleshooting network issues. 

To determine which ports are required for network connectivity review article below, then use any of the following methods to test if port requirements are met:
- [Service overview and network port requirements - Windows Server | Microsoft Learn](https://learn.microsoft.com/en-us/troubleshoot/windows-server/networking/service-overview-and-network-port-requirements)

### Test-NetConnection
The`Test-NetConnection`cmdlet in PowerShell (PS) is primarily designed for testing TCP ports and does not natively support testing UDP ports. Quickly check **TCP connectivity** to a single port by running:

    Test-NetConnection -ComputerName <hostname> -Port <port>

If the issue is intermittent and not consistently reproducible, you can use a continuous ping-like command similar to `ping -t` ([ping | Microsoft Learn](https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/ping)). For example, to check the availability of the SMB port `445` on the server `dc1.contoso.com`, you can use the following PS command:

  
    while ($true) {Test-NetConnection dc1.contoso.com -Port 445 -InformationLevel Quiet}

This PS code snippet is designed to test the connectivity of a list of TCP ports frequently used by Active Directory to specified server:

```powershell
$ports = 53,88,135,389,445,464,636,3268,3269
$ports | ForEach-Object {$port = $_; if (Test-NetConnection dc1.contoso.com -Port $port -InformationLevel Quiet -WarningAction SilentlyContinue) {"Port $port is open" } else {"Port $port is closed"} }
```
Ports list: _53 DNS, 88 Kerberos, 135 RPC, 389 LDAP, 445 SMB, 464 Kerberos Password, 636 LDAP SSL, 3268 3269 Global Catalog_.

Thetool can report the status of a port with one of three possible values:

- [ ] 0 (0x00000000)  The connection has been established successfully and the port is **available**;
- [ ] 1 (0x00000001)  The specified port is **unavailable or filtered**;
- [ ] 2 (0x00000002)  The ACK response is not returned, which is the **expected return code when verifying the availability of a UDP port**.


## Network capture

There are situations where checking port connectivity alone may not be sufficient, and you might need to capture and analyze network traffic for a more in-depth diagnosis. Capturing network traffic can help identify patterns or anomalies that occur during the problem.

Most frequently, you will obtain Event Trace Log (ETL) files generated by [TSS](/Tools/TSS), [AUTH scripts](/Tools/Auth-Scripts), or the command `netsh trace start capture=yes`. Please refer example below with the optional parameters you need or check public documentation:
 
- **Start** network capture (elevated CMD):

      netsh trace start capture=yes maxsize=1024 filemode=circular persistent=yes overwrite=yes traceFile=c:\%computername%_cap.etl

  If you use elevated PS change variable `c:\%computername%_cap.etl` with `C:\$env:computername_cap.etl`

- **Stop** network capture:

      netsh trace stop

- Reference
   - [Netsh Commands for Network Trace | Microsoft Learn](https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/jj129382(v=ws.11)) 
   - [How to capture a network trace on a Windows machine](https://techcommunity.microsoft.com/t5/core-infrastructure-and-security/introduction-to-network-trace-analysis-using-microsoft-message/ba-p/257467)

No configuration is required for Wireshark usage.

Wireshark digests packet capture files (normally pcap or pcapng), so those ETL files need to be converted.

### etl2pcapng

[etl2pcapng](https://github.com/microsoft/etl2pcapng) can convert the ETL file to a pcapng file for opening with Wireshark using the syntax below:

```plaintext

```

[Convert-Etl2Pcapng](https://github.com/microsoft/Convert-Etl2Pcapng) is a PowerShell wrapper that provides automated management and updates for etl2pcapng.exe and brings us the right-click context menu feature:

### Wireshark display filters

Please be aware that Wireshark filters are case sensitive. If you are getting started with them, please check [Wireshark Filters](https://www.wireshark.org/docs/man-pages/wireshark-filter.html).

The info column in Wireshark provides a summary of each packet's contents, which can be useful for quickly identifying specific types of traffic or issues. However, it's important to note that the info column is dynamically generated based on the packet's contents and may not always contain consistent or filterable information.

![image.png](/.attachments/image-ab5db92e-2c14-452e-848b-00a002e0cd80.png)

Best practices suggest that when creating display filters, it's more reliable to filter based on specific protocol fields or values that are known to be consistent. For example, filtering by IP addresses, ports, or protocol types can yield more predictable results.

Please find some examples below:

#### DNS
- `dns.qry.name contains "msftconnecttest.com"` // Standard DNS query (insert relevant URL)

#### Kerberos
- `kerberos` // Wireshark also shows SMB with Kerberos ticket
- `kerberos.CNameString like "username"` // Filter by Canonical Name (insert username or computername$)
- `kerberos.SNameString == "ldap"`
- `kerberos.error_code` // All Kerberos errors

#### LDAP
- `ldap || kerberos.SNameString == "ldap"` // LDAP (bind, search, payload*) + TGS
- `ldap.protocolOp == 0 or ldap.protocolOp == 1` // LDAP bind request and bind response

> In computer networking, data to be transmitted is the "payload". It is almost always encapsulated in some type of frame format, composed of framing bits and a frame check sequence.

#### NTLM authentication
- `ntlmssp`

#### Info column (search for any string)
- `(_ws.col.info contains "Your search string goes here")`

#### Process ID
Process ID will not always be populated; you need pcapng converted from ETL file.
- `frame.comment == "PID=4924"`

#### RPC
- `dcerpc` // Stands for "Distributed Computing Environment Remote Procedure Calls"
- `dcerpc.cn_bind_to_uuid == e3514235-4b06-11d1-ab04-00c04fc2dcd2` // RPC filtered by the UUID (Universally Unique Identifier) used to uniquely identify an interface. Other UUID examples:

  - UUID: 897e2e5f-93f3-4376-9c9c-fd2277495c27 - [dfsr](https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-frs2/54a89532-1bd4-486b-99ce-fc9488672eeb) (Distributed File System Replication)
  - UUID: e3514235-4b06-11d1-ab04-00c04fc2dcd2 - [drsuapi](https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-drsr/063618ed-b2e2-4983-ab13-3ed056700641) (Directory Replication Service)
  - UUID: c681d488-d850-11d0-8c52-00c04fd90f7e - [lsarpc](https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-efsr/1baaad2f-7a84-4238-b113-f32827a39cd2)
  - UUID: df1941c5-fe89-4e79-bf10-463657acf44d - [efsrpc](https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-efsr/1baaad2f-7a84-4238-b113-f32827a39cd2)

#### SAM traffic
- `samr.opnum == 3` // SAMR from a remote server to a DC may exhaust DC RPC resources. We have seen this situation with an app using very old API calls that caused the PDCe to slow down/stop replicating.

#### SMB
- `smb2.sesid == 0x0THIS0IS0EXAMPLE0CODE0` // SMB filtered by session ID (example)
- `smb2.nt_status == 0xc0000022` // SMB Access Denied
- `smb2.filename == "contoso.com\\Policies\\{00THIS00-0IS0-JUST-0AN0-000EXAMPLE0000}\\Machine\\Registry.tmp"` // SMB file name (example)
- `(smb2.filename contains "{00THIS00-0IS0-JUST-0AN0-000EXAMPLE0000}") && (smb2.cmd == 6)` // SMB commands issued for a file like creating a connection (5) or closing it (6). Other commands from [SMB2](https://wiki.wireshark.org/SMB2):

  - 0x00 SMB2/NegotiateProtocol
  - 0x01 SMB2/SessionSetup
  - 0x02 SMB2/SessionLogoff
  - 0x03 SMB2/TreeConnect
  - 0x04 SMB2/TreeDisconnect
  - 0x05 SMB2/Create
  - 0x06 SMB2/Close
  - 0x07 SMB2/Flush
  - 0x08 SMB2/Read
  - 0x09 SMB2/Write
  - 0x0a SMB2/Lock
  - 0x0b SMB2/Ioctl
  - 0x0c SMB2/Cancel
  - 0x0d SMB2/KeepAlive
  - 0x0e SMB2/Find
  - 0x0f SMB2/Notify
  - 0x10 SMB2/GetInfo
  - 0x11 SMB2/SetInfo
  - 0x12 SMB2/Break

#### SSL/TLS
- `ssl.handshake.type == 1` // Client Hello
- `ssl.handshake.type == 2` // Server Hello
- `tls.handshake.type == 14` // Server Hello done
- `tls.record.content_type == 21` // Encrypted alerts
- `!(tls.record.content_type == 23)` // Remove application data
- `tls.record.version == 0x0303` // Display only TLS v1.2 traffic. Choose from the hex values below for the version that fits your needs: `0x0300 SSL 3.0 // 0x0301 TLS 1.0 // 0x0302 TLS 1.1 // 0x0303 TLS 1.2`.
- `x509af.serialNumber == 10:30:68:d1:00:00:00:00:00:01` // Display packets containing certificate with specific serial number (you need to add a colon symbol, example here for SN 103068d1000000000001)

#### TCP
- `tcp.analysis.retransmission || tcp.analysis.fast_retransmission` // TCP resets. "TCP RST" after "FIN, ACK" is the quickest method to finish a conversation.
- `(tcp.flags.reset == 1)` // To review TCP retransmits, check that the sequence number (Seq) matches between the request and the retransmit.
- `frame.number == 142` // To review specific frame number

# RPC dynamic port range and the Endpoint mapper

Endpoint mapper (EPM) is a service that listens on the server and guides client apps to server apps by using port and UUID information. The Remote Procedure Call (RPC) in the client machine communicates with server via **fixed port 135**. It then registers its endpoint with the local Endpoint Mapper, which maps the interface/object UUID to a dynamic port. This enables clients to locate the server's dynamic port and continue the communication in that high range port assigned. This is described in detail in 
[Remote Procedure Call (RPC) errors troubleshooting guidance](https://learn.microsoft.com/en-us/troubleshoot/windows-client/networking/rpc-errors-troubleshooting).

Active Directory uses a mix of fixed well-known ports and dynamically assigned ports in the RPC ephemeral port range. By default, it uses ports 1024-5000 on WS2003 and ports **49152-65535** on WS2008 and beyond.

Public article [The default dynamic port range for TCP/IP has changed since Windows Vista and in Windows Server 2008](https://learn.microsoft.com/en-us/troubleshoot/windows-server/networking/default-dynamic-port-range-tcpip-chang) provides the commands to see and change the RPC port range.

```
netsh int ipv4 show dynamicport tcp
netsh int ipv4 show dynamicport udp
netsh int ipv6 show dynamicport tcp
netsh int ipv6 show dynamicport udp
```
This approach might not be practical, especially if the client and server must communicate through a firewall. Public [KB154596 - How to configure RPC dynamic port allocation to work with firewalls](https://learn.microsoft.com/en-us/troubleshoot/windows-server/networking/configure-rpc-dynamic-port-allocation-with-firewalls) refers to RPC parameters in the registry to define custom dynamic port range for RPC.

If the pool lacks sufficient dynamic ports due to improper configuration or an application consuming and not releasing them properly, you'll find System event ID **4227 or 4231** in eventvwr.msc. If you hit this scenario please follow public article [Troubleshoot port exhaustion issues](https://learn.microsoft.com/en-us/troubleshoot/windows-client/networking/tcp-ip-port-exhaustion-troubleshooting).

The RPC ports used by NTDS replication and Netlogon RPC Interface can be also hard-coded in the registry using the steps documented in public article [KB224196 - How to restrict Active Directory RPC traffic to a specific port](https://learn.microsoft.com/en-us/troubleshoot/windows-server/active-directory/restrict-ad-rpc-traffic-to-specific-port).

## Which dynamic port is used by a service?  
As you conduct the network investigation for a specific service, you might need to run `netstat -ano` to identify which process is using which set of ports. Alternatively, you can use PowerShell with the [Get-NetTCPConnection](https://learn.microsoft.com/en-us/powershell/module/nettcpip/get-nettcpconnection?view=windowsserver2022-ps) command to retrieve only the port that is currently in the 'Listen' state for that particular service. Example below:

```powershell
Get-NetTCPConnection -State Listen -OwningProcess (Get-Process -Name "dfsrs").Id
```

It is advisable to verify that these settings are applied not only in the gpresult but also in the registry path. This batch script collects specific registry information from all domain controllers in the network and saves it to a text file for further analysis:

```powershell

reg query \\%%i\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\ComputerName\ActiveComputerName\ /v Computername
reg query \\%%i\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa\
reg query \\%%i\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa\MSV1_0\
reg query \\%%i\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Cryptography\Configuration\Local\SSL\00010002\
reg query \\%%i\HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows NT\Rpc\
reg query \\%%i\HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows NT\DCOM\
) >> c:\temp\DCs-Keys.txt
```
