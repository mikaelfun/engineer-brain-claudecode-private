# How to Configure Provisioning

## Enabling Provisioning for Gallery Applications

Enabling provisioning for an application that has integration between Entra ID and the SaaS application is done via the Entra portal, using the Provisioning blade as well as the Users and Groups blade under Enterprise Applications.

- [Plan an automatic user provisioning deployment](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/plan-auto-user-provisioning)
- [Tutorial: Develop and plan provisioning for a SCIM endpoint](https://learn.microsoft.com/en-us/entra/identity/app-provisioning/use-scim-to-provision-users-and-groups)

## Enabling Provisioning for Non-Gallery Apps (BYOA)

Can be done using the provisioning option on a custom non-gallery application, as long as the target application supports SCIM 2.0.

**Important**: If an app is in the gallery, advise customer to use the gallery version. If not in gallery, suggest getting it added. This does not apply for line of business apps used in one tenant only.

## Scoping

### Assignment via Users and Groups

- "Sync only assigned users and groups": Only users/groups assigned to the Enterprise Application are evaluated
- "Sync all users and groups": All users/groups in Entra ID are in scope

### Scoping Filters

Attribute-based scoping filters applied after assignment evaluation. See [doc](https://docs.microsoft.com/en-us/azure/active-directory/manage-apps/define-conditional-rules-for-provisioning-user-accounts).

## Schema Modifications

- Apps supporting SCIM 2.0 can have attributes from the core schema added
- Entra ID supports limited schema extensions via Microsoft Graph
- Directory Extension Attributes from on-prem AD (via Entra Connect) are exposed via Graph API and consumable by Sync Fabric
- Customers can save/export their schema through MS Graph or portal "review your schema here" option under advance options

## Custom Mappings

Define how data flows between source and target systems. Each mapping consists of:
- Source attribute
- Target attribute
- Additional options (matching, default values, etc.)

See [doc](https://docs.microsoft.com/en-us/azure/active-directory/manage-apps/customize-application-attributes#editing-user-attribute-mappings).

## Test Connections and Authentication

Microsoft Entra provisioning service supports these token types:

1. **Long-lived bearer token**: Copy OAuth bearer token into the **Secret Token** field. Test tokens from `/scim/token` endpoint should not be used in production.

2. **Microsoft Entra bearer token**: If Secret Token field is left blank, Entra ID includes its own OAuth bearer token.
   - Application validates token issuer is Microsoft Entra ID
   - `iss` claim identifies issuer (e.g., `https://sts.windows.net/{tenantId}/`)
   - Audience is the **Application ID** for the gallery app
   - Custom app Application ID: `8adf8e6e-67b2-4cf2-a259-e3dc5476c621`
   - Token generated by Entra ID should only be used for testing, not production
