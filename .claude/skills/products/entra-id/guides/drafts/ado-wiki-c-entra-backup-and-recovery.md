# Entra Backup and Recovery — Support Guide (Public Preview)

> IMPORTANT: Before working any Public Preview Entra Backup and Recovery case, review Support Handling Expectations.

## Support Handling Expectations — Public Preview

All Entra Backup and Recovery cases use the **Scoping First Model**:
- Use the "Questions to ask" section as the mandatory starting point
- Complete scoping before troubleshooting, escalation, or engineering engagement
- Required data: tenant ID, backup ID, report/recovery IDs, timestamps, scope selections

### CSS Responsibilities
- Initial customer contact and expectation setting
- Issue scoping using required Questions to ask
- Collection of required troubleshooting data:
  - Tenant ID, Backup ID, Report/Recovery IDs, Timestamps, Scope selections
- Troubleshooting via: Audit logs, Kusto queries, Engineering documentation
- Resolve independently using available tooling before escalating

### Engineering Assistance (Teams — Public Preview)
- Use designated Teams channel for Public Preview cases
- Escalation criteria strictly enforced

## Scenarios Covered
- Backup selection issues
- Difference report generation
- Recovery execution

## ICM Escalation Paths
- Owning Service: IDX
- Owning Team: AAD Backup and Recovery Engineering
