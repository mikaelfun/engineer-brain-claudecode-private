# Description
A drop file is a file that is generated by Entra Connect via simulating a certain operation on a number of objects. This is particularly useful to check and validate the flow of certain attributes because this only shows attributes and their corresponding values. Note that, it does not show errors on the stage we are testing (for example export errors like attribute-must-be-unique)

# Steps to generate a drop file:


1- First, disable the Sync Scheduler:
	Set-ADSyncScheduler -SyncCycleEnabled $false
	
2- Open the synchronization services and go to "connectors" tab and then right click the connector we want to check:
	

3- Click on "Configure Run Profiles" and then you will see the below window:

  ![image.png](/.attachments/image-dafcabca-1234-47cb-92ac-39534b768dc0.png)


4- Click on "New Profile" and then give it any name:

 ![image.png](/.attachments/image-42ad167e-9d88-4ff8-b570-238fe1fd9873.png)

5- Here, first we want to select the operation that we want to simulate. If you see the issue only during "full import", then select it (in our example here, we are choosing an "Export" operation):

![image.png](/.attachments/image-717aadd6-cbb9-44a7-ae30-10fda9abaa1e.png)


6- The most important step, is that we want to generate a file that contains the info of this simulation or operation. So, we click on "Set Log File Options":

![image.png](/.attachments/image-826c0b5f-4484-426e-844b-92786cbc09db.png)

	
The options in red are your bread and butter. The first one will allow you to create the log file but it will perform the operation you have selected in the previous step. The second option will generate the log file but it will not commit the results to the connector (similar to generate preview and commit preview).
	
7- In the same window, enter the name of your file (here we will just call it "DropFile). After That click Next" and then "Finish".

8- After saving and closing, if you go and right click your selected connector, then select "Run", you will see a new run profile added:
	
	
9- Run the new run profile (here named "DropFile"). After it's done, you will fine your output file in: C:\Program Files\Microsoft Azure AD Sync\MaData\[your connector name]\

10- Re-enable the Sync scheduler: Set-ADSyncScheduler -SyncCycleEnabled $true

11- If you open the text file, you will see details about the objects on which the operation was performed

12- You can delete the run profile after the log file is done.
