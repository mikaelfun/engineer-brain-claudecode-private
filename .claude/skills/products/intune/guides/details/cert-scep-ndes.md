## Quick Troubleshooting Path

### Step 1: NDES service stops issuing SCEP certificates after installing Windows Server update KB5014702 on the CA server. Event log shows DCOM authentication...
**Solution**: Three options: 1) Install KB5014702 on the NDES server as well. 2) Uninstall KB5014702 from the CA server (not recommended). 3) Set registry key HKLM\SOFTWARE\Microsoft\Ole\AppCompat\RequireIntegrityActivationAuthenticationLevel (DWORD) to 0 on the CA server.
`[Source: onenote, Score: 9.5]`

### Step 2: NDES server stops issuing SCEP certificates and returns HTTP 500 error. Connector event log shows certificate validation failure. Certificates were...
**Solution**: 1. Check CRL status using PKIVIEW.msc on the CA jump server. 2. On the root CA, run 'certutil -crl' to renew CRL. 3. Check registry for actual CRL storage path (may differ from default). 4. Copy new CRL and publish: 'certutil -dspublish -f <CA-name>-CA.crl <CA-name>-CA'. 5. Verify CRL is renewed in PKIVIEW.
`[Source: onenote, Score: 9.5]`

### Step 3: Hybrid Azure AD joined device fails to receive user-based SCEP certificate. SCEP profile is configured and assigned but certificates are not delive...
**Solution**: Change the SCEP certificate profile KSP setting from 'Windows Hello for Business' to 'Enroll to TPM KSP if present, otherwise Software KSP'.
`[Source: onenote, Score: 9.5]`

### Step 4: Intune NDES/SCEP connector fails to communicate with Intune service. Error: System.NotSupportedException: The ServicePointManager does not support ...
**Solution**: On NDES server, navigate to C:\Program Files\Microsoft Intune\PFXCertificateConnector\ConnectorUI, open PFXCertificateConnectorUI.exe, edit proxy configuration to URL format: http://"proxy address" or https://"proxy address". Connector status will return to OK after change.
`[Source: onenote, Score: 9.5]`

### Step 5: Windows clients fail to receive SCEP certificate. SCEP profile shows error -2016409340. NDES event log shows 'Failed to verify SCEP request' with C...
**Solution**: Reset the NDES service account password. After password reset, SCEP certificates can be issued and profiles deploy successfully. Note: gMSA (Group Managed Service Account) is NOT supported by the unified Intune cert connector due to a known cert renewal failure issue with no workaround.
`[Source: onenote, Score: 9.5]`

### Step 6: SCEP certificates cannot be issued to clients via NDES. Connector event log shows Event ID 2: 'The client and server cannot communicate, because th...
**Solution**: Enable TLS 1.2 on the NDES server: 1) Set registry keys HKLM\SOFTWARE\WOW6432Node\Microsoft\.NETFramework\v4.0.30319 and HKLM\SOFTWARE\Microsoft\.NETFramework\v4.0.30319 with SystemDefaultTlsVersions=1 and SchUseStrongCrypto=1. 2) Enable TLS 1.2 Server and Client under HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.2. 3) Restart the server.
`[Source: onenote, Score: 9.5]`

### Step 7: Intune SCEP certificate issuance fails with error: The revocation function was unable to check revocation because the revocation server was offline.
**Solution**: 1) Untick Require SSL on the IIS site hosting the CRL distribution point; 2) Manually import/publish the updated .crl file to the CDP location; 3) Verify with certutil -verify -urlfetch cert.cer to check CRL chain validity.
`[Source: onenote, Score: 9.0]`

### Step 8: NDES URL returns HTTP 500 instead of 403
**Solution**: Reinstall IIS/NDES role. Verify all required IIS features are installed. Check NDES application pool status. If issue persists, transfer to Windows Directory Services team.
`[Source: ado-wiki, Score: 9.0]`
