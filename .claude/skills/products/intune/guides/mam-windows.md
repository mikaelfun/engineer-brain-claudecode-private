## 快速排查路径

1. **WIP policy fails to apply with error 0x807C0003 on EDPEnforcementLevel CSP; MDM diagnostic shows status 500 for EnterpriseDataProtection/Settings/EDPEnforcementLevel**
   → 1. Disable Direct Access on the device or exclude it from Direct Access policy; 2. Re-sync device with Intune; 3. Verify via MDM diagnostic report that EnterpriseDataProtection CSP returns 200; 4. ... `[ado-wiki, 🟢 9.0]`

2. **DFS namespace path (\domain.name) is not recognized as corporate by WIP, but direct FQDN path (\fileserver.domain.name) works correctly**
   → 1. In WIP Advanced settings > Network Perimeters > IPv4/IPv6 ranges, add the IP address range assigned to all servers that are members of the DFS namespace; 2. Keep the domain name in Protected dom... `[ado-wiki, 🟢 9.0]`

3. **Firefox, Chrome, WhatsApp desktop or other apps using custom network stack are blocked from accessing corporate resources even though they are in the WIP allowed apps list**
   → Add the /*AppCompat*/ string to the WIP Cloud resources setting to stop Windows from automatically blocking connections from apps that use IP addresses instead of FQDNs `[ado-wiki, 🟢 9.0]`

4. **MAM (Mobile Application Management) for Edge on Windows fails when device is Azure AD joined (AADJ) or Hybrid Azure AD joined (HAADJ). Conditional Access with 'Require app protection policy' blocks...**
   → 1) Ensure device Join Type is 'Azure AD registered'; 2) Uncheck 'Allow my organization to manage my device' during registration; 3) Verify MAM flags via edge://edge-dlp-internals (mslrmv2=Enabled, ... `[onenote, 🟢 8.0]`

5. **WIP without enrollment (WE) policy not applying / device cannot connect to WIP MAM endpoint wip.mam.manage.microsoft.com**
   → Ensure TCP port 444 is allowed through the customer firewall for WIP without enrollment connectivity. Also verify user has Azure AD Premium license (required for WIP WE). `[ado-wiki, 🟢 8.0]`
