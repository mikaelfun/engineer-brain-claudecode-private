## 快速排查路径

1. **Hybrid AADJ Autopilot provisioning delays 3-4 hours; device stuck at ESP Account Setup phase waiting for Azure AD registration**
   → Workaround: Deploy Intune PowerShell script during WhiteGlove to run 'Enable-ScheduledTask -TaskPath "\Microsoft\Windows\Workplace Join" -TaskName "Automatic-Device-Join"' to force-enable the task ... `[onenote, 🟢 9.5]`

2. **Intune device compliance state shows Unknown; ValidPolicyCompliancesCount is 0**
   → 1) Verify device checked in within 2 days; 2) Confirm not DEM-enrolled; 3) Use device-based groups for compliance targeting with GPO/Hybrid AADJ; 4) Kusto: IntuneEvent where ServiceName==StatelessC... `[onenote, 🟢 9.5]`

3. **Autopilot Hybrid AD Join 部署中设备报错 Domain not reachable 80070774，超时 25 分钟后失败**
   → 1. 确保设备与 Active Directory 域控制器在同一网络且 ICMP 可达；2. 如使用 VPN 部署，启用 SkipDomainConnectivityCheck 跳过 ICMP ping 步骤；3. 检查防火墙规则确保设备到 DC 的 ICMP 流量未被阻断 `[ado-wiki, 🟢 9.0]`

4. **Autopilot HAADJ 部署中 ESP User Setup 阶段超时，用户无法完成 Account Setup**
   → 1. 确认 AD Connect 已同步设备为 Hybrid Azure AD Joined 状态；2. 等待 AD Connect 同步周期完成后再让用户登录；3. 如 ESP User Setup 中 Account Setup 已启用，确保同步时间窗口足够；4. 用户需使用 domain\user 格式的本地凭据登录而非 Azure AD UPN `[ado-wiki, 🟢 9.0]`

5. **Autopilot HAADJ 部署失败，设备收到错误的 Deployment Profile（如收到 Azure AD only profile 而非 Hybrid profile）**
   → 1. 确保只有一个 Deployment Profile 显示 Assigned=Yes；2. 不同 join type 的 profile 不能分配给同一个 Dynamic Device Group；3. 删除或取消分配多余的 profile `[ado-wiki, 🟢 9.0]`
