## Quick Troubleshooting Path

1. Verify gMSA permissions with Get-ADServiceAccount; add DC or security group to PrincipalsAllowedToRetrieveManagedPassword; reboot DC or purge Kerberos ticket (klist -li 0x3e7 purge) `[Source: MS Learn]`
2. Ensure sensor can browse to *.atp.azure.com through the configured proxy without authentication; configure proxy bypass for Defender for Identity endpoints `[Source: MS Learn]`
3. Verify certificates with Get-ChildItem -Path 'Cert:\LocalMachine\Root'; download and install the missing DigiCert root certificate using Import-Certificate cmdlet `[Source: MS Learn]`
4. Use the complete command with ./ prefix: ./"Azure ATP sensor Setup.exe" /quiet NetFrameworkCommandLineArguments="/q" AccessKey="<Access Key>" `[Source: MS Learn]`
5. Disable IPv4 TSO Offload in VM NIC settings; run Disable-NetAdapterLso -Name {adapter-name}; may require VM restart `[Source: MS Learn]`
