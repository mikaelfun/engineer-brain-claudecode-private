Supportability requests are work items that CSS can request PG to add to their backlog for triage and development. They may include changes to docs, improvements to the service to reduce the number of cases, and changes to the tooling CSS uses to support the service.

**Supportability requests must either be filed or approved by any of the ARM TAs.**

PG meets monthly with the TAs to prioritize the CSS request backlog, and then triage with the team how much effort is required to complete the request. CSS asking PG to prioritize the request does not mean it will get done within a specific time window, it just means the PMs have an idea of what tasks are important to CSS and they can allocate the resources if possible.

## Filing a supportability request
PG has set up this [template](https://msazure.visualstudio.com/One/_workitems/create/Task?templateId=3d982190-c2c7-478b-965e-01b0cf1e6f90&ownerId=ed163089-2386-4f0f-81a9-d80fa71e2f55) for CSS to file these asks.

The template provides clear instructions on what changes need to be made to the tasks and what information is required on the description before saving. Missing any of the steps required can result in the task being lost and not trackable.

## Querying active supportability requests
When TAs meet with the PMs for prioritization, they will look at this [query](https://msazure.visualstudio.com/One/_queries/query/5ba39139-83f4-4d51-9436-532135524a21) which includes the tasks that have been filed against the backlog and are pending triage.

## Querying status for prioritized tasks
This [query](https://msazure.visualstudio.com/One/_queries/query/7b426171-7386-4c05-8382-46f04df5a864) contains the tasks that have been previously prioritized and their current status.
