Engineering team is using ICM Title to indicate if an ICM is pending CSS or Customer action/confirmation to continue. 

If the ICM that is related with Support Request that you are working on marked as "Waiting [for] CSS" or "Waiting [for] Customer" please do the necessary action to unblock those ICMs.

* Review the ICM for any pending actions on you
* Provide the required update
* Remove the tag if action is no longer on customer/css
* If you are still waiting an update from customer, please update the ICM and leave the tag in the title
* Confirm the status of the ICM, If the problem is still ongiong the ICM should be Active, and ICM is only mitigated if the problem has been mitigated and pending RCA. 

## After you remove the tag from ICM
* ### If ICM is still in engineering ICM team
  * If you are actively working on the issue with a specific engineer, make sure the ICM is assigned/re-assigned to that owner
  * If you are not actively working with someone, make sure the ICM is not assigned to anybody
  * or escalate to EEE/CSS escalation team to find the next proper action for that ICM

If you need help please reach out to EEEs ([azurepurvieweees](mailto:azurepurvieweees@microsoft.com))
