# Basic Metrics TSG

## Overview

This guide is intended to help you troubleshoot issues with basic metrics.

## Metrics are not being generated or showing up in Grafana dashboards

If you are not seeing metrics in Grafana, it is likely that the metrics are not being generated. To troubleshoot this issue, you can check the following:

### Check Kappie Pods are running

Check that all Kappie pods are running. You can do this by running the following command:

```shell
kubectl get pods -n kube-system -l k8s-app=kappie
```

If any of the pods are not running, you can describe the pod to see what the issue is. You can do this by running the following command:

```shell
kubectl describe pod -n kube-system <kappie pod name>
```

### Check Kappie Pod Logs

If kappie pods are running, you can check the logs for the pod to see what the issue is. You can do this by running the following command:

```shell
kubectl logs -n kube-system <kappie pod name>
```

you should see log lines like the following:

```shell
2023-04-15T23:52:08.046Z        info    dropreason      dropreason/dropreason_linux.go:208      DropReason data collection complete
2023-04-16T00:10:55.598Z        info    linuxutil       linuxutil/linuxutil.go:100      Updated ethTool metrics
```

### Check Kappie ConfigMap

If not then the plugin is not running correctly. Please check kappie config map and make sure the plugins is enabled.

```shell
kubectl get configmap kappie-config -n kube-system -oyaml
```

### Check Kappie Metrics are being generated

You can check that metrics are being generated by calling the metrics endpoint. You can do this by running the following command from inside one of the nodes:

```shell
curl http://<kappie-pod-ip>:10093/metrics | grep -i kappie
```

### Check if Managed Prometheus is scraping kappie metrics

If you are using managed Prometheus, you can check if it is scraping the kappie metrics by following below steps.

1. Check if AMA-Metrics pods are running

    ```shell
    kubectl get pods -n kube-system -l dsName=ama-metrics-node
    ```

2. If they are running, then port forward one of the AMA-Metrics pod to see if it can identify kappie pods

    ```shell
    kubectl port-forward -n kube-system <ama-metrics-pod-name> 9090:9090
    ```

3. Open the following URL in your browser to see if kappie pods are being discovered

    ```shell
    http://localhost:9090/targets
    ```

    ![prometheus-kappie-targets.png](/.attachments/prometheus-kappie-targets-5703bf92-0031-46ef-9e08-7db693e66553.png)

    a. check if service discovery is recognising kappie pods

    ```shell
    http://localhost:9090/service-discovery#kappie-pods
    ```

    ![prom-kappie-service-discovery.png](/.attachments/prom-kappie-service-discovery-bc740905-5b56-4a7c-8713-c01478da627f.png)

4. If kappie pods are not discovered then please check prometheus configuration and make sure the kappie-pods job is added to the scrape config.

    ```shell
    http://localhost:9090/config
    ```

    ![prom-kappie-config.png](/.attachments/prom-kappie-config-b7e9cc9f-e9b7-4133-82f8-db3a6611ad1a.png)

5. If the kappie-pods job is not present in the prometheus config, please check ama-metrics-node config map for the presence of this job of the above job in the ConfigMap.

    ```shell
    kubectl get configmap ama-metrics-prometheus-config-node -n kube-system -oyaml
    ```
