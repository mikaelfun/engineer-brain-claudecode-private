# Case Misroutes: CIRT — Determining Scope

**SAP:** Security \ Microsoft 365 Security Incident Response \ Malicious Microsoft 365 Activity Detected

## Restrictions

CIRT **cannot** support:
- Advanced Support for Partners (ASfP)
- Broad Commercial
- Internal Premier support

CIRT **can** support:
- Premier Support for Partners (PSfP)
- Unified and Premier (confirmed — this has changed)

## Reasons to Engage CIRT

- There is a **suspected compromise** (with indicators)
- Customer wants an **investigation** of the compromise (how it happened and/or what is happening)

For more information: [Engaging the CIRT team — Overview](https://dev.azure.com/ASIM-Security/Messaging%20Protection/_wiki/wikis/Messaging%20Protection%20Wiki/8607/Engaging-the-CIRT-team)

## How to Engage

- **Send the full case over to CIRT (transfer)** — do not collab
- If not sure → reach out directly to the CIRT team
- **No collabs**

## Active Security Incidents — Immediate Action Required

- Cases that come into the MDO queue with incident response scenarios/needs should be **routed to CIRT ASAP** for investigation and remediation
- **DO NOT TROUBLESHOOT** with the customer — send to CIRT immediately upon confirmation that it's an active incident
- It is okay to meet the SLA of the case if needed
- You can send an email to the customer informing them that you are transferring the case to the CIRT team
- If you need to call to scope the case properly, that is ok — but do not troubleshoot with them
