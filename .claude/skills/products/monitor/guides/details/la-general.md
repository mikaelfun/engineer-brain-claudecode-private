## Quick Troubleshooting Path

### Step 1: Fired alerts not showing in Log Analytics workspace Alert portal page, but visible when querying Alert table directly via KQL. Alert rules confirmed to be triggering successfully in ASC.

**Solution**: Use Azure Monitor → Alerts for a centralized/comprehensive view of all alerts across all resources, instead of viewing per-resource Alert pages. Per-resource Alert page only shows alerts whose targetResource matches that specific resource.

`[Source: OneNote, Score: 9.0]`

### Step 2: Cannot modify Data Retention of Log Analytics workspace on Azure portal. Changing retention days (e.g., 90 to 30 or 100) returns GatewayTimeout error: 'The gateway did not receive a response from M...

**Solution**: 1) Collect F12 HAR file to identify the failing PUT request. 2) Use ARM Kusto (armmcadx cluster) tables: EventServiceEntries for correlationId/status, HttpIncomingRequests for 504 errors, HttpOutgoingRequests to trace redirection to sub-providers (DAS→AMS). 3) Check Jarvis LogsDAS and LogsAMS nam...

`[Source: OneNote, Score: 9.0]`

### Step 3: Azure JS SDK monitor-query for Log Analytics in Mooncake throws AADSTS500011 error: 'The resource principal named https://api.loganalytics.azure.cn/.default was not found in the tenant'.

**Solution**: Change audience from 'https://api.loganalytics.azure.cn/.default' to 'https://api.loganalytics.azure.cn' (remove /.default). Keep endpoint as 'https://api.loganalytics.azure.cn/v1'.

`[Source: OneNote, Score: 9.0]`

### Step 4: Customer needs to query Log Analytics data via REST API in Mooncake to export >30000 rows. Confused about API permission vs RBAC.

**Solution**: Register App, assign RBAC Log Analytics Reader on workspace. Get token: POST login.partner.microsoftonline.cn/<tenant>/oauth2/token. Query: POST api.loganalytics.azure.cn/v1/workspaces/<id>/query. API supports up to 500K rows.

`[Source: OneNote, Score: 9.0]`

### Step 5: Cannot cross-query between ADX and Log Analytics workspace in Mooncake. Docs show wrong endpoint.

**Solution**: ADX to LA: use https://adx.loganalytics.azure.cn/subscriptions/<sub>/resourcegroups/<rg>/providers/microsoft.operationalinsights/workspaces/<name>. Need LA Reader role. LA to ADX via adx(): need Reader + ADX Database Viewer.

`[Source: OneNote, Score: 9.0]`
