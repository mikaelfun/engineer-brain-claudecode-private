# DocuHeros Project

---

As of late 2023 there has been an on going team that is addressing cases with the root cause of "Documentation Missing, or Insufficient" and work items are created for any case that has that root cause.

  

<div style="border=0px;margin-bottom:20px;padding:10px;min-width:500px;width:75%;border-radius:10px;background-color:#d7eaf8">

  

**Important**

  

Please reach out to your TA before beginning this process.   Your TA will submit the change to the documents team.

</div>

  

## What does this mean?

  

* You will be contacted by one of these TAs, SMEs, or Champions when a case creates a work item.

* You will be asked about the document that needs to be changed and what should change.

* We are asking you to please put in the notes before closing the case all the details about what change needs to be made.  It should hopefully be enough information to prevent us from needing to reach out to you.  

* **We intend to update you once we have published this document so you can put it on your connect as impact.**

  

# Learn Editor

---

You can contribute directly to product documentation using the [Learn Editor](https://aka.ms/savetime).  The Learn Editor allows quick editing and is available to use by all internal Microsoft employees.

  

To use the editor, navigate to [Learn Editor](https://aka.ms/savetime), paste in the URL of the article you want to edit and start editing.

  

For more information, see [Instructions](https://learn.microsoft.com/help/platform/learn-editor-overview?branch=main).  To help improve, please provide [Feedback](https://ceapex.visualstudio.com/Engineering/_workitems/create/Bug?templateId=a8539839-4fdb-406d-9c16-b21fd644a617&ownerId=8fde4a15-c97f-4df1-b93a-1da2dcb9ae89).

  

<div style="border=0px;margin-bottom:20px;padding:10px;min-width:500px;width:75%;border-radius:10px;background-color:#d7eaf8">

  

**Important**

  

Ensure you record the documentation link and the resulting pull request from the Learn Editor in your case notes and in the resolution summary (not your LQR to the customer).

  

Ensure to set the root cause to *Product Documentation is incorrect or incomplete - CSS submitted doc update*.

</div>

  

# Documentation Request

---

## API / CLI / SDK Content

  

For documentation where the Microsoft Learn URL starts with one of the following, you should submit an ICM to the relevant product group as modifying these articles requires an engineering effort (they have to update what is called a [Swagger or OpenAPI](https://learn.microsoft.com/en-us/aspnet/core/tutorials/web-api-help-pages-using-swagger)).

  

* https://learn.microsoft.com/rest

* https://learn.microsoft.com/cli

* https://learn.microsoft.com/powershell/

  

<div style="border=0px;margin-bottom:20px;padding:10px;min-width:500px;width:75%;border-radius:10px;background-color:#d7eaf8">

  

**Important**

  

Ensure you record the documentation link and the ICM link in your case notes and in the resolution summary (not your LQR to the customer).

  

Ensure to set the root cause to *Product Documentation is incorrect or incomplete - Needs PG review*.

</div>

# Reporting Translation Issues

---

You can submit a work item to report translation issues for any page hosted on learn.microsoft.com.  ([Learn more](https://learn.microsoft.com/en-us/help/contribute/localization-feedback-to-loc-content))

  

* To report a translation issue, navigate to https://aka.ms/localizationfeedback.  You will need to login using your Microsoft corporate account (@microsoft.com).  Populate the fields of the form and submit.

* To view existing workitems or to directly submit workitems to the Azure DevOps (ADO) repository, you will need to join the [CEAPEX Work Items Access](https://coreidentity.microsoft.com/manage/entitlement/entitlement/ceapexworkit-rdwl) group in [CoreIdentity](https://aka.ms/coreidentity).  Once approved, you can then submit a direct ADO workitem by navigating to https://aka.ms/icxlocfeedback.

  

<div style="border=0px;margin-bottom:20px;padding:10px;min-width:500px;width:75%;border-radius:10px;background-color:#d7eaf8">

  

**Important**

  

Ensure you record the documentation link in your case notes and in the resolution summary (not your LQR to the customer).

  

Ensure to set the root cause to *Product Documentation is incorrect or incomplete - Language translation or other localization issue*.

</div>
