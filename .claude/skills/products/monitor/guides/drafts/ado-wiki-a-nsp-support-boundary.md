**Overview:**
If customer is using Network Security Perimeter, there might be a need for collaboration between Azure Monitor and Azure Networking team to resolve customer issue.


**Boundaries**
If it's not clear as to whether the case falls within the Azure Monitor support boundaries, then you should accept ownership of the case, perform your first quality response (FQR) actions and engage with the customer to better understand their issue. Once you understand the issue, follow the below guidelines and either transfer the full case or send a collaboration task to the proper team.


## **Monitor Issues:**


| **Issue Description** | **Queue**  |
|--|--|
|Issues related to adding Monitor resources to NSP  | Azure Monitoring |
| Issues related to LA ingestion using AMA | Azure Monitoring  |
| Issues related to Log Analytics workspace, Billing, retention | Azure Monitoring |
|Application Insights Ingestion | Azure Monitoring|
|LA query related issues|Azure Monitoring|
|Log Search Alerts |Azure Monitoring|
|Action Groups|Azure Monitoring|
|Application Insights Live Metrics|Azure Monitoring|



## **Networking Issues:**

|**Issue Description**|**Queue**|Support topic|
|--|--|--|
|Issues related to creating and deleting NSP resource | Azure Networking| Network Security Perimeter|
