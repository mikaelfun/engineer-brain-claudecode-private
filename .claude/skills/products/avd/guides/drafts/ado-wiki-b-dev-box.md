# Microsoft Dev Box - Support Routing

## Overview

Windows 365 has built a first party API platform. It is an extension of the public facing Graph API that lets internal partners within Microsoft build their own flexible, customized Cloud PC experiences. One such partner is Microsoft Dev Box.

Microsoft Dev Box is owned by Azure's Developer Division and is billed on consumption, sold through Azure and has the objective of accelerating developer velocity.

## Support Process for Dev Box cases

Microsoft Dev Box will have its own dedicated support path through Azure support. The Dev Box CSS teams will open collaboration tasks with other teams such as Azure Networking, AVD, or Windows 365 as necessary for related issues.

Issues that must be escalated to engineering will be done by individual CSS teams involved in these collaboration. SaaF involvement will continue on a case-by-case basis for these as it does for other support escalations.

## Key points

- Dev Box uses W365 platform but has its own Azure support path
- Dev Box CSS teams handle primary support
- Collaboration tasks opened with AVD/W365/Networking as needed
- Engineering escalations handled by individual CSS teams
