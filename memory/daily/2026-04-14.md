## Patrol v2 WebUI 验证失败

### 问题 1: Haiku 不遵循 v2 runner 指令
- casework-light.md 改了 model: haiku，但 haiku 读了 v2 spec 后自行即兴走 v1 逐步骤模式
- 证据: casework-light.log 无今天条目、casework.log 有 STEP B4/B5 旧格式、每 case 3-9min
- 修复: model 改回 sonnet，haiku 指令遵循不够强

### 问题 2: teams-search-queue spawn prompt 用了 Windows 绝对路径
- Agent Monitor 显示 Session failed: casesRoot=C:\Users\fangkun\...
- 根因: patrol 主 agent 把 ./cases resolve 成 C:\...\cases 传给 subagent
- 反斜杠 \U \f \D 在 Bash 中被当转义符
- 修复: patrol SKILL.md + CLAUDE.md 加路径约束：spawn prompt 必须用相对路径

### 教训
- Haiku 不适合需要精确遵循 200 行 spec 的 agent（会"即兴创作"）
- Windows 平台上 LLM 会"帮忙" resolve 相对路径为绝对路径，但用了 Windows 格式
- 三种路径陷阱: (1) python3 /c/ 字面目录 (2) spawn prompt 反斜杠 (3) Bash vs PowerShell 格式
