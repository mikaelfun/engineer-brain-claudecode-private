
## cc-connect 排查与修复

### 问题
cc-connect 通过飞书发消息后，返回 400 Bad Request（response_len 固定 128 字符）

### 根因
kunnewapi.net 对 `claude-opus-4-6` + Claude Code CLI 的某些请求参数组合返回 400。curl 直接测试 API 全部 200，但 Claude Code CLI（v2.1.74）在 EngineerBrain work_dir 下加载完整 CLAUDE.md + MCP tools 后（~51K input tokens），触发了上游 provider 的 400。

### session 文件里的实际错误
```
API Error: 400 {"error":{"type":"<nil>","message":"Bad Request\n (request id: 20260329114637685872106TMss7SB2)"},"type":"error"}
```

### 修复
1. **模型**: `claude-opus-4-6` → `claude-sonnet-4-6`（解决 400）
2. **quiet 模式**: config.toml 加 `quiet = true`（隐藏 tool 调用中间消息）
3. **无窗口启动**: 创建 `start-cc-connect.vbs`，scheduled task 改为调用 vbs（不弹 cmd 窗口）

### 关键文件
- `~/.cc-connect/config.toml` — 主配置
- `~/.cc-connect/start-cc-connect.bat` — 启动脚本（清 session + 设环境变量）
- `~/.cc-connect/start-cc-connect.vbs` — 无窗口启动器
- `~/.cc-connect/sessions/*.json` — session 文件（含历史和错误信息，排查利器）

### 排查技巧
- 400 错误在 cc-connect 日志里不显示为 ERROR，而是正常 `turn complete` + 固定 `response_len=128`
- 真正的错误信息藏在 `sessions/*.json` 的 history 字段里
- `config.toml` 设 `level = "debug"` 可看到 Claude Code 启动参数和事件流
