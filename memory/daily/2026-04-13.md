
## Patrol path bug discovery

### Root cause
- python3 open() with POSIX /c/Users/... writes to C:\c\Users\... on Windows
- Git Bash MSYS2 path translation does NOT apply inside python3 -c strings
- File appears "missing" because it's written to wrong directory tree

### Fix applied
- casework-light.md: removed Write tool from tools list, added path redline
- CLAUDE.md: added python3 path trap rule under platform rules
- Rule: python3 must use relative paths (./cases/...) or Windows format (C:/...)
- Never pass Bash $CASE_DIR (containing /c/) into python3 string

- Cleaned ghost directory C:\c\ created by previous buggy runs
