The Phoenix _OciArtifactPublisher_ component is responsible for pushing security assessment artifacts into customer container registries.
The component is implemented as service hosted in the MDC CIEng Kubernetes clusters.

**Summarized essential flow**
1. Consume messages from the _publish-topic/registry-publisher-subscription_ ServiceBus subscription. The messages are generated by Phoenix Assessor component.
2. Read assessments from Azure storage.
3. Obtain registry access token through Pricing Identity service.
4. Push the assessment to registry.
5. Generate assessment signature using the ESRP system.

**Monitor meaning**

The monitor is triggered when no successful message processing happens at all for a defined period of time for a component in a specific geo & trigger type (notification/snapshot). This means that either no messages arrive or all messages fail processing.

**Investigation steps**

Check the Grafana dashboard, does all message processing fails or no messages arrive at all?

1. If no messages arrive, investigate why.
   - Is Assessor sending messages? Check lifecycle events:
     ```kql
     (cluster('romeeus.eastus.kusto.windows.net').database('ProdRawEvents').Phoenix_Assessments_LifeCycleEvents | extend Geo = "US")
     | union (cluster('romeuksouth.uksouth.kusto.windows.net').database('ProdRawEvents').Phoenix_Assessments_LifeCycleEvents | extend Geo = "EU")
     | where GeneratedTimestamp > ago(1h) and Component == "Assessor" and LifeCycleEvent == 'SentToPublishStep'
     | take 100
     ```
   - Are there any problems with receiving the messages? Check the component error logs as in item (2) below.

2. If all message processing fails, check Grafana dashboard for hint on error reasons. Also, check error logs as below:

   - Check for error hot-spots:
     ```kql
     cluster('mdcprd.centralus.kusto.windows.net').database('Detection').Span
     | where (k8s_pod_name startswith "ociartifactpublisher") and TIMESTAMP > ago(1hr) and success == False
     | summarize count() by name
     ```
   - Check _resultDescription_ column for the thrown exception details:
     ```kql
     cluster('mdcprd.centralus.kusto.windows.net').database('Detection').Span
     | where (k8s_pod_name startswith "ociartifactpublisher") and TIMESTAMP > ago(1hr)
     | where success == False and name == "InstrumentedServiceBusProcessor.ProcessMessageWrapper"
     | take 100
     ```
