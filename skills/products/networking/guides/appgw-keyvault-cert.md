## 快速排查路径
1. Delete principalId and clientId fields from the identity section in the ARM template before deployin `[来源: onenote]`
2. Customer needs to re-bundle the certificate correctly and re-upload it to Key Vault. Azure Networkin `[来源: ado-wiki]`
3. Re-upload the SSL certificate to Key Vault, then go to the Listener blade and re-select the new cert `[来源: ado-wiki]`
4. Reapply the Managed Identity on AppGW via PowerShell:
$AppGw = Get-AzApplicationGateway -Name $AppGw `[来源: ado-wiki]`
5. Remove the unused SSL certificate from AppGW via PowerShell:
$AppGW = Get-AzApplicationGateway -Name `[来源: ado-wiki]`
