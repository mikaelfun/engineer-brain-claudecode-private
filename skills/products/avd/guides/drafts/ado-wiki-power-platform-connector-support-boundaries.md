# Windows 365 Power Platform Connector — Support Boundaries

## What CSS Supports (Windows 365 Connector Layer)
- Availability of the Windows 365 connector
- Connector actions related to Cloud PCs (lifecycle operations, provisioning policy management)
- Validation of prerequisites, permissions, and roles
- Issues where connector calls fail due to Windows 365 service issues or backend APIs unavailable

CSS responsibility is limited to Windows 365 service behavior and connector exposure.

## What CSS Does NOT Support
- Creation, design, or debugging of Power Automate flows, Power Apps, or custom workflows
- Logic errors or misconfiguration inside Power Platform
- Third-party or non-Microsoft connectors
- Automation design decisions or optimization

## Explicit Boundary Statement
- The Power Platform Connector is an extension of Windows 365 Connectors
- All prerequisites, setup steps, and troubleshooting follow Windows 365 Connector guidance
- CSS does not provide Power Platform support

## Collaboration Model
- CSS may collaborate (without ownership transfer) with Power Platform teams
- Ownership does not transfer unless explicitly directed via internal escalation
