# Overview
The App Assure team handles the following issues: 
- Customer Developed Line of Business (LOB) Applications
  - Determine root causes and recommend code changes to applications
  - Shim applications where source code is not available
  - Address Microsoft product regressions impacting application compatibility
- Independent Software Vendor (ISV) Applications
  - Determine root causes and assist ISVs with remediating their applications
  - Engage ISVs to onboard them to the newest versions of Microsoft products and secure support statements
  - Address Microsoft product regressions impacting application compatibility 
- Microsoft First-Party Products - Determine root cause and address compatibility issues with:
  - Microsoft Office, including macros and add-ins
  - Microsoft Edge and Internet Explorer
  - All other Microsoft first-party products 

# How to Transfer to App Assure
How do you determine if customer reported issue is an app compatibility case? Look for key words such as the following: 
- App crashes, fails to launch. 
- App will not install, install crashes, install fails to complete.
- App display issue, glitch, rendering incorrectly.
- App performance issues, unresponsiveness, takes a long time to load.
- Authentication repeatedly required. 

If a case for W365 has been opened and there is an application compatibility issue, please engage App Assure team using the following process: 

**Step 1:** Send an email to App Compat Help (achelp@microsoft.com) with the following information in the body of the email:
- Customer Name: <customer name>
- Customer Tenant ID: <tenant GUID>
- Customer Location: <location>
- Customer Contact Name: <first and last name>
- Customer contact email address: <email address>
- Customer contact phone number: <phone number>
- Title: <case title>
- Description: <short description>
- Issue description: <full details>

**Step 2:** Enter a note in the DfM case that this has been forwarded to the App Assure team.

**Step 3:** Reply to customer that the issue is now being handled by the App Assure Team and expect an email from that team with a new case number to track the issue.

**Step 4:** Close the DfM case.
