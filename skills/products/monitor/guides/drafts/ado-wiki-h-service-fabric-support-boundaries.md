# Service Fabric - Application Insights Support Boundaries

## Overview

Azure Service Fabric is a distributed systems platform that makes it easy to package, deploy, and manage scalable and reliable microservices and containers. Service Fabric managed clusters streamline deployment and cluster management.
See: https://learn.microsoft.com/azure/service-fabric/overview-managed-cluster

## Cluster-Level Integration

- Service Fabric cluster has integration with Application Insights enabled in two ways:
  1. **During cluster creation** - UI provides a place to provide an Application Insights key. This enables the cluster to send data via Azure Diagnostics to Application Insights.
  2. **After creation** - Additional resources can be configured to send data via Azure Diagnostics.
- Data shows up in Application Insights predominantly in the **Traces** table with `sdkVersion` prefaced with **"WAD"**.

## Microservice-Level Integration

- The Application Insights SDK can be added to microservices running on Service Fabric containers, same as standard web apps.
- There is an add-on NuGet package - a **Service Fabric telemetry initializer** - that enriches telemetry with Service Fabric environment details.
  - See: [Microsoft Application Insights for Service Fabric](https://github.com/microsoft/ApplicationInsights-ServiceFabric)
- Telemetry generated by this add-on will come with the SDK prefix **"serviceremoting"**.
- **These packages are part of an old collaboration between the .NET and Service Fabric Product teams and are no longer active/maintained.** Application Insights CSS team will offer best-effort support, but there will be no further updates or fixes by PG.
- There are early discussions with Product teams on developing an **OpenTelemetry distro** to support Service Fabric workloads as a replacement.
