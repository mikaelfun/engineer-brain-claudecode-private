# How to get an Azure Tenant and Subscription via Visual Studio benefits

## Introduction

While the Microsoft Internal Subscription within the Microsoft Tenant provides access to an environment for reproduction steps, you cannot create new user accounts within the Microsoft Tenant, meaning you cannot test behavior of accounts with different permissions and roles. For these scenarios, use the benefits of your Visual Studio Subscriptions.

> **Note**: Visual Studio Subscriptions are available only for Microsoft Full Time Employees (FTE). Vendors and outsourced resources (alias starting with "a-" or "v-") are not eligible.

## Before you begin

- You will need a Microsoft **CORPNET** account (@microsoft.com) to login into the registration page
- You will require a **personal Microsoft Account (MSA)** (@live.com, @hotmail.com, or @outlook.com). Create one at: https://account.microsoft.com/account

## Instructions

1. Open a browser, navigate to https://my.visualstudio.com and login using your **CORPNET account**
2. Locate the benefit titled **Azure** ($150 monthly credit) and click on **Activate**
3. Follow the instructions to redeem your $150 Azure Credits to your **personal MSA account**
   - **Important**: Use the **personal MSA account**, otherwise you may lose access to the Visual Studio monthly credit benefit
4. You can start using this subscription immediately. The credit will be reset to $150 after each month, regardless if you have spent it.
