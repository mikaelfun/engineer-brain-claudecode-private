# SecurityPermissions Checker Module

The **SecurityPermissionsChecker** PowerShell module provides a unified way to analyze and map administrator permissions across:

- Microsoft Defender XDR (URBAC)
- Legacy Microsoft 365 RBAC (Security & Compliance Center / EXO)
- Microsoft Entra ID roles
- Delegated admin roles (GDAP / B2B)

This tool allows support engineers, consultants, and administrators to quickly identify:

- What permissions a user actually has
- Where those permissions come from
- Missing permissions causing feature failures
- Role mapping between Legacy RBAC and Unified RBAC
- Temporary vs permanent admin rights

## Key Features

- Unified view of all security-related permissions
- Automatically parses: Entra roles, Defender URBAC roles, Legacy RBAC roles, B2B / GDAP delegation
- HAR file parsing support
- Highlights missing permissions required for features
- Generates structured HTML file output for troubleshooting

## Requirements

- Latest PowerShell Version
- Security & Compliance PowerShell module

## Engineering escalations

In any case where we escalate to the Engineering **Permissions** team, we must include the full output of the SecurityPermissions Checker (or equivalent permissions diagnostic) so that the assigned roles and effective permissions of the impacted admin are clearly documented.

Ideally, the **HTML report** generated by the tool should be attached to DTM as part of the case notes.

## Installation

```powershell
Install-Module SecurityPermissionsChecker -Scope CurrentUser
```
