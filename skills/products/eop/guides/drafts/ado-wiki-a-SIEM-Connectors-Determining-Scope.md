# SIEM Connectors: Determining Scope

## Routing Table

| Support Topic | Owner / Article |
|--------------|----------------|
| Alerts and SIEM Integration API for detections and automated investigations | **MDO owns this.** [SIEM integration with MDO](https://learn.microsoft.com/en-us/defender-office-365/siem-integration-with-office-365-ti) |
| Threat Detection - Cross-SaaS apps, manual/automatic alert remediation, SIEM connector, MS Intelligent Security Graph, activity policies | **Security for Cloud Apps (MDA)** |
| SIEM connectors and integration for MDO | **MDO.** Verify it fits the [SIEM integration article](https://learn.microsoft.com/en-us/defender-office-365/siem-integration-with-office-365-ti) |
| SIEM connectors and integration for MDE | **MDE.** Use Microsoft Defender for Endpoint APIs |
| SIEM connectors and integration for MDA | **MDA.** Cross-SaaS apps, alert remediation, MS Intelligent Security Graph, activity policies |
| SIEM connectors and integration for MDI | **MDI via MDA.** MDI alerts available via MDA integration. Requires Unified SecOps and MDI connected to MDA |
| SIEM connectors and integration for Sentinel | **Sentinel.** [M365 Defender integration with Microsoft Sentinel](https://learn.microsoft.com/en-us/azure/sentinel/microsoft-365-defender-sentinel-integration) |
