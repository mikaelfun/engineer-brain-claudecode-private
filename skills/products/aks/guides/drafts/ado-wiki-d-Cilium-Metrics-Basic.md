# Metrics TSG (Basic Cilium)

***Please follow the generic [Metrics TSG](/Azure-Kubernetes-Service-Wiki/AKS/TSG/Monitoring/Network-Observability-\(Kappie\)/Metrics) first.***

### 1. Check Cilium Pods are running

Check that all cilium pods are running:

```shell
kubectl get pods -n kube-system -l k8s-app=cilium
```

If any of the pods are not running, you can describe the pod to see what the issue is:

```shell
kubectl describe pod -n kube-system <cilium-pod-name>
```

If the Pod is in a CrashLoop, or if there have been restarts, try getting previous logs using `-p`:

```shell
kubectl logs --timestamps -p -n kube-system <cilium-pod-name>
```

### 2. Check Pod Logs

If cilium pods are running, you can check the logs for the pod to check for issues:

```shell
kubectl logs --timestamps -n kube-system <cilium-pod-name>
```

Look for lines that start with err, warn, fail, or fatal:

```shell
kubectl logs --timestamps -n kube-system <cilium-pod-name> | grep -i "err\|warn\|fail\|fatal"
```

### 3. Check ConfigMap

If the customer is complaining about having too many metrics, double check the following.

Get the ConfigMap:

```shell
kubectl get configmap cilium-config -n kube-system -oyaml
```

and make sure hubble is disabled:
```yaml
  enable-hubble: "false"
```

### 4. Check Cilium Metrics are being generated

You can check that metrics are being generated by calling the metrics endpoints.

These are on node port 9962.

Either:
1. Run this from inside one of the nodes:

    ```shell
    curl http://<cilium-pod-ip>:9962/metrics
    ```

2. *OR* port-forward cilium Pod, then curl localhost:

    ```shell
    kubectl port-forward -n kube-system <cilium-pod> 9962
    # in another window
    curl localhost:9962/metrics
    ```

You should see metrics like the following:


### 5. Check if Prometheus is scraping metrics

#### If Using Azure Managed Prometheus

If you are using managed Prometheus, you can check if it is scraping the cilium metrics by following below steps.

1. Check if AMA-Metrics pods are running

    ```shell
    kubectl get pods -n kube-system -l dsName=ama-metrics-node
    ```

2. If they are running, then port forward one of the AMA-Metrics pod to see if it can identify cilium pods

    ```shell
    kubectl port-forward -n kube-system <ama-metrics-node-pod> 9090
    ```

3. Go to http://localhost:9090/targets in your browser to see if cilium pods are being discovered:

    ![alt text](../../../images/network-observability/prom-basic-cilium-targets.png)

    a. Can go to http://localhost:9090/service-discovery to check if service discovery is recognising cilium pods.

    ![alt text](../../../images/network-observability/prom-basic-cilium-service-discovery.png)

4. If cilium pods are not discovered then please check prometheus configuration (go to http://localhost:9090/config in your browser) and make sure "networkobservability-cilium" job is added to the scrape config.

    ```yaml
    - job_name: networkobservability-cilium
      honor_timestamps: true
      track_timestamps_staleness: false
      scrape_interval: 30s
      scrape_timeout: 10s
      metrics_path: /metrics
      scheme: http
      follow_redirects: true
      enable_http2: true
      relabel_configs:
      - source_labels: [__meta_kubernetes_namespace, __meta_kubernetes_service_name, __meta_kubernetes_service_port_name]
          separator: ;
          regex: kube-system;network-observability;cilium
          replacement: $1
          action: keep
      - source_labels: [__address__]
          separator: ;
          regex: (.*)
          target_label: __address__
          replacement: 10.224.0.4
          action: replace
      - source_labels: [__address__, __meta_kubernetes_service_port_number]
          separator: ;
          regex: ([^:]+)(?::\d+)?;(\d+)
          target_label: __address__
          replacement: $1:$2
          action: replace
      - source_labels: [__address__]
          separator: ;
          regex: (.*)
          target_label: instance
          replacement: aks-nodepool1-22539176-vmss000001
          action: replace
      metric_relabel_configs:
      - source_labels: [__name__]
          separator: ;
          regex: '|cilium_drop.*|cilium_forward.*'
          replacement: $1
          action: keep
      kubernetes_sd_configs:
      - role: service
          kubeconfig_file: ""
          follow_redirects: true
          enable_http2: true
    ```

5. If the job is not present in the prometheus config, please check ama pod logs:

    ```shell
    kubectl logs --timestamps -n kube-system <ama-metrics-node-pod>
    ```

    a. Look for the following logs:
    ```
    prometheus-config-merger::Updating scrape interval config for /opt/microsoft/otelcollector/default-prom-configs/networkobservabilityCiliumDefaultDs.yml
    prometheus-config-merger::Adding keep list regex or minimal ingestion regex for /opt/microsoft/otelcollector/default-prom-configs/networkobservabilityCiliumDefaultDs.yml
    prometheus-config-merger::Done merging 7 default prometheus config(s)
    ```

#### If Using BYO Prometheus

Can similarly ask customer to port-forward their prometheus server.

Then follow similar steps, making sure the scrape job exists etc.
