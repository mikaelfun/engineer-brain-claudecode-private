# Microsoft Entra Application Proxy Connector Event Logs

This article explains all the logs that are created by the Microsoft Entra private network connector and Microsoft Entra private network connector updater services. All of these logs get collected by the Microsoft Entra Application Proxy Data Collector Script.

## Event Viewer Logs

Event Viewer -> Applications and Services Logs -> Microsoft -> Microsoft Entra private network ->

### I. Connector -> Admin
Provider: Microsoft-Microsoft Entra private network-Connector

This log shows events related to the connector service (start / stop / errors), warnings & errors related to some network connectivity issues and Integrated Windows Authentication (Kerberos Constrained Delegation).

### II. Updater -> Admin
Provider: Microsoft-Microsoft Entra private network-Updater

This log shows events related connector update downloads and update installation.

### III. Connector -> Sessions (Analytic/Debug)
Provider: Microsoft-Microsoft Entra private network-ConnectorDiagnostics

**Logging is not enabled by default.** This log gives more insight about what the connector service is doing (request handling).

**Filtering the session log on transaction ID:**
```
*[EventData[Data[@Name="transactionID"] and (Data = "{REPLACE_WITH_THE_TRANSACTION_ID}")]]
```

### IV. Updater -> Sessions (Analytic/Debug)
**Logging is not enabled by default.** This log does not contain any useful information.

## Exporting Event Definitions

To export all the possible events generated by the provider:

```powershell
(Get-WinEvent -ListProvider "REPLACE_WITH_THE_PROVIDER").Events | %{
    $id = $_.Id; $desc = $_.Description
    $_.Level.DisplayName | % {
        New-Object -TypeName psobject -Property @{ Level = $_ ; ID = $id ; Description = $desc }
    }
} | Export-Clixml PrivateConnector.xml
```

The script must be executed on a connector server.

**Important:** Events related to Connector installation or crash etc. are recorded in the **Application log** (not the connector-specific logs).

## Best Practice

To display the Microsoft Entra private network connector events in readable format in Event Viewer on your computer, install the connector first (silent installation without registration):

```
MicrosoftEntraPrivateNetworkConnectorInstaller.exe REGISTERCONNECTOR="false" /q
```

## Debug Logs (Traces)

### a) Default Trace Logs

Location: `%ProgramData%\Microsoft\Microsoft Entra Private Network Connector\Trace\`

- **EntraPrivateNetworkConnector_GUID.log** - Every service start-up creates a new log file. Enabled by default. Includes call stack of exceptions caused by network issues, failed authentication requests (RequestId / error codes).

- **EntraPrivateNetworkConnectorUpdater_GUID.log** - Every updater service start-up creates a new log file. Enabled by default. Includes call stack of exceptions from network issues, failed authentication requests.

### b) Additional Tracing

Enabling additional tracing is possible using the `-ServiceTraceOn` switch for the Data Collector Script, which provides more insight about connector service operations.
