**Summary:**   
This guide provides steps to disable Windows Hello for Business, including preventing new user provisioning and removing strong credentials from Windows 10 PCs. It also explains the impact of the `certutil -deleteHelloContainer` command on the system.

[[_TOC_]]

# "certutil -deleteHelloContainer"
This command will delete PIN, face, and fingerprint data.

---

# Disabling Windows Hello for Business
Consider this scenario:  
After testing Windows Hello for Business, you want to disable it in your organization. How do you prevent new users from provisioning and also remove the strong credential from the user's Windows 10 PC?

 **Note:**  
Changing the Group Policy Object (GPO) [Computer or User] > Administrative Templates > Windows Components > Windows Hello for Business > Use Windows Hello for Business to Disabled is not sufficient to disable previously configured PCs.  
Also note, the following commands must be executed on each machine where Windows Hello for Business is configured and deployed.

To disable, follow these steps:  
- Change the GPO used to deploy Windows Hello for Business to Disabled to prevent further enrollment/provisioning.  
  [Computer or User] > Administrative Templates > Windows Components > Windows Hello for Business > Use Windows Hello for Business should be set to Disabled.

- Remove the Windows Hello for Business container on the local machine. Remove any biometric sign-in information on the local machine.  
  Launch `cmd.exe` in the user's context, NOT ELEVATED AS ADMIN.  
  In the window, execute the following command to delete the NGC container and biometric data:  
  _Note:_ This command will delete the container and the biometric data.  
  ```
  certutil -deleteHelloContainer
  ```

# What exactly happens on the system when we run the certutil -deleteHelloContainer command?

At a high level, the following are removed:  
- User certificates bound to a Windows Hello for Business key.
- Pre-generated data around Windows Hello for Business keys, such as pre-generated keys (keys generated by the popkeysvc) and key attestation certificates (AIK).

These keys and certificates work like any other key/certificate in Windowsthey are stored on the file system (generated by the TPM and only usable by the TPM).  
The Windows Hello for Business container of the calling user. The container is internally a folder structure in an undisclosed location so that customers are not tempted to play with its content.  
Some local Azure Active Directory (Azure AD) metadata which keep track of the link between the Azure AD user account and the Windows Hello for Business key. This does not include device registration, only Windows Hello for Business.

The command is local onlyit does not delete anything in Active Directory (AD) or Azure AD. It removes the Windows Hello for Business provisioning state on the local device for the current user. It removes the keys, certificates, the Windows Hello for Business container, and biometric enrollments.

The command is a troubleshooting tool, not intended for everyday use.  
There is a flaw with the design if it is being consistently run. Ideally, you run it to restore a client so they can reenroll. The original stated goal was to determine who was and was not enrolled in Windows Hello for Business.  
That can be done using the attribute. If users are clearing Windows Hello for Business on their own, that is a social engineering problem, not a technical one.  
Users have controls in Settings that allow them to delete their biometric data once they are enrolled. If they have a biometric enrollment on their device, they should always be able to access these controls.
