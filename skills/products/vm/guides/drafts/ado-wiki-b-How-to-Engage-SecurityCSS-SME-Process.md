> **IMPORTANT: FIRST CONFIRM THAT THE CUSTOMER IS PREMIER.**

# Summary

This article provides instructions on how to engage the Security CSS Subject Matter Expert (SME) when dealing with security-related cases. The goal is to ensure that Support Engineers (SE) and Support Escalation Engineers (SEE) can efficiently collaborate with the Security CSS SME to provide high-quality support to customers.

# Steps to Engage the Security CSS SME

## 1. Identify the Need for SME Engagement

- Determine if the case requires specialized security knowledge that goes beyond standard procedures.
- Common scenarios include complex security incidents, advanced threat detection, and specialized security configurations.

## 2. Gather Relevant Information

- Collect all necessary details about the case, including customer information, incident description, and any preliminary findings.
- Ensure that you have documented all steps taken so far and any relevant logs or evidence.

## 3. Contact the Security CSS SME

- Reach out to the Security CSS SME through **AVA2.0**.
- In the post opened in AVA2.0, the SME should engage the Security CSS engineers.
- Once engaged, the Security CSS engineer should review the case.

## 4. Collaborate with the SME

- Work closely with the SME to analyze the case and develop a resolution plan.
- Follow the SME's guidance and ensure that all recommended actions are documented and communicated to the customer.

## 5. Escalate if Necessary

- If the case requires further escalation, the SME will guide you on the appropriate steps to take.
- Ensure that all escalation procedures are followed, and the case is handed over to the appropriate team if needed.
