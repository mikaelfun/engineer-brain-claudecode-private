# Recommendations Disappear and Reappear - Troubleshooting Guide

## Summary

Troubleshooting cases when customers have recommendations that disappeared and reappeared or are not showing anymore without any interaction with them.

**Important:** If the recommendations already reappeared, we might not be able to provide a conclusive RCA of why they disappeared in the first place.

## Troubleshooting Steps

### 1. Check Permissions

Verify customer still has necessary permissions on both Advisor blade and target resources.

- Recommendations can only be seen if customer has at least **read permissions** on the resource.
- If user lost read access and gained it back, recommendations could disappear and later reappear with delay.
- Reference: [Permissions in Azure Advisor](https://learn.microsoft.com/en-us/azure/advisor/permissions)

### 2. Check for Dismissed/Postponed Recommendations

Confirm if customer or a coworker already dismissed/postponed the recommendation, or inadvertently followed the recommendation steps (e.g., enabled ADE as part of a hardening project).

- This would cause the originating service to stop sending the recommendation to Advisor (up to 24h to reflect).
- Verify via ASC: ASC > Subscription > Advisor Recommendations tab.
- Reference: [Find out recommendations shown to Customers](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/506377)

### 3. Check Advisor Configuration

Review if customer modified their Advisor configuration:

- Excluded the subscription containing the resource
- Modified VM CPU utilization settings and then reverted
- **Note:** Only resource groups with recommendations are shown in the dropdown (not all resource groups the user has access to)
- Check if customer is inadvertently filtering recommendations or changed the **Commitment period** (which filters Cost recommendations)
- Reference: [Advisor Best Practice](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/610911)

### 4. Check Service-Specific Advisor Blade

Ask customer to check the specific service's Advisor recommendation blade (if available) to confirm whether the recommendation is not being generated by the service or if Advisor is not showing it.

- Examples: CosmosDB service blade, AppServices service blade

## Known Issues

### Security Recommendations Service Bus Queue Clogging

There is a known issue with Security recommendations processed by service bus queues that get clogged often, causing security recommendations to not be visible at times.

- **Workaround:** Ask customer to use Defender for Cloud to see recommendations
- PG is redesigning the process (ADO work item: https://dev.azure.com/msazure/One/_workitems/edit/16306284/)
- No ETA currently available
- Reference: [Public documentation](https://learn.microsoft.com/en-us/azure/advisor/advisor-security-recommendations)
