# ARM Operations Basics — How It Works

## Summary

High-level overview of Azure Resource Manager (ARM) operations and the difference between Synchronous and Asynchronous Operations. Includes Kusto query examples for ARM data investigation.

---

## What is Azure Resource Manager?

Azure Resource Manager (ARM) is the deployment and management service for Azure. It provides a management layer that enables customers to Create, Update, and Delete resources in Azure subscriptions.

---

## ARM Operations Classification

### Synchronous Operations (Sync)

A Sync operation refers to the lifecycle of a single HTTP request during which ARM fulfills the request by running internal operations and interacting with target resource providers.

Sync data includes:
- Customer side: user agents and IP address
- Customer request details: request info and authorization
- ARM operation details: internal call stack traces and storage operations
- RP Sync operation terminal status: execution status and latency

### Asynchronous Operations (Async)

An Async operation is triggered when ARM or the target RP needs to fulfill a customer request asynchronously. ARM runs jobs to track and manage operation status for all parties.

Async data includes:
- Azure control plane Async operation end-to-end (E2E) status and latency
- Interdependency among operations among ARM RPs (e.g., AKS deployment)
- Customer workload info (e.g., Terraform and DataBricks deployments)

**Simplified flow**: User sends PUT → ARM forwards to RP → RP responds with async ACK → ARM creates long operation job to poll RP → long operation job tracks provision completion.

---

## ARM Data Key Concepts

| Concept | Description |
|---|---|
| **ActivityId** | GUID defining ARM atomic operations. Links traces and underlying services during HTTP requests. Always generated by ARM. |
| **CorrelationId** | GUID for grouping ARM atomic operations (e.g., grouping operations during an Async operation). Default: ActivityId of root HTTP request. End users can override via CorrelationId header. |
| **SubscriptionId** | Identifier of an Azure account / owner of Azure resources. |
| **PrincipalOid** | Principal Object ID — modern identifier for users, may be associated with RBAC permissions. |
| **PrincipalPuid** | Principal Personal User Identifier — classic administrator identifier used by RDFE only. |
| **ResourceURI** | `/subscriptions/{subscriptionId}/resourceGroups/{groupName}/providers/{providerNamespace}/{type}/{name}` |
| **TargetURI** | `Host Name + ResourceURI + API version` (e.g., `https://management.azure.com:443/subscriptions/.../providers/Microsoft.Compute/virtualMachineScaleSets/name?api-version=2022-03-01`) |

## Kusto Queries

### Synchronous Operations — HttpIncomingRequests + HttpOutgoingRequests

```kusto
cluster('armprodgbl.eastus.kusto.windows.net').database('ARMProd')
| macro-expand isfuzzy=true ARMProdEG as X (
    X.database('Requests').HttpIncomingRequests
    | where PreciseTimeStamp between (datetime(2021-09-27) ..1d)
    | where ActivityId == "4c626f3f-e5f4-4de7-9479-fd3d05b3ee43"
    | project
        Table = "HttpIncomingRequests",
        TaskName, PreciseTimeStamp, ActivityId, correlationId, operationName
)
| union (
    cluster('armprodgbl.eastus.kusto.windows.net').database('ARMProd')
    | macro-expand isfuzzy=true ARMProdEG as X (
        X.database('Requests').HttpOutgoingRequests
        | where PreciseTimeStamp between (datetime(2021-09-27) ..1d)
        | where ActivityId == "4c626f3f-e5f4-4de7-9479-fd3d05b3ee43"
        | project
            Table = "HttpOutgoingRequests",
            TaskName, PreciseTimeStamp, ActivityId, correlationId, operationName
    )
)
```

**Table roles:**
- `HttpIncomingRequests`: logs request received from user (HttpIncomingRequestStart) and response sent back (HttpIncomingRequestEndWithSuccess)
- `HttpOutgoingRequests`: logs ARM → RP outbound call (HttpOutgoingRequestStart) and RP response (HttpOutgoingRequestEndWithSuccess)

### Asynchronous Operations — JobExecutionStatus

> ⚠️ This table is now restricted to PG only.

```kusto
cluster('armprodgbl.eastus.kusto.windows.net').database('ARMProd')
| macro-expand isfuzzy=true ARMProdEG as X (
    X.database('Jobs').JobExecutionStatus
    | where PreciseTimeStamp between (datetime(2021-09-27) ..1d)
    | where correlationId == "4c626f3f-e5f4-4de7-9479-fd3d05b3ee43"
    | where jobType == "ResourceLongOperationJob"
    | project PreciseTimeStamp, ActivityId, correlationId, operationName, executionStatus, nextExecutionTime, jobCompletionStatus
)
```
