# Summary

This article provides instructions on how to work with a ticket where the customer has security vulnerabilities that can impact Microsoft products, including Azure, Windows, or Linux OS, and they need a Microsoft response.

# Expectations

We need to collect the information required for the CSS Security PM team to ensure timely resolution. While waiting for the CSS Security PM team to be engaged, continue running additional diagnostics.

**Questions to share with customers:**

1. How did you discover the attack?
2. What was the sign of the attack?
3. When did the issue start, and what is the alert displayed?
4. What is the number of machines affected, and what are the names of those machines?
5. Include all information (screenshots, alert ID if applicable, and others) that could help define the path in this case.

# Next Steps

1. Check information available at https://aka.ms/talkingpoints for reactive statements and guidance.
2. If unable to find necessary information, send an email to the CSS Security PMs team at **csspm@microsoft.com**, including:
   - Customer information
   - Incident description
   - Details collected from the questions above
3. The CSS Security PM team will assist and work to resolve the issue.
