## Collect Guest Logs

The first thing we want to do is to collect guest logs. The most common way to do this is via Inspect IaaS Disk in [Azure Support Center](https://azuresupportcenter.msftcloudes.com/), but there are multiple methods you can use and they are all explained in detail here: [Guest Logs Collection TSG](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/495009)

Once you have collected logs, proceed to either the Windows or Linux section below depending on the OS of the VM you are troubleshooting.

### Windows VM

For Windows VMs, Azure Support Center Resource Explorer has a feature called WinGuestAnalyzer to automatically collect & parse the Guest Logs.

It's described in the [Guest Logs Collection TSG](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/495009/Guest-Logs-Collection?anchor=winguestanalyzer)

So the first thing to do is to open that report and check for findings. If one of the findings matches the issue, apply the mitigation and check if it resolves the issue. If not, you might have to look into the Guest Logs yourself.

Here are the most useful logs for troubleshooting the Extensions:

- **C:\WindowsAzure\Logs\WaAppAgent.log** *(Guest Agent log where you can find the extension's download/install/enable/disable/etc. operations and their results)*
- **C:\WindowsAzure\logs\plugins\\<ExtensionName>\\<Extension>.log** *(Extension's own logs)*

Note that each extension has its own logging features, but we usually retrieve the following pattern:

- **CommandExecution.log**: This log contains the latest handler's execution and its result (e.g., enable, download, install, etc.).
- **CommandExecution_<Timestamp>.log**: This log contains the handler execution at the mentioned timestamp. This might get deleted with newer executions.
- **CustomScriptHandler.log**: This log is for CustomScriptHandler and shows details on its execution.
- **IaaSBcdrExtension<number>.log**: This log is for Azure Backup / VMSnapshot - it shows details on its execution. Note that it's not correctly ordered by timestamp.
- etc.

Open them with Visual Studio Code and search for Warnings / Errors at the time of the issue - or go directly to the end to search for why the extension is failing.

Also, don't forget to look for the Extension's **status files**, as they might also include interesting information.

e.g., *\Packages\Plugins\Microsoft.Compute.CustomScriptExtension\1.9.5\Status\1.status*

Look at the **StdOut** and **StdErr** fields where some extensions are reporting their output.

The following Warning is often seen, and it simply means that the extension was re-executed with the exact *same* settings - so The Guest Agent prevents it from running again to avoid mistakes. If one wants to force re-execution, use the [ForceUpdateTag](https://docs.microsoft.com/en-us/dotnet/api/microsoft.azure.management.compute.models.virtualmachineextension.forceupdatetag?view=azure-dotnet) property.

``` 
Current sequence number, 1, is not greater than the sequence number of the most recently executed configuration. 
```

If you spot an error and don't know what it means, [search the wiki](https://supportability.visualstudio.com/AzureIaaSVM/_search?text=&type=wiki&filters=Project%7BAzureIaaSVM%7D) for any TSGs that match the error/symptoms you're seeing. Many of the extension TSGs that we have will be in [this folder](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/554374).

### Linux VM

For Linux VMs, the Extensions are usually logging to the following places:

- **/var/log/waagent.log** - (Guest Agent log where you can find the extension's download/install/enable/disable/etc. operations and their results)
- **/var/log/azure/<ExtensionName>/<Extension>.log** - (Extension's own logs)

Open these files with Visual Studio Code or Notepad++ and search for Warnings / Errors at the time of the issue - or go directly to the end to search for why the Extension is failing.

If you spot an error and don't know what it means, [search the wiki](https://supportability.visualstudio.com/AzureIaaSVM/_search?text=&type=wiki&filters=Project%7BAzureIaaSVM%7D) for any TSGs that match the error/symptoms you're seeing. Many of the extension TSGs that we have will be in [this folder](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/554374).

**Note:** if the error message is encoded in LinuxDiagnostic log,


You can decode it with the following command.

```
echo "<error_message>" | base64 -d | pigz -d
```

#### Check for a Forked Linux Agent

The Linux Guest Agent is open source so it is possible that an image provider creates and/or installs a customized or "forked" version of the Linux Guest Agent. This normally isn't the case, but it is more common on Network Virtual Appliances (NVA) where the entire OS is extensively customized. If it is confirmed to be a forked GA then we are very limited to the extent that we can help as extensions may not work at all and the customer will have to contact the image provider for guidance.

Some common signs of this are:
- VM is an NVA. Common NVA vendors include, but are not limited to Fortinet, Barracuda, Kemp, Palo Alto, and Cisco
- Guest Agent logs aren't in their correct locations
- SSH is disabled on the image or if they can they have limited access to the shell or file system
- _Very_ outdated Guest Agent version

More information can be found in the [Forked GA article](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/494976)

## Check for the latest Guest Agent version

We require that customers meet a minimum version to be officially supported. You should be able to see the Guest Agent version by selecting the VM in ASC and looking under properties. You can click here to see if it meets the [minimum supported version](https://docs.microsoft.com/en-US/troubleshoot/azure/virtual-machines/support-extensions-agent-version) and if not see this section for steps to [update the GA](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/495025/Guest-Agent-Advanced_AGEX-Workflow?anchor=check-for-the-latest-guest-agent-version---and-upgrade).

## Known issues with Extensions

Analyze the logs and [search the wiki](https://supportability.visualstudio.com/AzureIaaSVM/_search?text=&type=wiki&filters=Project%7BAzureIaaSVM%7D) for any TSGs that match the error/symptoms you're seeing. Many of the extension TSGs that we have will be in [this folder](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/554374)

If you don't find your error there, you may try to uninstall the extension from the Azure Portal or through PowerShell/CLI and then reinstall it.

Useful links:
- [How to uninstall VM Extension](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/494970)
- [How to manually remove VM extension](https://supportability.visualstudio.com/AzureIaaSVM/_wiki/wikis/AzureIaaSVM/494967)

If this doesn't solve the issue, refer to the following resources to get help.
