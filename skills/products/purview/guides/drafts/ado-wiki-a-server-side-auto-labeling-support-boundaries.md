Use this Support Area Path (SAP) for support tickets with issues, questions, collaboration, or transfers related to Service issues in Sensitivity labels from Microsoft Purview Information Protection. Sensitivity labels in the context of a service refer to the application and management of these labels within cloud services and platforms. For example, in Microsoft Purview Compliance, sensitivity labels can be applied to control access to sensitive content created within services like Microsoft Teams, SharePoint, and OneDrive. These labels help enforce data handling policies by determining who can access the content, how it can be shared, and applying protection actions like encryption.             |<a href="https://dev.azure.com/ASIM-Security/Compliance/_wiki/wikis/Information%20Protection/8996/Troubleshooting-Scenarios-Server-Side-Auto-Labeling" target="_blank">Server Side Auto labeling (Opens in new window or tab)</a>                  |
