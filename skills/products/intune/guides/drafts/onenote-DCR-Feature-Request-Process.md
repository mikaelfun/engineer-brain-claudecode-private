# DCR (Design Change Request) / Feature Request Submission Process

**Source**: OneNote — Intune ICM Process / Intune Work-Item reference
**Status**: draft

## Overview

When a customer requires a feature that does not exist in Intune, submit a DCR (Design Change Request) through the M365 CRM portal.

## Portal

- **URL**: https://m365crm.microsoftcrmportals.com/dcrrequest/dcrrequest-create/

## Required Information

| Field | Description |
|-------|-------------|
| DCR Title | Short descriptive title |
| Case Number | Associated support case |
| Tenant Id | Customer tenant |
| AccountId | Customer account |
| Context ID | Context identifier |
| Break-Fix IcM | Link if related IcM exists |

## Scenario Section

1. Describe the customer's problem
2. How was the issue found
3. Support engineer's assessment of customer's design

## Impact Section

- **Blocking**: Adoption / PreSales / Deployment / None
- **Priority**: P0 (Must fix ASAP), P1 (Required for rollout), P2 (Nice to have)
- Include financial impact, project delays, lost productivity details
- **Category**: Data Loss / Reliability of service / Privacy / Security / Others
- Number of users/devices impacted
- Duration of the issue
- Frequency (Daily / Weekly / Monthly)

## Additional Fields

- Impact to end users/admins (mandatory)
- Workaround status and details
- Business Impact Statement:
  1. Business problem in detail
  2. Day-to-day business/Operations impact
  3. What changed in product/service to trigger this
  4. Latest date needed for resolution
