# Specification: Test Loop 双层数据源 — Live + Synthetic 分离

**Track ID:** testloop-data-layer_20260328
**Type:** Feature
**Created:** 2026-03-28
**Status:** Draft
**Source:** ISS-120

## Summary

为 test loop 增加双层数据源架构，将测试数据分为 Live（真实 D365 数据）和 Synthetic（本地生成模拟数据）两层，增大样本随机性，减少频繁拉取真实数据，同时覆盖更多边界场景。

## Context

当前 test loop 所有测试都使用 `cases/active/` 下的真实 case 快照（10 个 case），没有 mock 能力。问题：
- 真实 case 数量有限，样本单一
- 每次 E2E 测试都要 backup/restore 真实数据
- 无法测试边界场景（0 emails、超大 note、异常格式、compliance 不合规等）
- Live case 池静态硬编码在 env.yaml，不会动态更新

## User Story

As an Azure Support Engineer, I want test loop to use synthetic data for logic testing and real D365 data for integration testing, so that I can increase test coverage with diverse scenarios while reducing unnecessary D365 API calls.
