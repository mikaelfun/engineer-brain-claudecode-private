# Specification: Agent Monitor Session SSE 修复

**Track ID:** agent-monitor-sse_20260323
**Type:** Bug
**Created:** 2026-03-23
**Status:** Draft

## Summary

Agent Monitor 页面点开 session 后没有实时 SSE 消息显示。根因是 CaseSessionDetail 组件从 case 级别的聚合 bucket (`messages[caseNumber]`) 读取消息，而非 session 级别的 bucket (`sessionMessages[caseNumber:sessionId]`)。

## Problem Description

1. SSE 事件正确写入两个 bucket：aggregate `messages[caseNumber]` 和 per-session `sessionMessages[caseNumber:sessionId]`
2. Agent Monitor 的 CaseSessionDetail 只读取 aggregate bucket
3. 当多个 session 存在或 aggregate bucket 为空时，展开的 session 显示空白

## Acceptance Criteria

- [x] CaseSessionDetail 从 `sessionMessages[caseNumber:sessionId]` 读取消息
- [x] 回退兼容：如果 sessionId 不可用，fallback 到 case-level messages
- [x] 消息恢复（recovery）也按 session 维度工作

## Dependencies

None

## Out of Scope

- 后端消息持久化按 session 分离（当前 recovery 仍然是 case 级别）
