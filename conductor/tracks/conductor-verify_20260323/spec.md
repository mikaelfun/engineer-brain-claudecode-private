# Specification: /conductor:verify CLI Command

**Track ID:** conductor-verify_20260323
**Type:** Feature
**Created:** 2026-03-23
**Status:** Draft
**Source:** ISS-101

## Summary

增加 `/conductor:verify` CLI 命令，对齐 WebUI verify 按钮的完整生命周期操作（verify/mark-done/reopen），让 CLI 和 WebUI 的 issue 状态流转完全一致。

## Context

目前 CLI 通过 `/conductor:implement` 完成 track 后，track 变 `complete`、issue derive 为 `implemented`，但没有后续命令执行验证把 issue 推到 `done`。WebUI 已有完整的 verify 按钮逻辑（运行 unit + UI 测试 → 写入 metadata.json verification 字段 → derive 状态变化），CLI 侧缺少对应命令。

## Acceptance Criteria

- [ ] `/conductor:verify [trackId|ISS-xxx]` 运行 unit tests + UI tests，结果写入 `metadata.json` 的 `verification` 字段
- [ ] 验证通过时 issue derive 为 `done`；验证失败时保持 `implemented`
- [ ] `/conductor:verify --mark-done [trackId|ISS-xxx]` 跳过测试验证，写入 `verification.status = "skipped"`，issue derive 为 `done`
- [ ] `/conductor:verify --reopen [trackId|ISS-xxx]` 清除 verification 字段，issue 回到 `implemented`；同时更新 tracks.md 状态标记为 `[~]`
- [ ] 支持通过 ISS-xxx 反查 trackId（读取 issue JSON 的 trackId 字段）
- [ ] 前置检查：track.status 必须是 complete/completed 才允许 verify/mark-done
- [ ] 验证结果格式与 WebUI 一致：`{ status, method, result: { unitTest, uiTest, verifiedAt } }`

## Dependencies

- 现有 conductor 命令基础设施（`.claude/commands/conductor/`）
- WebUI 后端的 `conductor-reader.ts` 中 `deriveIssueStatus()` 已定义 verification → status 映射规则
- `conductor/workflow.md` 已定义 verification strategy（Unit Tests + UI Tests）

## Out of Scope

- 不修改 WebUI 后端代码（WebUI 的 verify 逻辑保持不变）
- 不修改 `deriveIssueStatus()` 映射规则
- 不增加 SSE/进度推送（CLI 直接在终端输出）

## Technical Notes

- 这是一个纯 CLI 命令定义文件（`.claude/commands/conductor/verify.md`）
- 命令逻辑由 Claude Code agent 执行，不需要 TypeScript 代码
- 验证过程直接调用 `npm test`（unit）和读取 plan.md Verification Plan（UI）
- metadata.json 写入格式需与 WebUI `issues.ts` 中的写入格式完全一致
