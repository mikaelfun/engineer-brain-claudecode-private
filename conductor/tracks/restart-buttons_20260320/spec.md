# Specification: Issue 页面添加重启前后端按钮

**Track ID:** restart-buttons_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft
**Issue:** ISS-012

## Summary

在 Dashboard Issues 页面添加重启前端/后端/全部的按钮，方便开发过程中验证修复后快速重启服务。

## Context

Dashboard 当前通过两个独立进程启动（开发模式）：
- 前端: `cd dashboard/web && npx vite --port 5173`
- 后端: `cd dashboard && npx tsx src/index.ts`

修复 bug 或实现 feature 后，需要手动停止并重启进程，操作繁琐。后续计划改为构建后一键启动，届时重启逻辑需适配。

## User Story

As a developer, I want restart buttons on the Dashboard so that I can quickly restart frontend/backend after code changes without switching to terminal.

## Acceptance Criteria

- [ ] Issues 页面有三个按钮：重启前端、重启后端、重启全部
- [ ] 点击后端重启按钮，后端进程重启，API 恢复可用
- [ ] 点击前端重启按钮，前端 vite 进程重启，页面自动重连
- [ ] 重启全部 = 先后端再前端
- [ ] 按钮有 loading 状态，防止重复点击

## Dependencies

- 后端需要新增 restart API 端点
- Windows 环境下进程管理（taskkill / kill-port）

## Out of Scope

- 生产环境/构建后一键启动模式（后续 track）
- PM2 / systemd 集成

## Technical Notes

- 后端自重启：先 spawn 替代进程再 exit，或由前端检测断连后轮询重连
- 前端重启：后端 spawn shell 杀 vite 进程（按端口 5173）再 spawn 新 vite
- Windows：`taskkill /F /PID` 或 `npx kill-port 5173`
- 前端断连后需轮询 /api/health 等待恢复
