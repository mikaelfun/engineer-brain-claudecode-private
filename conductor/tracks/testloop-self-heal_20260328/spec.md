# Specification: Test Loop 自愈能力

**Track ID:** testloop-self-heal_20260328
**Type:** Feature
**Created:** 2026-03-28
**Status:** Draft

## Summary

让 test loop 具备智能自愈能力：在任意阶段遇到反复失败时，自主检测 systemic failure pattern → 诊断根因 → 自动跳过受影响测试并注入 learning → 将框架级 bug 作为高优先级 fix item 加入下一轮 FIX 阶段。

## Context

EngineerBrain 的自动化测试框架通过 6 阶段状态机（SCAN→GENERATE→TEST→FIX→VERIFY→DONE）运行。当前 FIX/VERIFY 阶段遇到反复失败只会机械重试直到 retryCount 上限，无法识别 systemic failure pattern（如多个测试因 poll_for_completion 超时），也无法自主诊断根因。用户不得不从 supervisor session 手动诊断和下达 skip/fix 指令，这违背了"自我迭代和进化"的设计目标。

## User Story

As a test loop operator, I want the test loop to autonomously detect repeated failure patterns, diagnose root causes, skip stuck tests, and prioritize framework-level fixes so that the loop can self-heal without human intervention.

## Acceptance Criteria

- [ ] test loop 在 VERIFY/FIX 阶段检测到同一测试连续失败 2+ 次且 failure signature 相同时，自动触发 pattern analysis
- [ ] pattern analysis 能识别 systemic failure（多个测试因同一根因失败），并聚合为单一 framework-level issue
- [ ] 识别到 systemic failure 后，自动跳过受影响的测试（注入 skip directive + learning），不阻塞其他测试
- [ ] 将诊断出的框架级 bug 作为高优先级 fix item 加入 fixQueue（`priority: 1`, `category: framework`），在下一轮 FIX 阶段优先处理
- [ ] 所有自愈动作记录到 `tests/results/fixes/{pattern-id}-self-heal.md`，supervisor evolution 模式可展示
