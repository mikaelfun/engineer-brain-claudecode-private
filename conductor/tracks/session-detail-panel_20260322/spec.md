# Specification: Agent Monitor Session Detail Panel

**Track ID:** session-detail-panel_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft

## Summary

Add interactive session detail panels to the Agent Monitor page. Each session row can expand to show real-time SSE message stream, with chat input for case sessions and a terminate button for active sessions. Completed sessions are grouped and collapsed by default.

## Context

The Agent Monitor page (delivered in ISS-044 / `agent-session-view_20260321`) shows a unified view of all session types (case, implement, verify, track-creation) with grouping, filtering, and type-specific badges. However, sessions are currently summary-only — there is no way to view live progress, interact with a session, or terminate one from this page.

The CaseAIPanel component on the CaseDetail page already has the full pattern for SSE-driven message display, chat input, and session end. This track extracts and reuses that pattern for the Agent Monitor page.

## User Story

As a support engineer, I want to expand any session in Agent Monitor to see its live status messages and optionally interact with it, so that I can monitor and control all agent activity from a single page without navigating to individual case details.

## Acceptance Criteria

- [ ] Each session row in AgentMonitor is expandable (click to toggle detail panel)
- [ ] Expanded panel shows real-time SSE message stream (thinking, tool-call, tool-result, completed, failed, system) for case sessions
- [ ] For active/paused case sessions, a chat input box allows sending messages (resume session)
- [ ] Active sessions show a "Stop" button that calls the session end API
- [ ] Completed sessions are grouped under a collapsible "Completed Sessions" section, collapsed by default
- [ ] Message display reuses the same grouping/collapsing logic as CaseAIPanel (consecutive tool calls collapsed, consecutive thinking collapsed)
- [ ] On page refresh, persisted messages are recovered from the backend for case sessions

## Dependencies

- `agent-session-view_20260321` (completed) — provides the unified session list, type badges, status dots
- Existing `caseSessionStore` (Zustand) — provides per-case message storage
- Existing `useSSE` hook — provides real-time SSE events
- Existing `CaseAIPanel` message display components — `ProcessedMessageList`, `MessageBubble`, `CollapsedGroup`

## Out of Scope

- Implement/verify/track-creation session message streaming (these session types don't have per-session SSE events yet — they use different SSE channels)
- Session creation from AgentMonitor (users still start sessions from CaseDetail or Issues pages)
- Drag-and-drop reordering of sessions
- Session log export/download

## Technical Notes

- Case sessions have full SSE support: `case-session-thinking`, `case-session-tool-call`, `case-session-tool-result`, `case-session-completed`, `case-session-failed`
- Non-case sessions (implement, verify, track) use different SSE channels (`issue-implement-*`, `issue-verify-*`, `issue-track-*`) — these can show a simplified progress view (latest message only) but not full message streams
- The `ProcessedMessageList` component from `CaseAIPanel.tsx` should be extracted into a shared component so both pages can use it
- Chat input only makes sense for case sessions (which support `POST /case/:id/chat` resume)
- Terminate button maps to `DELETE /case/:id/session` for case sessions; implement/verify sessions don't have a terminate API yet
- Completed sessions separation: filter `allSessions` into active/paused group vs completed group, render completed under a separate collapsed card
