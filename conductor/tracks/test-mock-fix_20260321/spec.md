1. **`POST /issues/:id/start-implement`** — Now calls `updateTrackMetadata(trackId, { status: 'in_progress' })`

   `updateTrackMetadata(issue.trackId, { verification: undefined })` instead of

3. **Lock behavior test** — Same as #1, asserts `mockUpdateIssue` which is no longer called.

The `mockEnrichIssueFromTrack` defaults to identity function `(issue) => issue`, so the
returned issue always has the original status instead of the derived status.
