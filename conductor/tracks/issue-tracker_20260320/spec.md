# Specification: Dashboard Issue Tracker

**Track ID:** issue-tracker_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft

## Summary

Dashboard 增加 Issue Tracker 页面，支持在 WebUI 中直接记录问题/需求，查看所有 issue 列表（分页+筛选），并与 conductor track 关联。数据存储在本地 `issues/` 目录。

## Context

当前问题记录散落在 Finetune.txt、FEATURE_REQUESTS.md、conductor tracks 等多处。需要一个统一的 WebUI 入口来记录、查看和管理所有问题。

## User Story

As an engineer, I want to report issues directly from the Dashboard and see all issues in one place so that I don't need to switch to other tools or files to track problems.

## Acceptance Criteria

- [ ] CLI skill `/issue "描述"` 可直接创建 issue（Dashboard 挂了也能记录）
- [ ] Dashboard 新增 Issues 页面（顶部导航入口）
- [ ] 支持创建 issue：标题 + 描述 + 类型（bug/feature/refactor/chore）+ 优先级（P0/P1/P2）
- [ ] Issue 列表分页显示，支持按状态筛选（pending/tracked/in-progress/done）
- [ ] Issue 详情页显示完整描述，关联的 conductor track ID（如有）
- [ ] 数据存储在 `issues/` 目录（每个 issue 一个 JSON 文件）
- [ ] CLI 和 WebUI 读写同一个 `issues/` 目录，数据天然同步
- [ ] 已有 conductor tracks 的问题在 issue 列表中显示 tracked 状态
