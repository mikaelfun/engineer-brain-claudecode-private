# Specification: Create Track 交互提问突出展示

**Track ID:** track-question-prominent_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-049

## Summary

Create Track 流程中的交互提问当前混在 SSE 流式消息中（TrackProgressPanel 内联小表单），用户很难快速定位需要回答什么。将提问改为 Modal Dialog 形式，字体更大更突出，引用内容清晰展示。

## Context

Dashboard Issue Tracker 页面的 Create Track 功能通过 SSE 推送进度和问题。当前 `TrackProgressPanel` 中的 `QuestionForm` 以 amber 边框内联卡片形式展示，与 thinking/tool-call 消息混在一起，视觉层级不够突出。

## User Story

As a support engineer, I want the track creation questions to be displayed prominently in a dialog box with larger font, so that I can quickly identify what I need to answer without searching through streaming text.

## Acceptance Criteria

- [ ] Create Track 流程中的每个交互提问，以 Modal Dialog 形式展示（覆盖在 TrackProgressPanel 上方）
- [ ] 引用内容（预填 summary、user story 等）在对话框内清晰展示，与普通 SSE 流文本区分
- [ ] 用户回答后对话框关闭，SSE 流继续正常展示
- [ ] 现有 Track 创建功能不受影响（正常创建 spec + plan）

## Dependencies

- `dashboard/web/src/components/TrackProgressPanel.tsx` — 现有 QuestionForm 内联实现
- `dashboard/web/src/stores/issueTrackStore.ts` — pendingQuestions 状态管理
- `dashboard/web/src/api/hooks.ts` — useAnswerTrackQuestion mutation

## Out of Scope

- 不修改后端 SSE 协议或问题推送逻辑
- 不修改 Claude SDK 的 AskUserQuestion 拦截机制
- 不修改 Implement 流程的问题交互（仅影响 Create Track）

## Technical Notes

- 将 `QuestionForm` 从内联卡片改为 Modal/Dialog overlay
- 提问标题用更大字号（text-lg/text-xl），引用内容用 blockquote 样式
- Dialog 关闭后消息列表中保留问题记录（已有 question message kind）
