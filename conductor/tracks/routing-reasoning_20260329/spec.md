## Summary

将 casework SKILL.md Step B5 的固定 actualStatus→agent 枚举映射改为 LLM 轻量推理路由，读取 meta+summary+analysis+最近邮件后输出路由计划。同步更新 casework-fast-path.sh 的 NEED_AGENT 判断。

## Context

当前 B5 路由表问题：
1. 不复用已有 analysis（2h 前有 fresh 报告仍无条件 spawn troubleshooter，浪费 Opus tokens）
2. pending-customer + days<3 = 零行动（即使客户说"问题严重影响生产"）
3. pending-pg 永远无行动（即使 ICM 刚 Resolved）
4. researching 强制 troubleshoot（已有结论只需写邮件时也跑）
5. 不传递 emailType（drafter 的 auto 模式从头推断）

7 个 agent 交叉验证一致认为这是系统中**最刚性的核心决策点**，也是最值得改进的。
