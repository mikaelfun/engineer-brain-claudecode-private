# Specification: Cancel Track Creation Button

**Track ID:** cancel-track-creation_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft

## Summary

During track creation (SSE progress phase), provide a Cancel button so users can abort the process before the agent finishes. Currently there's no way to stop a track creation once started.

## Acceptance Criteria

- [ ] Cancel button visible in TrackProgressPanel header during active creation
- [ ] Cancel button hidden after completion/error or when a question is pending
- [ ] Clicking Cancel calls backend endpoint to stop tracking + reset issue to pending
- [ ] Backend sets a cancellation flag so the running agent's output is discarded
- [ ] SSE broadcasts cancellation event to update all connected clients
- [ ] After cancel, the panel shows "Cancelled" state with option to retry

## Technical Notes

- The Claude Agent SDK `query()` iterator cannot be directly aborted mid-run.
- Instead, use a cancellation flag: set `issueTrackState` to cancelled, issue status back to pending.
- The running async `trackAsync` / `resumeTrackSession` will eventually finish, but `finalizeTrackCreation` should check the flag and discard results.
