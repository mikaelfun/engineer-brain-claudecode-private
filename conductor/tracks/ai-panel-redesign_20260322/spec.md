## Summary

将 AI Assistant tab 从「日志控制台」重构为「工作流引擎」。参考 Issues 页面 TrackProgressPanel 的成熟架构，引入语义化消息模型、专用状态管理、question/answer 交互拦截、刷新恢复、状态驱动按钮逻辑。

## Context

Dashboard 中 Issues 页面的 create-track/implement 流程使用专用状态机架构（issueTrackStore + issue-track-state.ts），提供：语义化消息、显式生命周期状态、QuestionForm 交互卡片、刷新恢复、状态驱动按钮。AI Assistant tab 使用通用 session viewer，仅有扁平消息数组，无结构化交互能力，体验不可用。

## Acceptance Criteria

- [ ] 后端新建 `case-step-state.ts`，参考 `issue-track-state.ts`，支持 messages/isActive/pendingQuestion/cancelled
- [ ] 后端 `steps.ts` 添加 AskUserQuestion 拦截 → 发送 `case-step-question` SSE 事件
- [ ] 后端新增 `POST /case/:id/step-answer` 路由，接收用户回答并 resume session
- [ ] 后端新增 `GET /case/:id/step-progress` recovery 端点，返回 messages + isActive + pendingQuestion
- [ ] 前端新建 `caseAssistantStore.ts`，包含 messages/status/pendingQuestion per caseNumber
- [ ] 前端 `useSSE.ts` 添加 `case-step-question` 事件处理
- [ ] 前端 `CaseAIPanel.tsx` 重写：状态驱动按钮 + QuestionForm 卡片 + 按步骤显示消息

- [ ] 页面刷新后能恢复消息流 + pending question 状态
