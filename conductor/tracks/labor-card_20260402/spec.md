# Specification: Case 详情页 Labor 估算卡片

**Track ID:** labor-card_20260402
**Type:** Feature
**Created:** 2026-04-02
**Status:** Draft
**Issue:** ISS-210

## Summary

在 Case 详情页右侧边栏（AI Assistant 下方）新增 Labor Estimate 卡片组件，展示当天估算结果，支持编辑时间、写入 D365、忽略操作。

## Context

当前 labor estimate 只在 Daily Ops 页面批量展示。Case 详情页用户无法直接看到和操作单个 case 的 labor 估算。现有后端 API（`/labor-estimate/:caseNumber`）和前端 hooks（`useLaborEstimate`、`useLaborEstimateUpdate`）已完备，只缺 UI 组件。

## User Story

As a support engineer, I want to see and act on labor estimates directly in the case detail page, so that I can quickly confirm or adjust time without switching to Daily Ops.

## Acceptance Criteria

- [ ] AC1: Case 详情页右侧边栏显示 LaborEstimateCard，位于 CaseAIPanel 下方
- [ ] AC2: 卡片显示估算总分钟数、分类（classification）和简要描述
- [ ] AC3: 时间值可编辑（inline input），修改后通过 PUT API 保存
- [ ] AC4: "Write to D365" 按钮调用 submit API 写入 labor
- [ ] AC5: "Dismiss" 按钮隐藏卡片（当天不再显示，前端状态）
- [ ] AC6: 无估算数据时显示 "Estimate" 按钮触发 AI 估算
- [ ] AC7: 暗色/浅色主题适配

## Dependencies

- 现有 API: GET/PUT/POST `/labor-estimate/:caseNumber`、POST `/labor-estimate/:caseNumber/submit`
- 现有 hooks: `useLaborEstimate()`, `useLaborEstimateUpdate()`, `useLaborEstimateTrigger()`
- CaseDetail.tsx 右侧边栏布局

## Out of Scope

- 批量 labor 操作（ISS-205 范围）
- Labor calibration/preference 编辑
- record-labor.ps1 脚本修改

## Technical Notes

- 组件位置：`dashboard/web/src/components/LaborEstimateCard.tsx`
- 放置位置：`CaseDetail.tsx` 右侧栏 `CaseAIPanel` 下方
- 使用现有 TanStack Query hooks，无需新 API
- Dismiss 状态用 `sessionStorage` 或 Zustand 存储（刷新后重置）
