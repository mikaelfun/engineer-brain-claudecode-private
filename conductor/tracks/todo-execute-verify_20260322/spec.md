# Specification: Todo Execute Verification Mechanism

**Track ID:** todo-execute-verify_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-066

## Summary

Add an execution lifecycle to the Todo Execute button: executing state with spinner + timer, post-execution D365 read-back verification, and final success/failure status display.

## Context

The EngineerBrain Dashboard Todo page shows per-case todo items. 🟡 (yellow) actionable items (Note/Labor/SAP) have an "Execute" button that triggers D365 write operations via PowerShell scripts. Currently, clicking Execute fires-and-forgets: no visual feedback during execution, no verification that the write succeeded, and no final status indication.

## User Story

As a support engineer, I want to see real-time execution progress and post-execution verification when I click Execute on a Todo item, so that I can be confident the D365 write operation succeeded without having to manually check D365.

## Acceptance Criteria

- [ ] Click Execute → button immediately enters "Executing..." state with spinner + elapsed timer
- [ ] During execution, the button is disabled (no double-click)
- [ ] Backend streams SSE progress events (todo-execute-progress) during script execution
- [ ] After D365 write script completes, backend automatically runs the matching read-back script to verify
- [ ] Verification uses: fetch-notes.ps1 (for note), view-labor.ps1 (for labor), refresh-timeline.ps1 (for sap)
- [ ] Verification pass → button changes to "Execution Complete" (green/gray, disabled)
- [ ] Verification fail → button changes to "Execution Failed" (red, click to retry)
- [ ] SSE event (todo-execute-result) carries { success, verificationDetails } for frontend to render
- [ ] Todo markdown checkbox auto-checked [x] only after verification passes
- [ ] Failed execution does NOT auto-check the checkbox

## Dependencies

- Existing TodoView.tsx Execute button (ISS-066 base)
- Existing case-routes.ts POST /todo/:id/execute route
- Existing case-session-manager.ts executeTodoAction() function
- D365 verification scripts: fetch-notes.ps1, view-labor.ps1, refresh-timeline.ps1

## Out of Scope

- Batch execute (executing multiple Todo items at once)
- Undo/rollback of executed D365 operations
- Execute button for 🔴 (red) items (these remain human-decision only)
- Changing the existing Todo file format

## Technical Notes

- The verification step should run the read-back script AFTER the write script completes within the same agent session
- Timer is purely frontend (start on click, stop on SSE result event)
- SSE event types: `todo-execute-progress` (during), `todo-execute-result` (final)
- The agent prompt needs to be enhanced to include a verification step:
  1. Run write script → 2. Run read-back script → 3. Compare/confirm → 4. Report result
- Frontend state machine per-item: idle → executing → verifying → success | failed
