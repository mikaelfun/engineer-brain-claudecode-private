# Specification: Fix AI Assistant Session 消息流体验

**Track ID:** ai-assistant-session_20260322
**Type:** Bug
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-077

## Summary

AI Assistant tab 中操作按钮（data-refresh 等）的 session 消息流体验远落后于 Issues 页面 create-track 流程。存在四个核心问题：无时间戳、过程细节被过度压缩、完成后按钮不释放、不支持交互。

## Context

Dashboard 中存在两套 session 消息流 UX 模型：Issues 页面的 create-track 使用专用状态机（TrackProgressPanel），有丰富的生命周期事件、交互问答支持和恢复机制；AI Assistant tab 使用通用 session viewer，将 paused 的可恢复 session 当作"活跃"状态，导致按钮锁死，且消息展示过于精简。

## Problem Description

**Steps to Reproduce:**
1. 打开 Case Detail 页面，切换到 AI Assistant tab
2. 点击 data-refresh（或其他步骤按钮）
3. 观察消息流和按钮状态

**Root Causes:**
1. **按钮锁死**：`isLiveRunning = !!activeSessionId`，但 backend 步骤完成后 session 状态为 `paused`（非 `completed`），`getActiveSessionForCase()` 仍返回该 session → 按钮永远 disabled
2. **无时间戳**：`SessionMessageList` 中 `MessageBubble` 有 timestamp 数据但从不渲染
3. **过程细节缺失**：连续 tool-call/result 和 thinking 被折叠为 collapsed group，只显示 start + completion 两条
4. **不支持交互**：完成后 session 为 paused，chat input 依赖 `activeSessionId` 但按钮又被 disabled，无法发送消息

## Acceptance Criteria

- [ ] 步骤完成后，操作锁释放，其他按钮恢复可点击
- [ ] 消息气泡显示时间戳（HH:MM:SS 格式）
- [ ] 过程消息默认展开显示（不折叠 tool-call 和 thinking），保留手动折叠能力
- [ ] 步骤完成后，chat input 可用，支持向同一 session 发送消息进行交互

## Dependencies

依赖现有组件：
- `CaseAIPanel.tsx` — 按钮 disabled 逻辑
- `SessionMessageList.tsx` — 消息展示
- `useSSE.ts` — SSE 事件处理
- `case-session-manager.ts` — session 状态管理
- `steps.ts` — 后端步骤路由

## Out of Scope

- 重写为 Issues 页面完全一样的 TrackProgressPanel 架构
- AskUserQuestion 交互支持
- 步骤恢复/recovery 机制升级

## Technical Notes

核心修改点：
1. `CaseAIPanel.tsx`：按钮 disabled 逻辑改用 `hasActiveOperation`（操作锁），不再依赖 `activeSessionId`
2. `SessionMessageList.tsx`：MessageBubble 添加时间戳渲染
3. `SessionMessageList.tsx`：默认不折叠消息，或降低折叠阈值
4. `CaseAIPanel.tsx`：chat input 在有 paused session 时可用
