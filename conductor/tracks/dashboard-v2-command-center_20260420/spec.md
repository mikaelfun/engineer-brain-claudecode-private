# Specification: Dashboard v2 Redesign — Command Center

**Track ID:** dashboard-v2-command-center_20260420
**Type:** Feature
**Issue:** ISS-233
**Created:** 2026-04-20
**Status:** Draft

## Summary

Redesign the Dashboard homepage from a passive statistics page into an action-driven Command Center. Three-zone layout: Action+Todo list (urgency-scored) → Case Pipeline (4-column Kanban) → Health Metrics (SVG visuals) + Activity Feed (personal + team intel).

## Context

EngineerBrain Dashboard currently shows stat cards, severity/status bar charts, and response metrics. As a TSE managing 10-15 active cases across Azure products (VM, AKS, Azure Monitor, Entra ID) for 21V/Mooncake customers, the first question when opening Dashboard is "what should I do now?" not "how many cases do I have?". The current design answers the wrong question.

Design inspiration: Linear (minimal + keyboard-first), Plane (zero-config dashboard), Shortcut (WIP Limits), Geckoboard (data-to-ink ratio), PagerDuty (priority sorting + countdown), Datadog (real-time event stream).

Visual companion: `dashboard-redesign-brainstorm.html` (light mode mockup).

## User Story

As a Technical Support Engineer, I want the Dashboard homepage to show me prioritized actions, case pipeline status, and recent activity so that I can immediately start working on the most urgent item without navigating to multiple pages.

## Acceptance Criteria

- [ ] **AC1**: Dashboard shows an "Actions & Todo" list combining case actions and patrol/manual todos, sorted by urgency score (0-100), with score pill and factor badges visible per item
- [ ] **AC2**: Dashboard shows a 4-column Case Pipeline (Pending Engineer → Pending Customer → Pending PG → Ready to Close) with clickable cards that navigate to Case Detail
- [ ] **AC3**: Pipeline cards display severity badge, age, and business tags (S500/21V/RDSE)
- [ ] **AC4**: Health Metrics section uses SVG ring charts, stacked bars, sparkline, and pipeline mini-bar instead of plain text rows
- [ ] **AC5**: Activity Feed has two sections: "My Activity" (from case data/patrol) and "Team Intel" (known issues/LSI placeholder)
- [ ] **AC6**: Top bar shows greeting, time, key alert badges (SLA Risk / Need Action / Last Patrol), and Patrol trigger button
- [ ] **AC7**: Actions list supports tab filtering (All / Case / Todo / Manual)
- [ ] **AC8**: Todo page is removed from sidebar navigation; its functionality is absorbed into Dashboard Actions
- [ ] **AC9**: Urgency score is computed from 6 factors: prod_impact, customer_sentiment, severity, contact_gap, teams_pressure, sla_bonus
- [ ] **AC10**: Dashboard respects existing dark/light theme via CSS variables

## Dependencies

- Existing `/api/cases` endpoint (provides meta.actualStatus, severity, daysSinceLastContact, irSla, ccAccount, compliance)
- Existing `/api/todos` endpoint (provides patrol and manual todos)
- Existing case directory schema (`meta.json` fields)
- Existing design system (CSS variables, Card/Badge components)

## Out of Scope

- Drag-and-drop case reordering in pipeline (future enhancement)
- Real-time WorkIQ integration for Activity Feed (use static/cached data for now)
- Real-time Teams swarming integration for Team Intel (placeholder with mock data, implement ingestion in future track)
- Customer sentiment NLP analysis (use keyword heuristic for v1)
- Teams pressure detection via MCP search (use presence of Teams mentions in notes for v1)

## Technical Notes

- **Frontend only for Phase 1-3**: All data already available from existing APIs. Urgency score computed client-side in Phase 1, moved to backend in Phase 4.
- **Recharts available**: Already in tech stack for sparkline/charts, but prefer lightweight SVG for ring charts.
- **No new backend endpoints in Phase 1-3**: Reuse `/api/cases` + `/api/todos`. New `/api/actions` endpoint in Phase 4.
- **Design system**: Use existing CSS variables (`--accent-red`, `--bg-surface`, etc.) for theme compatibility.
- **File changes**: Primarily `Dashboard.tsx` (rewrite), `Layout.tsx` (remove Todo nav), new components in `dashboard/web/src/components/dashboard/`.
