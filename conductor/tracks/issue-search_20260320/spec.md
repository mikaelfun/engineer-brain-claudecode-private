# Specification: Issue Tracker Keyword Search

**Track ID:** issue-search_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft
**Source:** ISS-024

## Summary

Add a keyword search box to the Dashboard Issue Tracker page, allowing users to filter issues by matching title, description, or ID.

## Context

The Issue Tracker page (`Issues.tsx`) currently supports filtering by status and type dropdowns, but has no text search. With 20+ issues accumulating, finding a specific issue requires scrolling through the list.

## User Story

As an Azure Support Engineer, I want to search issues by keyword so that I can quickly find specific issues without scrolling.

## Acceptance Criteria

- [ ] A search input box is visible on the Issues page alongside existing filters
- [ ] Typing a keyword filters the issue list in real-time (case-insensitive)
- [ ] Search matches against issue title, description, and ID (e.g. "ISS-021")
- [ ] Search works in combination with existing status/type filters
- [ ] Clearing the search box restores the full filtered list

## Dependencies

Depends on existing code: `dashboard/web/src/pages/Issues.tsx`, `dashboard/src/routes/issues.ts`

## Out of Scope

- Full-text search engine / indexing
- Search history or saved searches
- Server-side search (client-side filtering is sufficient for ~100 issues)

## Technical Notes

**Approach:** Client-side filtering is simplest and sufficient for the expected issue count (<100). Add a `search` state to the Issues page, filter the fetched issue list before rendering. No backend changes needed.

Alternatively, add a `search` query param to the backend `GET /api/issues` and filter in `listIssues()`. This keeps the backend as the single filter source but adds complexity. Given the small dataset, client-side is preferred.
