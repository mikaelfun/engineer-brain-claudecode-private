# Specification: Improve Process Execution Visibility

**Track ID:** process-visibility_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft

## Summary

Case full process 执行时，UI 几乎只显示 "Thinking" 标签，看不到实际在做什么操作。需要在整个 SSE 管道中修复信息丢失问题，让用户能看到有意义的操作细节（正在执行哪个 step、调用了什么工具、tool 参数和结果摘要）。

## Context

Dashboard 通过 SSE 从 Claude Agent SDK 的 streaming 输出推送实时消息到前端。信息在多个环节被丢失或降级：
1. Backend `getSSEEventType()` 将未识别的 SDK 消息类型默认为 `thinking`
2. Backend `formatMessageForSSE()` 不提取 tool arguments
3. Frontend store 丢弃非字符串 content（`typeof content === 'string' ? '' : content`）
4. Chat route 硬编码所有消息为 `case-session-thinking`
5. 持久化恢复时类型名不匹配（`assistant` vs `thinking`）

## User Story

As an Azure Support Engineer, I want to see what operations are actually happening during case processing (which step, what tool is being called, what it found) so that I can monitor progress and understand results without waiting for completion.

## Acceptance Criteria

- [ ] Tool-call 消息显示工具名称和关键参数摘要（如 `Bash: pwsh scripts/get-case-info.ps1`）
- [ ] Tool-result 消息显示结果摘要（截取前 300 字符，非空）
- [ ] Step 切换有明显的视觉分隔（如 "📋 data-refresh started"）
- [ ] 非字符串 content（数组/对象）被正确序列化而非丢弃
- [ ] Chat route 也能区分 thinking/tool-call/tool-result
- [ ] 页面刷新后恢复的持久化消息类型正确渲染

## Dependencies

None

## Out of Scope

- 改变 CaseAIPanel 的整体布局设计
- 添加 tool-result 的结构化渲染（表格、JSON 树等）
- 添加进度百分比或进度条
- 修改 Claude Agent SDK 本身的 streaming 行为
