## Summary

Issue Tracker 页面中各状态分组（Pending/Tracked/Implementing/Implemented/Done）内的 issue 列表按 `updatedAt` 降序排列（最近修改的在最前），方便快速定位最新变动的 issue。

## Context

Dashboard Issue Tracker 页面将 issue 按状态分为五组展示。当前各组内 issue 顺序取决于 API 返回顺序（文件系统读取顺序），不保证按时间排序，用户难以快速找到最近变动的 issue。

## User Story

As a support engineer, I want issues within each status group sorted by last modified time (newest first) so that I can quickly find the most recently updated issues.

## Acceptance Criteria

- [ ] 各分组内 issue 按 `updatedAt` 字段降序排列
- [ ] 没有 `updatedAt` 的 issue fallback 到 `createdAt`
- [ ] 排序在客户端（前端）完成，不依赖后端排序

## Dependencies

## Out of Scope

- 后端 API 排序支持
