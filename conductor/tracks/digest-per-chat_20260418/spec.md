## Summary

## Problem Description

### 根因

Case 2602130040001034 的 Sushanth chat 文件 3404 字符，`read_file(cf, max_chars=3000)` 截断后丢失 4/17 客户同意关单的对话（3001-3404 区间）。

```

```

### 临时修复

## Acceptance Criteria

- [ ] 每个 chat 文件完整输入 LLM，不截断

- [ ] low relevance chat 仅输出 relevance 判定，不提取 key facts（省 token）
- [ ] OneNote digest 逻辑不受影响（独立函数）
