# Specification: Fix Create Track Question Dialog Not Showing

**Track ID:** track-question-fix_20260321
**Type:** Bug
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-027

## Summary

Create Track 按钮点击后，后端 agent 的 AskUserQuestion 交互提问无法在前端显示对话框，导致 agent 等待超时、track 创建失败。根因是 closure 时序 bug：`canUseTool` 回调读取 `sessionId` 时值仍为 `undefined`。

## Context

Dashboard Issues 页面的 Create Track 功能通过 Claude Agent SDK `query()` 启动 `/conductor:new-track`。Agent 会通过 `AskUserQuestion` 发起交互提问（如确认 track type、acceptance criteria 等），`canUseTool` 回调拦截这些提问并通过 SSE 广播到前端。前端 `TrackProgressPanel` 监听 `issue-track-question` 事件，显示 `QuestionForm` 让用户回答。

## Problem Description

### Steps to Reproduce

1. 打开 Issues 页面
2. 在某个 pending issue 上点击 "Create Track"
3. 后端 agent 开始执行 `/conductor:new-track`
4. Agent 发出 AskUserQuestion → `canUseTool` 拦截
5. 前端没有显示问题对话框
6. Agent 因 `interrupt: true` 停止，但用户无法回答，track 创建最终失败

### Expected vs Actual Behavior

**Expected:** AskUserQuestion 被拦截后，SSE 广播 `issue-track-question` 事件（含 sessionId + questions），前端显示 QuestionForm，用户回答后 resume session。

**Actual:** SSE 广播的 `sessionId` 为 `undefined`。前端 `useSSE.ts` 中 `if (d.sessionId && d.questions)` 条件为 false，`setIssuePendingQuestion` 不被调用，QuestionForm 不显示。

### Root Cause

**Backend timing bug in `trackAsync` (issues.ts:325-362):**

```typescript
const trackAsync = async () => {
  let capturedSessionId: string | undefined          // ← starts as undefined

  const canUseTool = createCanUseTool(id, () => capturedSessionId) // ← closure reads undefined

  const queryIter = query({ ... })
  const result = await processQueryMessages(id, queryIter, undefined) // ← iterates...
  capturedSessionId = result.sessionId                // ← assigned AFTER iteration complete
```

`canUseTool` fires during `processQueryMessages` iteration (when AskUserQuestion appears in the stream). At that moment, `capturedSessionId` is still `undefined` because it's only assigned on line 362 after the `await` completes. The closure `() => capturedSessionId` returns `undefined`.

**Frontend guard in `useSSE.ts:296`:**
```typescript
if (d.sessionId && d.questions) {
  setIssuePendingQuestion(issueId, d.sessionId, d.questions)
}
```
When `sessionId` is `undefined`, this guard blocks the store update, so `pendingQuestions` is never set and the QuestionForm never renders.

## Technical Notes

**Fix strategy: Shared mutable ref pattern**

将 `trackAsync` 中的 `let capturedSessionId` 替换为 `const sessionRef = { current: undefined as string | undefined }`。`createCanUseTool` 的 getter 改为 `() => sessionRef.current`。`processQueryMessages` 的 `sessionRef` 参数允许在迭代过程中实时写入 `session_id`，确保 `canUseTool` 能立即读到。
