## Summary

`/issue` skill 创建新 issue 前，先扫描已有 issues 列表检测相似项，给用户合并（活跃 issue）、reopen（已完成 issue）或单独新建的选择，避免重复 issue 产生。

## Context

当前 `/issue` skill（`.claude/skills/issue/SKILL.md`）直接创建新 issue，没有查重机制。随着 issues 数量增长（当前 180+），同一问题可能被重复创建。项目已有 issue 合并的实践（如 ISS-054 合并到 ISS-045），但依赖人工识别。

此功能是纯 SKILL.md 修改——改变 Claude 执行 `/issue` 时的行为流程，不涉及代码变更。

## User Story

As an engineer using `/issue`, I want the skill to automatically detect similar existing issues so that I can merge into or reopen them instead of creating duplicates.

## Acceptance Criteria

- [ ] AC1: 执行 `/issue` 时，在生成新 issue 前扫描所有 `issues/*.json`，提取 title 和 description 进行关键词匹配
- [ ] AC2: 匹配到活跃 issue（pending/tracked/in-progress/implemented）时，提示用户选择"合并到已有"或"单独新建"
- [ ] AC3: 匹配到已完成 issue（done + 有 closedReason/closedAt）时，提示用户选择"Reopen 并更新描述"或"单独新建"
- [ ] AC4: 合并/Reopen 场景提供描述处理选项：追加 / 替换 / 手动编辑
- [ ] AC5: Reopen 时清空旧的 trackId、verifyResult、closedReason、closedAt，status 回到 pending
- [ ] AC6: 无匹配时继续原有新建流程，用户体验无变化
