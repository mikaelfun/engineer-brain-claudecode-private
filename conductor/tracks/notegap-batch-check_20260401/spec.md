# Specification: Daily Ops NoteGap 批量检测 + 生成

**Track ID:** notegap-batch-check_20260401
**Type:** Feature
**Created:** 2026-04-01
**Status:** Draft
**Issue:** ISS-204

## Summary

在 Daily Ops 页面的 Note Gaps 区域增加 "Check All" 按钮，一键扫描所有活跃 Case 的 note gap 状态，对有 gap 的 case 自动生成 note draft。

## Context

Daily Ops 页面已有 Labor Estimate 的 "Estimate All" 批量功能。Note Gaps 区域目前只被动展示已有的 `note-draft.md` 文件，缺少主动触发检测的入口。用户需要在 CLI 逐个执行 `/note-gap` 才能生成 draft。

## User Story

As a support engineer, I want to batch-check all case note gaps from the Daily Ops page so that I can quickly identify and address cases that need note updates without running CLI commands individually.

## Acceptance Criteria

- [ ] Daily Ops 页面 Note Gaps 区域显示 "Check All" 按钮
- [ ] 点击后扫描所有活跃 Case，检测哪些有 note gap（≥ threshold 天未更新）
- [ ] 对有 gap 的 case 自动生成 `note-draft.md`（使用与 `/note-gap` skill 相同的规则）
- [ ] 检测过程中显示进度（检测中... X/Y cases）
- [ ] 检测完成后自动刷新 gaps 列表

## Dependencies

- 现有 `GET /api/note-gaps` 批量读取接口
- 现有 `note-gap-routes.ts` 单 case 处理逻辑
- `notes.md` 文件的时间戳解析

## Out of Scope

- 批量写入 D365（仍需用户逐个确认写入）
- 修改 note-gap 检测阈值 UI（保持使用 config.json）
- 自动刷新 notes.md（仅检测已有的 notes.md 文件）

## Technical Notes

Draft 生成逻辑是规则化的（固定 title "fangkun note"、YYMMDD-- 格式），可在 Node.js 中直接实现而不需要 LLM session，性能更好。核心步骤：解析 notes.md 最新时间 → 判断 gap → 从 case-summary.md 提取进展 → 生成 draft。
