# Specification: Fix data-refresh hang on TodoWrite tool call

**Track ID:** todowrite-hang-fix_20260322
**Type:** Bug
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-079

## Summary

Dashboard SDK session hangs when the agent calls `TodoWrite` (Claude Code built-in task management tool) during data-refresh step execution, because this tool requires REPL/terminal interaction not available in headless SDK mode.

## Context

The Dashboard step API (`POST /case/:id/step/data-refresh`) spawns a Claude SDK session to execute the data-refresh skill. The `createStepCanUseTool` callback in `steps.ts` only intercepts `AskUserQuestion` — all other tools pass through with `{ behavior: 'allow' }`. When the agent decides to call `TodoWrite` (a built-in Claude Code task management tool for tracking progress), the SDK subprocess hangs because `TodoWrite`/`TodoRead`/`TaskCreate`/`TaskUpdate`/`TaskList`/`TaskGet` are interactive REPL-only tools that don't function in headless SDK mode.

## Problem Description

**Steps to Reproduce:**
1. Dashboard → Case Detail → AI Panel → click "Data Refresh"
2. data-refresh skill executes Steps 0-3 normally
3. After writing data-refresh.log, agent calls `TodoWrite` tool
4. SDK session hangs — UI shows "AI is processing..." indefinitely

**Expected:** data-refresh completes all steps and returns normally.
**Actual:** Hangs on `TodoWrite` tool call, session never completes.

## Root Cause

In `dashboard/src/routes/steps.ts`, the `createStepCanUseTool` function only handles `AskUserQuestion`:

```typescript
if (toolName === 'AskUserQuestion') { ... }
return { behavior: 'allow' as const }
```

It does not block REPL-only tools like `TodoWrite`, `TodoRead`, `TaskCreate`, `TaskUpdate`, `TaskList`, `TaskGet`, `EnterPlanMode`, `ExitPlanMode`. These tools are meant for interactive Claude Code REPL sessions and hang or error in headless SDK mode.

## Acceptance Criteria

- [ ] `TodoWrite` and other REPL-only tools (`TaskCreate`, `TaskUpdate`, `TaskList`, `TaskGet`, `EnterPlanMode`, `ExitPlanMode`) are denied in SDK step sessions
- [ ] Denied tools return a helpful message telling the agent to skip task tracking and proceed with the actual work
- [ ] data-refresh step completes without hanging when the agent would otherwise call these tools
- [ ] Existing tool interception (AskUserQuestion) continues to work unchanged

## Dependencies

None

## Out of Scope

- Changing the data-refresh SKILL.md prompt (the fix should be defensive at the SDK session layer)
- Adding new tools to SDK sessions

## Technical Notes

- The fix is in `createStepCanUseTool` in `dashboard/src/routes/steps.ts`
- Should also apply to `processCaseSession` in `case-session-manager.ts` if it has the same gap
- The `allowedTools` list in `processCaseSession` is `['Read', 'Write', 'Bash', 'Glob', 'Grep', 'Agent']` which should already restrict built-in tools — but when resuming via `chatCaseSession` there's no `allowedTools` restriction, which may be the actual entry point for TodoWrite
