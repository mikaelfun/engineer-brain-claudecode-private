# Specification: Draft Tab 折叠优化

**Track ID:** draft-collapse_20260331
**Type:** Bug
**Created:** 2026-03-31
**Status:** Draft

## Summary

WebUI Case Detail 的 Draft tab 当前显示所有历史 draft，应只展示最近一次生成的 draft，其余 draft 参考 Todo History 的折叠模式收起。

## Context

Dashboard Case Detail 页面有多个 tab（Inspection、Draft、Todo 等）。DraftsTab 在 `CaseDetail.tsx` 中实现，直接循环渲染所有 draft。而 `DraftsPage.tsx`（全局 Drafts 页面）已有可折叠的 `DraftCard` 组件可复用。Todo tab 已有 History 文件选择器 + "Historical" 标签的参考模式。

## Problem Description

- Draft tab 循环渲染所有 draft，页面冗长
- 用户通常只关注最新一封 draft，历史 draft 占用大量视觉空间
- Todo tab 已有成熟的 history 折叠模式，draft tab 应保持一致体验

## Acceptance Criteria

- [ ] Draft tab 默认只展示最近一次生成的 draft（完整展开）
- [ ] 历史 draft 折叠显示（可展开查看），参考 todo history 的折叠模式
- [ ] 折叠的历史 draft 显示基本信息（类型、时间）便于识别
- [ ] 展开/折叠交互流畅，无闪烁

## Dependencies

- `DraftsPage.tsx` 中已有的 `DraftCard` 折叠组件
- `CaseTodoTab` 中的 history 模式作为 UX 参考

## Out of Scope

- Draft 编辑功能
- Draft 删除功能
- DraftsPage（全局页面）的改动

## Technical Notes

- 复用 `DraftsPage.tsx` 的 DraftCard 折叠逻辑或提取为共享组件
- 按创建时间排序，最新的展开，其余折叠
- 保持 Tailwind CSS 样式一致
