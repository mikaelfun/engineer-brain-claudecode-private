## Summary

将 `generate-todo.sh` 从纯 Bash if-else 规则矩阵改为混合架构：脚本提取结构化 context → LLM 读 context + todo recipes 生成个性化 todo → 用户编辑反馈循环演进 recipes。

## Context

当前 `generate-todo.sh` 有 7 个盲区：
1. 无严重性感知（SevA 和 SevC 同阈值）
2. 无 ICM 语境（ICM resolved 但 pending-pg 无 "通知客户" todo）
3. 无 analysis 结论引用（分析报告存在但 todo 不引用）
4. 无 Teams 消息感知
5. 固定 3/5 天阈值
6. 无历史感知（连续 N 次无变化不提示停滞）
7. 无 21v Convert 特殊处理

改造后效果对比：
- 当前: `🟡 客户 4 天无回复，建议发 follow-up 邮件`
- 推理: `🟡 客户 4 天未回复。上次请求 NSG flow logs 但客户可能不知如何获取 — 建议发 follow-up 并附 log 获取步骤（参考 analysis/20260329-1430-nsg-analysis.md）`
