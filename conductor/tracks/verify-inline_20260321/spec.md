# Specification: Verify 内联进度展示

**Track ID:** verify-inline_20260321
**Type:** Bug
**Created:** 2026-03-21
**Status:** Draft

## Summary

点击 Verify 按钮后，当前使用全屏 loading overlay + 完成后弹出结果 modal 的方式展示。应改为参照 Create Track 的 inline panel 模式，在 issue 行内实时展示测试进度和结果。

## Problem Description

**当前行为：**
1. 点击 Verify → 全屏 loading overlay ("Running tests... This may take up to 5 minutes")
2. 后端 `POST /api/issues/:id/verify` 同步执行 unit test + UI test（最长 5 分钟）
3. 完成后弹出 modal overlay 显示测试结果

**期望行为：**
1. 点击 Verify → issue 行内展开 inline panel（类似 TrackProgressPanel）
2. 后端异步执行测试，通过 SSE 实时推送进度（"Running unit tests..."、"Unit tests passed ✅"、"Running UI tests..."）
3. 测试结果在 inline panel 内展示，不弹窗
4. 页面刷新后可恢复进度状态

## Context

Create Track 已有完整的 inline progress 模式：
- 后端异步执行 + SSE 广播事件
- `issueTrackStore` Zustand 存储进度
- `TrackProgressPanel` 组件渲染
- `useSSE` hook 全局监听
- `GET /track-progress` 刷新恢复

Verify 需要复用这套模式。

## Acceptance Criteria

- [ ] 点击 Verify 后在 issue 行内展开 inline panel 显示实时进度
- [ ] 移除全屏 loading overlay 和结果 modal
- [ ] 后端 verify 改为异步执行，通过 SSE 推送 unit test / UI test 的开始、通过/失败、输出
- [ ] 测试结果（pass/fail + output）在 inline panel 内展示
- [ ] 页面刷新后可恢复 verify 进度状态（类似 track-progress recovery）
- [ ] verify 执行中不可重复点击
