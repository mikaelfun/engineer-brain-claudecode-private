# Specification: Test Loop 架构优化 — 批量处理 + subagent 隔离

## Summary

重构 test-loop 架构，从「每次 cron 触发只处理 1 个 item」改为「每次 loop 迭代批量处理当前 phase 的所有 queue items，用 subagent 隔离每个 item」。同时为 executor 增加 progress 日志机制，让 supervisor 实时可见执行进度。

## Context

当前 test-loop 设计（SKILL.md 第 367 行 "每轮只执行一个阶段的一个动作"）存在三个问题：
1. **Throughput 极低** — 19 个测试 × ~10min/个 = 3h+，每次 cron 触发 + 新 session 启动开销大
2. **Subagent 隔离无实际意义** — 每次只处理 1 个 item，spawn subagent 只是多余的 context 切换
3. **重复开销** — 每次新 session 都要重新读 env.yaml、learnings.yaml、safety.yaml

改为方案B后：主 session 编排 loop，遍历当前 phase 的所有 queue items，每个 item spawn 对应 agent（TEST=sonnet, FIX=opus, UI=haiku），agent 返回后立即更新 state.json。

## Acceptance Criteria
