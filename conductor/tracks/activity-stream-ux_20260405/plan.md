## Phase 2: Human-Readable Message Formatting

Replace raw event type display with meaningful summaries derived from event data payloads.

### Tasks

- [x] Task 2.1: Create `formatActivityMessage(event: ActivityEvent): string` utility in a new file `dashboard/web/src/utils/activityFormatter.ts`
- [x] Task 2.2: Implement message templates for each event type:
  - `test-state-updated` → "Phase {phase}, cycle {cycle} — Queue: {passed}P {failed}F {verified}V {regression}R"
  - `test-pipeline-updated` → "Pipeline moved to {stage} (Round {round})"
  - `test-discoveries-updated` → "Discovered {count} new issues"
  - `test-result-updated` → "{testId}: {status} ({duration})"
  - `test-evolution-updated` → "Framework evolution: {milestone}"
  - `runner-status-changed` → "Runner {status}" (started/stopped/paused)
  - `test-self-heal` → "Self-heal triggered for {testId}"
  - `test-learning` → "New learning captured"
  - `test-strategy` → "Strategy updated: {detail}"
  - fallback → original event type + detail for unknown events
- [x] Task 2.3: Integrate `formatActivityMessage` into ActivityStream component, replacing raw `event.type` / `event.action` display
- [x] Task 2.4: Update the `detail` column to show the formatted message instead of raw queue abbreviations

### Verification

- [ ] Each event type renders a human-readable sentence
- [ ] Unknown/new event types gracefully fall back to raw display
- [ ] Timestamps and phase badges still display correctly
