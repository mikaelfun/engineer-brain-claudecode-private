# Specification: WebUI Evidence界面 claims.json 路径不匹配

**Track ID:** evidence-claims-path_20260423
**Type:** Bug
**Created:** 2026-04-23
**Status:** Draft
**Issue:** ISS-242

## Summary

WebUI 的 Evidence Chain Tab 无法显示 claims 数据，因为后端 API 读取 claims.json 的路径与 troubleshooter 实际写入路径不一致。

## Problem Description

**步骤复现：**
1. 对某个 case 执行 troubleshooter 分析，生成 claims.json
2. 打开 Dashboard → Case Detail → Evidence Chain Tab
3. 显示 "No evidence chain" 空状态

**期望行为：** Evidence Chain Tab 正确加载并展示 claims 数据表格。

**实际行为：** 始终显示空状态，因为后端找不到 claims.json。

**根因分析：**

| 组件 | 路径 |
|------|------|
| 后端 API (`dashboard/src/routes/cases.ts:881`) | `join(caseDir, 'claims.json')` |
| troubleshooter 实际写入 | `{caseDir}/.casework/claims.json` |

后端 API 在 case 根目录查找 `claims.json`，但 troubleshooter 写入到 `{caseDir}/.casework/claims.json`。路径差了一层 `.casework/` 子目录。

## Acceptance Criteria

- [ ] 后端 API `GET /api/cases/:id/claims` 正确读取 `{caseDir}/.casework/claims.json`
- [ ] Evidence Chain Tab 正常渲染 claims 数据（已有 claims.json 的 case）
- [ ] 无 claims.json 时仍返回 `{ claims: null }`（向后兼容）
