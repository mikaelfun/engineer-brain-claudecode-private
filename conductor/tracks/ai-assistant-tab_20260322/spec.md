# Specification: Case 详情页全屏 AI Assistant 分页

**Track ID:** ai-assistant-tab_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-067

## Summary

在 Case 详情页的 tab 栏（与 Inspection / Emails 并排）新增 "AI Assistant" tab，提供全宽的 AI 交互界面。同时精简右侧栏为快捷操作入口，保留跳转按钮。

## Context

当前 CaseDetail 页面的 AI Assistant 以右侧栏形式展示（xl+ 屏幕宽度 w-64 = 256px），空间太小，消息历史和聊天交互体验差。用户需要一个全宽的 AI 交互界面，同时保留侧栏的快捷操作能力。

## User Story

As an Azure Support Engineer, I want a full-width AI Assistant tab in the Case Detail page so that I can interact with the AI assistant with more space for reading messages and typing responses.

## Acceptance Criteria

- [ ] CaseDetail tab 栏新增 "AI Assistant" tab，与 Inspection/Emails 等并排
- [ ] AI Assistant tab 占据完整内容区宽度，包含：操作按钮区、消息历史流、聊天输入框、Session 管理
- [ ] 右侧栏精简为快捷按钮（Full Process / Refresh 等）+ "Open AI Assistant" 跳转按钮
- [ ] 点击侧栏跳转按钮自动切换到 AI Assistant tab
- [ ] AI Assistant tab 和精简侧栏共享同一个 session 状态（不会出现两套独立状态）

## Dependencies

- 现有 `CaseAIPanel.tsx` 组件（重用其逻辑，拆分为 full 和 compact 两种模式）
- 现有 tab 系统（`Tabs` 组件 + `activeTab` state）

## Out of Scope

- 移除右侧栏（侧栏保留精简版）
- 修改 AI session 后端 API
- 移动端响应式布局调整（小屏仍保持底部 panel）

## Technical Notes

- CaseAIPanel 需要支持两种渲染模式：`compact`（侧栏）和 `full`（tab）
- tab 模式下布局需要调整：操作按钮横排 + 消息区域占满剩余高度 + 底部输入框
- 两种模式共享 hooks/state（session store、SSE events、operation status）
