# Specification: Agent Monitor Session Stop Buttons

**Track ID:** session-stop-btns_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-057

## Summary

Agent Monitor 的 Case Session 列表行直接显示 Stop 按钮（不需展开详情），并在 Case Session 分组区域增加"Stop All"一键释放所有活跃 session 的按钮。

## Context

目前 Stop 按钮只在展开 CaseSessionDetail 后的底部 action bar 中可见（L441-450）。用户需要：点击展开 → 滚到底部 → 点 Stop。当有多个活跃 session 时操作繁琐。

## User Story

As a support engineer, I want to quickly stop individual or all case sessions from the session list, so that I can release resources without having to expand each session.

## Acceptance Criteria

- [ ] 每个 active/paused 的 case session 行右侧直接显示 Stop 按钮（红色 ■ 图标，不展开即可点击）
- [ ] Case Session 分组头部增加"Stop All"按钮，点击可停止该分组内所有 active 的 case session
- [ ] Stop 按钮有 loading 状态（disabled + spinner），避免重复点击
- [ ] 停止后 session 状态自动刷新

## Dependencies

- `dashboard/web/src/pages/AgentMonitor.tsx`（前端唯一修改文件）
- 后端 `DELETE /api/case/:id/session` 已存在，无需修改

## Out of Scope

- 非 case 类型 session 的 stop 功能（implement/verify/track-creation 有独立生命周期）
- 后端 API 变更

## Technical Notes

- SessionRow 的 Stop 按钮需要 `e.stopPropagation()` 防止触发行展开
- "Stop All"按钮并行调用多个 `apiDelete`，用 `Promise.allSettled`
- Stop 按钮仅对 case 类型且 status 为 active/paused 的 session 显示
