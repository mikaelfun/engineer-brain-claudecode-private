## Summary

创建 email-recipes 目录（按场景索引），LLM 读 case context 匹配 recipe 生成草稿，用户发送后比较 draft vs 发送版本，提取模式更新 recipe。

## Context

当前问题：
1. email-drafter auto-selection 是固定 status→type 映射（粒度太粗）
2. pending-engineer 可能需要 direct-answer / acknowledge-logs / relay-ICM-findings，但只区分 follow-up 和 result-confirm
3. **最大空白**：系统从不学习用户对草稿的编辑（最有价值的反馈信号被丢弃）

用户改写草稿包含丰富信号：
- 改写 opening → tone 不对
- 添加 section → drafter 遗漏了重要 context
- 删除技术细节 → 对此客户太 verbose
- 改变 email type → auto-selection 判断错误
