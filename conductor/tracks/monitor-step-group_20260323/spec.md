# Specification: Agent Monitor Step-Grouped Message Display

**Track ID:** monitor-step-group_20260323
**Type:** Feature
**Created:** 2026-03-23
**Status:** Draft

## Summary

Agent Monitor 展开 case session 时，不同 step 的消息混在一起（如 teams-search started → status-judge thinking → teams-search completed），因为多个 step 共享同一个 SDK session。需要按 step 分组折叠显示。

## Context

当前 Agent Monitor 的 `CaseSessionDetail` 从 `caseSessionStore` 读取消息，直接传给 `SessionMessageList` 平铺渲染。由于 `stepCaseSession()` 会复用同一 SDK session（per-case 共享），多个 step 的消息交错在一起。

CaseAIPanel 已用 `caseAssistantStore`（按 executionId 分桶）解决了这个问题，但 Agent Monitor 走的是不同的数据路径。

## Acceptance Criteria

- [ ] Agent Monitor 展开 case session 时，消息按 step 分组显示
- [ ] 每个 step 组有 header（step 名称 + 状态标识：running/completed/failed）
- [ ] step 组可折叠/展开，默认展开最后一个 step，其余折叠
- [ ] 无 step 标记的消息归入一个通用分组
- [ ] 保持与现有 SessionMessageList 组件的兼容性（非 case session 不受影响）

## Dependencies

- ISS-082 已完成：step 消息已镜像到 caseSessionStore，每条消息有 `step` 字段

## Out of Scope

- CaseAIPanel 不变（它已有自己的 tab 机制）
- 不改 caseSessionStore 的数据结构

## Technical Notes

- 每条 `CaseSessionMessage` 已有 `step?: string` 字段（ISS-082 镜像时添加）
- 分组逻辑可在 `SessionMessageList` 内部处理，或在 `CaseSessionDetail` 预处理后传入
- 推荐在 `SessionMessageList` 新增一个 `groupByStep` prop，仅在 Agent Monitor case session 场景启用
