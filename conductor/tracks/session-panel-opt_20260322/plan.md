## Phase 1: Per-Session Message Store

Extend caseSessionStore to key messages by `${caseNumber}:${sessionId}` instead of just `caseNumber`, and update SSE listener to route messages to per-session keys.

### Tasks

- [x] Task 1.1: Extend caseSessionStore — add `sessionMessages: Record<string, CaseSessionMessage[]>` keyed by `${caseNumber}:${sessionId}`; keep existing `messages` (by caseNumber) for backward compatibility; add `addSessionMessage(caseNumber, sessionId, msg)` and `getSessionMessages(caseNumber, sessionId)` actions
- [x] Task 1.2: Update SSE listeners — in useSSE.ts, when handling case-session-thinking/tool-call/tool-result events, also call `addSessionMessage` with the sessionId from the event payload (alongside existing addMessage by caseNumber)
