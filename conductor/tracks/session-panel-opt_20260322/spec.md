# Specification: AI Assistant Session Panel Optimization

**Track ID:** session-panel-opt_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft

## Summary

优化 CaseAIPanel 中的 Session 管理区域：过滤已完成 session、调整布局位置、增加 Agent Monitor 跳转链接、添加 Stop 按钮、支持多 session 消息 Tab 切换。

## Context

CaseAIPanel (AI Assistant tab) 当前 session 历史显示在底部折叠区域，包含所有 session（含 completed）。消息面板只能显示当前 case 的最新消息流（keyed by caseNumber），无法切换查看不同 session 的消息。缺少对单个 session 的 stop 控制和到 Agent Monitor 的导航。

## User Story

As a support engineer using the AI Assistant tab, I want to see only active/paused sessions with stop controls and be able to switch between session message tabs, so that I can efficiently manage and monitor multiple concurrent sessions.

## Acceptance Criteria

- [ ] Session 列表只显示 active/paused session，不显示 completed
- [ ] Session 区域位于 AI Assistant tab 右侧面板下方（compact sidebar 模式下）
- [ ] 有链接可跳转到 Agent Monitor 页面
- [ ] 每个 active/paused session 有独立 Stop 按钮
- [ ] 有一键 Stop All 按钮（当多个 session 存在时）
- [ ] 消息面板支持多 Tab 切换不同 session 的实时消息

## Dependencies

- CaseAIPanel.tsx — 当前 session 历史和消息面板
- caseSessionStore.ts — 消息 store（当前 keyed by caseNumber，需扩展为 per-session）
- useSSE.ts — SSE 消息包含 sessionId，可用于 per-session 分流
- api/hooks.ts — useEndCaseSession, useEndAllCaseSessions, useCaseSessions

## Out of Scope

- Agent Monitor 页面本身的改动
- 历史 session 消息回看（只支持当前活跃 session 的实时消息流）
- Session 内搜索

## Technical Notes

- caseSessionStore 需要从 `Record<caseNumber, messages[]>` 扩展为 `Record<caseNumber:sessionId, messages[]>`
- SSE 事件已包含 sessionId，只需在 addMessage 时用 `${caseNumber}:${sessionId}` 作为 key
- Session Tab 切换可用简单的 selectedSessionId state
- Stop 按钮复用 useEndCaseSession hook，传入 sessionId
