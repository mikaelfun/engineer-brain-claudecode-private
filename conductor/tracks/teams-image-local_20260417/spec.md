# Specification: Teams Chat 图片本地化

**Track ID:** teams-image-local_20260417
**Type:** Feature
**Created:** 2026-04-17
**Status:** Draft
**Issue:** ISS-226

## Summary

Teams chat 消息中的 inline 图片（截图、日志截图等）以 Graph API hostedContents URL 存储，需要 Bearer token 认证。当前 write-teams.ps1 虽保留了 `![image](url)` markdown 格式，但 WebUI 无法加载 Graph API URL。本 track 实现图片本地化：data-refresh 阶段下载图片 → write-teams.ps1 替换 URL → generate-digest.py vision 分析 → WebUI 内联显示。

## Context

- write-teams.ps1 Strip-Html 已将 `<img src="url">` 转为 `![image](url)` markdown（commit ca8862a）
- CaseDetail.tsx TeamsTab 已改用 MarkdownContent 渲染（支持 img 标签 + rehype-raw）
- Graph API hostedContents URL 格式：`https://graph.microsoft.com/v1.0/chats/{chatId}/messages/{messageId}/hostedContents/{contentId}/$value`
- agency proxy（`agency.exe mcp teams --transport http --port PORT`）在 data-refresh 阶段运行，持有 Graph API auth
- OneNote 已有完整 vision + assets 管道（ISS-224 / track onenote-vision_20260417）可复用
- generate-digest.py 的 call_llm 已支持 vision（user_prompt 可为 list[dict]）

## User Story

As an Azure Support Engineer, I want Teams chat 中客户发送的截图（Portal 界面、错误信息、配置截图等）能被下载到本地、在 WebUI 中直接查看、并被 LLM 分析提取关键信息，so that 我无需手动打开 Teams 就能获取图片中的诊断线索。

## Out of Scope

- 视频文件下载（Teams 视频通过 Stream 托管，不在 hostedContents）
- 图片 OCR（依赖 LLM vision 内建能力）
- 非 hostedContents 图片（外部链接图片不下载）
- agency proxy 代码修改（只使用其现有能力）

## Technical Notes

- 图片下载需在 agency proxy 生命周期内完成（data-refresh.sh 的 teams 步骤内）
- 图片下载方式：通过 agency proxy 的 HTTP 端口转发 Graph API 请求，或从 agency 获取 token 后直接下载
- 文件命名策略：`{msgId}_{index}.{ext}`，避免冲突
- 图片大小一般 50-500KB/张，需设上限（跳过 >1MB 的图片）
- chat md 文件中同一图片可能在多次搜索中重复出现，需去重
