# Specification: Fix Agent Monitor Session Message Truncation

**Track ID:** session-msg-truncate_20260321
**Type:** Bug
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-053

## Summary

Agent Monitor 展开 session 查看实时状态消息时，消息内容被硬截断显示不完整，无法查看全文。

## Problem Description

`SessionMessageList.tsx` 中的 `MessageBubble` 组件对消息内容进行了多层硬截断：

| 截断点 | 限制 | 位置 |
|--------|------|------|
| MessageBubble (normal) | 200 字符后 `slice + …` | `SessionMessageList.tsx` |
| MessageBubble (compact) | 100 字符后 `slice + …` | `SessionMessageList.tsx` |
| Compact CSS | `line-clamp-2`（仅显示 2 行） | `SessionMessageList.tsx` |
| Collapsed thinking preview | 60 字符后 `slice + …` | `SessionMessageList.tsx` |
| Tool summary labels | CSS `truncate` | `SessionMessageList.tsx` |

用户展开 session 后看到的消息全部被截断，没有点击展开查看完整内容的方式。

### Steps to Reproduce

1. 打开 Dashboard → Agent Monitor 页面
2. 展开任意一个有消息的 session
3. 观察消息内容 — 较长的消息被截断为 `…`
4. 无法点击或展开查看完整消息

### Expected Behavior

- 消息默认可以显示适当长度的预览
- 用户可以点击/展开查看完整消息内容
- collapsed group 展开后应能看到每条消息的完整内容

### Actual Behavior

- 消息被硬截断到 200/100 字符，无展开方式
- 即使展开 collapsed group，内部消息仍被截断到 100 字符 + `line-clamp-2`

## Acceptance Criteria

- [ ] MessageBubble 正常模式：默认展示较长预览（如 500 字符），超出部分可点击展开查看全文
- [ ] MessageBubble compact 模式（collapsed group 内部）：展开后显示完整内容
- [ ] Collapsed thinking preview 保持 60 字符预览（这是 collapsed 状态，合理）
- [ ] 展开/收起状态切换流畅，不影响滚动行为

## Dependencies

- `dashboard/web/src/components/session/SessionMessageList.tsx`（唯一需要修改的核心文件）

## Out of Scope

- 后端消息存储/推送逻辑不变
- 前端 store 的 100 条消息上限不变
- collapsed group 的分组逻辑不变

## Technical Notes

- 修改集中在 `SessionMessageList.tsx` 的 `MessageBubble` 组件
- 添加展开/收起状态管理（per-message `useState` 或 per-bubble toggle）
- 保持现有的 `compact` prop 语义，但让 compact 消息在展开 group 后可点击查看全文
