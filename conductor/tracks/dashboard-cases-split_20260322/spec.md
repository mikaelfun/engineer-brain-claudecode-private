# Specification: Cases 列表独立页面 + Dashboard 统计可视化

**Track ID:** dashboard-cases-split_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-076

## Summary

将当前 Dashboard 页面中的 Case 列表拆分到独立的 "Cases" 导航页面，Dashboard 页面改为以统计指标和可视化卡片为主（SLA 状态分布、Case 分类饼图、响应时间指标、工作量趋势等）。

## Context

当前 Dashboard 页面同时承载统计卡片和完整 Case 列表，功能混杂。作为 Azure 支持工程师的日常工具，需要一个专门的统计概览页面快速掌握整体工作状态，同时 Case 列表应该有独立入口便于浏览和操作。

## User Story

As an Azure Support Engineer, I want the Dashboard to show high-level statistics and the Case list to have its own dedicated page, so that I can quickly assess my overall workload from the Dashboard and manage individual cases from the Cases page.

## Acceptance Criteria

- [ ] 侧栏新增 "Cases" 导航项，点击进入独立的 Case 列表页面
- [ ] Case 列表页面包含原 Dashboard 的全部 Case 功能（过滤卡片、排序、Case 卡片、操作按钮）
- [ ] Dashboard 页面移除 Case 列表，改为统计可视化卡片布局
- [ ] Dashboard 统计卡片至少包含：Case 分类分布（饼图/数字）、SLA 状态汇总、平均响应天数、最近巡检时间
- [ ] 路由 `/cases` 指向 Case 列表页面，`/` 指向新 Dashboard

## Dependencies

- 依赖现有 `useCases()` hook 和 `/api/cases` API
- 依赖现有 `Dashboard.tsx` 中的 Case 列表组件和 `classifyCase()` 逻辑

## Out of Scope

- 不添加图表库（echarts/chart.js 等），用 CSS + 数字卡片实现统计可视化
- 不修改后端 API
- 不修改 CaseDetail 页面

## Technical Notes

- Case 列表页面可直接从 Dashboard.tsx 中提取（stat cards + case list + sort + filter）
- Dashboard 统计卡片用纯 CSS（progress bar、数字高亮、简单 grid）实现，无需引入图表库
- 侧栏导航添加 Cases 项，位于 Dashboard 下方
