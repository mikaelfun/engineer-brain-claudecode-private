# Specification: Full Process 应继承已有 Session 而非总是新建

**Track ID:** session-reuse_20260323
**Type:** Bug
**Created:** 2026-03-23
**Status:** Draft

## Summary

`processCaseSession` 总是创建新 session，不检查 `caseIndex` 中是否已有可 resume 的 session。应像 `stepCaseSession` 一样先检查，有则 resume，避免孤儿 session。

## Problem Description

1. 用户执行 Step（如 Refresh）→ 创建 session A → paused
2. 用户执行 Full Process → `processCaseSession` 直接 `query()` 新建 session B → `caseIndex` 覆盖为 B
3. Session A 变成孤儿：仍在 `sessions` 中状态为 `paused`，但不可达

## Acceptance Criteria

- [ ] `processCaseSession` 在已有可 resume session 时 resume 而非新建
- [ ] 不影响 patrol（patrol 的 per-case agent 应独立）
- [ ] TypeScript 编译通过

## Dependencies

- `dashboard/src/agent/case-session-manager.ts`

## Out of Scope

- 历史孤儿 session 清理（后续可加 GC）
