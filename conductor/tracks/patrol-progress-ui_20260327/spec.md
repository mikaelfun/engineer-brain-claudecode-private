# Specification: Dashboard Patrol 实时进度展示

**Track ID:** patrol-progress-ui_20260327
**Type:** Bug
**Created:** 2026-03-27
**Status:** Draft
**Source:** ISS-094

## Summary

Dashboard 页面点击 Start Patrol 后没有任何实时进度反馈。按钮瞬间恢复为 "Start Patrol"，用户不知道 patrol 是否正在运行、处理到哪个 case 了。

## Context

EngineerBrain Dashboard 是 Azure Support Engineer 的工作台，Patrol 是批量巡检功能，会依次处理所有活跃 Case。

当前架构中 SSE 基础设施完整：
- 后端 `/api/events` SSE 端点已存在，patrol 事件（`patrol-progress`、`patrol-case-completed`、`patrol-updated`）已在广播
- 前端 `useSSE` hook 已全局挂载，`patrolStore`（Zustand）已接收 SSE 事件
- Agent Monitor 页面已经消费 `patrolStore` 展示实时进度

**唯一的问题是 Dashboard 页面没有接入 `patrolStore`。**

## Problem Description

### Steps to Reproduce
1. 打开 Dashboard 页面（`/`）
2. 点击 "Start Patrol" 按钮
3. 观察页面

### Expected Behavior
- 按钮变为 running 状态（如 "Patrolling..." + 取消按钮）
- 显示实时进度：当前处理哪个 case、已处理/总数、进度条
- Patrol 完成后按钮恢复，展示结果摘要

### Actual Behavior
- 按钮短暂显示 "Patrolling..."（约 1 秒，因为 POST 请求瞬间返回 202），然后恢复为 "Start Patrol"
- 没有任何进度指示
- 用户完全不知道 patrol 是否在运行

### Root Cause
`Dashboard.tsx` 的 Start Patrol 按钮仅使用 `startPatrol.isPending`（TanStack Query mutation state）来控制 loading 状态。由于 `/api/patrol` 返回 202 accepted 后 mutation 立即结束，按钮瞬间恢复。Dashboard 没有读取 `patrolStore` 的 `isRunning`、`phase`、`processedCases` 等实时状态。

## Acceptance Criteria

- [ ] Start Patrol 按钮在 patrol 运行期间保持 disabled + 显示 running 状态
- [ ] Dashboard 显示 patrol 实时进度（当前 case、已处理数/总数、进度条）
- [ ] Patrol 运行期间提供 Cancel 按钮
- [ ] Patrol 完成后显示结果摘要（处理了多少 case、发现了多少变化）
- [ ] 页面刷新后如果 patrol 仍在运行，能恢复显示进度状态
