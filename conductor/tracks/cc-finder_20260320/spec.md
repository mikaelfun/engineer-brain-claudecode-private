# Specification: CC Finder Integration

## Summary

Integrate RDSE customer CC (Customer Contact) lookup into the casework workflow. During compliance-check, automatically match case customer name against `mooncake-cc.json` and persist CC emails. Email-drafter consumes CC data for initial-response drafts, and inspection-writer adds a todo reminder.

## Context

RDSE customers have designated TAM/SDM contacts that must be CC'd on initial response emails. The data source (`mooncake-cc.json`) and scraping script (`fetch-powerbi-cc.js`) already exist but are not integrated into the casework workflow.

## User Story

As an Azure Support Engineer, I want the CC email list for RDSE customers to be automatically found and displayed in my initial response email draft, so that I don't have to manually look up CC contacts for each case.

## Acceptance Criteria

- [ ] compliance-check fuzzy matches case customer name against `mooncake-cc.json` accounts
- [ ] Matched CC emails are written to `casehealth-meta.json` (fields: `ccEmails`, `ccAccount`, `ccKnowMePage`)
- [ ] `<Replace with POD alias>` placeholder is replaced with `config.json → podAlias` (default: `mcpodvm@microsoft.com`)
- [ ] Unmatched customers are silently skipped (no CC fields written)
- [ ] email-drafter adds `CC:` header line in initial-response drafts when `ccEmails` exists
- [ ] Non-initial-response drafts do not include CC line even if `ccEmails` exists
- [ ] Dashboard draft renderer shows a copy button next to CC line (copies email list only, no prefix)
- [ ] inspection-writer adds a todo reminder with CC list when `ccEmails` exists
- [ ] `config.json` has new `podAlias` field, editable via Dashboard Settings

## Dependencies

- Existing `mooncake-cc.json` data file at `{dataRoot}/mooncake-cc.json`
- Existing compliance-check skill (`.claude/skills/compliance-check/SKILL.md`)
- Existing email-drafter agent (`.claude/agents/email-drafter.md`)
- Existing inspection-writer skill (`.claude/skills/inspection-writer/SKILL.md`)
- Dashboard draft rendering components

## Out of Scope

- Auto-refresh of mooncake-cc.json (manual script already exists)
- CC lookup for non-RDSE customers
- CC in follow-up/closure emails
- Auto-sending emails with CC (engineer always sends manually)

## Technical Notes

- Fuzzy matching: account names contain multi-alias strings with `/` separator, mixing Chinese and English
- mooncake-cc.json has ~10 accounts, simple in-memory scan is sufficient
- CC email list format: semicolon-separated (`;`)
- Know-Me page link is nullable
