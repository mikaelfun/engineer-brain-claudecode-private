# Specification: SYNTHESIZE Topic-Level Incremental Regeneration

**Track ID:** synthesize-topic-regen_20260419
**Type:** Feature
**Created:** 2026-04-19
**Status:** Draft
**Issue:** ISS-230

## Summary

Replace the product-level "20 entries threshold" for incremental SYNTHESIZE with topic-level on-demand regeneration. Only topics whose inputs have changed get regenerated, reducing agent cost from ~100/product to ~3-6 per change event.

## Context

The product-learn skill's SYNTHESIZE pipeline currently uses a blunt threshold: if fewer than 20 new JSONL entries exist since last synthesis, incremental synthesis is skipped entirely. This causes three problems:
1. High-value single entries (score 9.5 critical bug fixes) are ignored until 20 entries accumulate
2. Draft file content changes don't independently trigger workflow regeneration
3. When synthesis does trigger, it rebuilds ALL topics even if only 1-2 changed

## Acceptance Criteria

- [ ] A single new JSONL entry triggers regeneration of its matched topic (Agent-A + B + C)
- [ ] Draft file content hash change triggers regeneration of associated topic's workflow
- [ ] Kusto query file content hash change triggers regeneration of associated topic's workflow
- [ ] Unassigned entries (no topic match) accumulate; small-range clustering triggers at >=10
- [ ] Phase 1 full re-clustering is NOT triggered for matched entries (only for unassigned overflow)
- [ ] synthesizeState tracks kqlHashes and topicRegenLog
- [ ] The old 20-entry threshold code path is removed
