## Summary

为 dashboard/web 配置 Vitest + React Testing Library + jsdom 环境，建立组件/hook 测试模板和共享 mock 工具，使前端代码具备可测试性。

## Context

当前 dashboard/web/ 有 5 个组件、1 个 hook、8 个页面，但零测试覆盖。后端（dashboard/src/）已有 Vitest 配置（node 环境，3 个 .test.ts），但前端缺少独立的 jsdom 环境配置。conductor implement 流程中 UI 变更只能靠 ad-hoc Playwright 脚本验证，缺乏组件级回归保障。

## Motivation

建立前端测试基础设施后，每个 feature/bug track 可自动生成组件级测试，提升交付质量和回归信心。

## Success Criteria

- [ ] `cd dashboard/web && npx vitest run` 能跑通
- [ ] 至少 1 个组件测试示例（如 SessionBadge.test.tsx）
- [ ] 至少 1 个 hook 测试示例（如 useSSE.test.ts）
- [ ] 共享 test-utils/ 目录（含 render helper、mock router、mock query client）
- [ ] dashboard/ 根的 npm test 能同时跑后端 + 前端测试（或提供独立命令）
- [ ] workflow.md 补充前端测试 mock patterns 和约定

## Dependencies

- 现有 dashboard/vitest.config.ts（后端，node 环境）— 需共存不冲突
- React 18.3.1 + React Router 7.1.1 + TanStack Query 5.x — 测试工具版本需匹配

## Out of Scope

- 不写全量组件/页面测试（只提供模板示例）
- 不配置 Playwright E2E 测试框架
- 不改现有后端测试配置

## Technical Notes

- 前端 Vitest 使用 jsdom 环境，独立于后端 node 环境
- dashboard/ 根可用 Vitest workspace 或 npm script 并行跑两套测试
- React Testing Library 配合 @testing-library/user-event 模拟用户交互
