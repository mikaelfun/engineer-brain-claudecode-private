# Specification: Case 列表显示 Refresh 时间、Teams 消息截止时间、Refresh/Full Process 按钮

**Track ID:** case-list-actions_20260322
**Type:** Feature
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-060

## Summary

在 Dashboard Case 列表为每个 case 卡片增加：数据刷新时间（fetchedAt）、Teams 消息最后时间、以及 Refresh Data / Full Process 操作按钮。

## Context

当前 Dashboard.tsx 以 card 列表展示活跃 case，仅显示基本信息（severity, status, customer, age）。刷新时间和 Teams 时间戳数据在磁盘上可用（case-info.md snapshot header、teams/_chat-index.json），但 GET /api/cases 返回的 CaseSummary 未包含这些字段。Refresh Data 和 Full Process 按钮仅存在于 CaseDetail 的 CaseAIPanel 中，用户必须逐个打开 case 才能操作。

## User Story

As a support engineer, I want to see when each case's data was last refreshed and Teams messages were last fetched directly in the case list, and trigger refresh/process actions without opening each case, so that I can quickly identify stale cases and take action in batch.

## Acceptance Criteria

- [ ] Case 列表每个 card 显示 fetchedAt 时间（相对时间格式，如 "5m ago"）
- [ ] Case 列表每个 card 显示 Teams 消息最后时间（从 _chat-index.json 提取最晚的 lastMessageTime）
- [ ] 每个 case card 有 Refresh 按钮，点击触发 POST /api/case/:id/step/data-refresh
- [ ] 每个 case card 有 Full Process 按钮，点击触发 POST /api/case/:id/process
- [ ] 按钮在有活跃 operation 时禁用（使用 GET /api/operations/active 数据）
- [ ] 后端 GET /api/cases 返回扩展的 CaseSummary 包含 fetchedAt 和 teamsLastMessageTime

## Dependencies

- 现有 `POST /api/case/:id/step/data-refresh` 端点
- 现有 `POST /api/case/:id/process` 端点
- 现有 `GET /api/operations/active` 端点
- 现有 `case-reader.ts` 的 `extractSnapshotTime()` 方法

## Out of Scope

- 批量 Refresh/Process 所有 case
- Case 列表改为 table 布局（保持 card 列表）
- 实时 SSE 进度在列表中展示（进度仍在 CaseDetail 查看）

## Technical Notes

- `fetchedAt` 从 case-info.md 的 snapshot header 解析（已有 `extractSnapshotTime()`）
- Teams 最后消息时间从 `teams/_chat-index.json` 的各 chat 的 `lastMessageTime` 取最大值
- 按钮触发后无需在列表等待完成，可跳转到 CaseDetail 查看进度
