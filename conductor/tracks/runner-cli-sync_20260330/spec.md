# Specification: TestLab Runner 状态与 CLI Run 同步

**Track ID:** runner-cli-sync_20260330
**Type:** Enhancement
**Created:** 2026-03-30
**Status:** Draft
**Issue:** ISS-178

## Summary

从 CLI 执行 `/test-supervisor run` 时，WebUI TestLab 的 runner 控制条（Start/Stop 按钮、状态指示器）无法感知运行状态，始终显示 idle。需要让 runner status API 也检查 supervisor.json 的 status 字段，实现 file-based 外部 run 检测。

## Context

当前 runner 状态（`runnerState`）是 `dashboard/src/routes/test-runner.ts` 中的内存变量，只有通过 `POST /api/tests/runner/start` 发起的 run 才会更新。CLI 直接 spawn Agent 修改 `tests/supervisor.json` state 文件，file-watcher 能捕获 pipeline/queues/stats 变化并广播 SSE 事件，但 runner 控制层不知道有 run 在进行。

## Problem Description

- Pipeline 进展、阶段变化、队列数据能正常实时显示 ✅
- Runner 状态条显示 idle（应显示 running）❌
- Stop 按钮无效（没有 abortController）❌

## Acceptance Criteria

- [ ] AC1: GET /api/tests/runner/status 在内存 runnerState 为 idle 时，检查 supervisor.json 的 status 字段，若为 running 则返回 `{ status: 'external', source: 'cli' }`
- [ ] AC2: TestLab 前端 runner 控制条识别 `external` 状态，显示 "External run detected" 状态指示
- [ ] AC3: Stop 按钮在 external run 时写入 pause directive（`tests/supervisor.json` 的 `directive: "pause"`）而非依赖 abortController
- [ ] AC4: 当外部 run 结束（supervisor.json status 回到 idle/completed），runner 控制条自动恢复 idle 状态
- [ ] AC5: 通过 WebUI Start 启动的 run 行为不受影响（向后兼容）

## Dependencies

- `dashboard/src/routes/test-runner.ts` — runner 状态管理
- `dashboard/src/services/test-reader.ts` — supervisor.json 读取
- `dashboard/web/src/pages/TestLab.tsx` — 前端 runner 控件
- `tests/supervisor.json` — 状态文件

## Out of Scope

- 让 CLI run 完全接管 WebUI runner state（保持双轨：内存 for WebUI, file for CLI）
- 修改 test-supervisor SKILL.md 的执行流程
- 新增 SSE 事件类型（复用现有 `runner-status-changed` 事件）

## Technical Notes

- `test-reader.ts` 已有 `readSupervisorState()` 方法可直接复用
- file-watcher 已监听 supervisor.json 变化并发 SSE，可在 `runner-status-changed` 事件中携带 external 标记
- Stop 的 pause directive 机制已存在于 test-supervisor SKILL.md
