# Specification: Verify Two-Phase — Track Plan + Agent-Driven Execution

**Track ID:** verify-two-phase_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-058

## Summary

Implement a two-phase UI test strategy: (1) Track creation generates a Verification Plan section
in plan.md with test outlines, and (2) Verify button spawns a Claude agent that reads the plan,
generates Playwright code, and executes it.

## Context

Currently the Verify button runs a hardcoded two-step process:
- Step 1: `npm test -- --reporter=verbose` (unit tests)
- Step 2: `node scripts/browser-test.mjs` (UI tests — often doesn't exist or is too simple)

This produces generic tests that don't exercise the actual Issue scenario. ISS-056 added
workflow.md guidance for issue-driven test planning, but the verify route doesn't use it.

## User Story

As a developer, I want Track creation to pre-plan verification steps and the Verify button
to execute them intelligently, so that UI tests actually validate the Issue's acceptance criteria.

## Acceptance Criteria

- [ ] `conductor:new-track` SKILL.md plan template includes a `## Verification Plan` section
- [ ] Verification Plan contains: test type classification + high-level test steps per acceptance criterion
- [ ] Verify route (Step 2) spawns a Claude agent instead of running `node scripts/browser-test.mjs`
- [ ] Agent reads plan.md's Verification Plan + workflow.md's interaction patterns
- [ ] Agent generates a temporary Playwright script, executes it, captures results
- [ ] Unit tests (Step 1: `npm test`) remain unchanged — still hardcoded
- [ ] SSE progress messages show agent thinking + tool calls during verification
- [ ] Existing verify lock, result persistence, and status derivation logic remain intact

## Dependencies

- ISS-056 (workflow.md UI test strategy) — completed
- Existing verify route architecture (SSE, locks, result persistence)

## Out of Scope

- Changing unit test strategy (Step 1)
- Persistent Playwright test suite (tests are generated per-verify run)
- Changes to implement or reopen routes

## Technical Notes

Add `## Verification Plan` after the last Phase section, before Post-Implementation Checklist.

### Phase 2: Verify Route Refactor
Replace `node scripts/browser-test.mjs` with Claude agent SDK `query()`:
- Prompt: "Read plan.md Verification Plan. For each Interaction test, generate and run Playwright code. For Visual tests, take screenshots. Report results."
