# Specification: Case Detail Inspection 默认首位展示 + 格式美化

**Track ID:** inspection-beautify_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft
**Source:** ISS-022

## Summary

将 Case Detail 页面的 Inspection Tab 放在首位并设为默认展示，同时修复 markdown 渲染样式问题。

## Context

当前 Inspection tab 排在第 2 位（Info 之后），默认展示的是 Info tab。inspection 内容使用 react-markdown + remark-gfm 渲染，但因 @tailwindcss/typography 未安装，prose 类无效，导致 markdown 内容裸渲染（无表格边框、无标题样式、无列表缩进等）。

## User Story

As a support engineer, I want to see inspection report first when opening a case so that I can quickly understand the case status without extra clicks, and the content should be well-formatted for readability.

## Acceptance Criteria

- [ ] Inspection Tab 排在所有 tab 的第一位
- [ ] 打开 Case Detail 页面时默认展示 Inspection Tab
- [ ] Markdown 表格有清晰的边框和交替行色
- [ ] 标题层级有明显的视觉区分（字号、粗细）
- [ ] 列表（有序/无序）有正确的缩进和标记
- [ ] 行内代码和代码块有背景色区分
- [ ] 引用块（blockquote）有左边框和背景色
- [ ] 整体深色主题适配（prose-invert）

## Dependencies

- 需安装 @tailwindcss/typography 包

## Out of Scope

- 不改变 inspection 文件的内容结构
- 不添加 inspection 历史选择器
- 不修改其他 tab 的渲染样式（但 typography 插件会自动改善所有使用 prose 类的地方）

## Technical Notes

- 根因是 @tailwindcss/typography 未安装，prose 类空转
- 安装后配合 prose-invert 适配深色主题
- 可添加 ReactMarkdown components override 进一步定制表格等元素样式
