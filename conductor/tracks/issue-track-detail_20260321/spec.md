# Specification: Issues 页面 Tracked Issue 显示 Track 分析与修复计划

**Track ID:** issue-track-detail_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-034

## Summary

当 issue 有关联的 conductor track 时，在展开的只读详情面板中显示 track 的 spec（问题原因/分析）和 plan（修复计划/任务进度），让用户无需离开 Issues 页面即可了解完整的分析与实施方案。

## Context

当前 Issues 页面的展开面板仅显示 description、创建/更新时间、trackId 和 Edit 按钮。用户看到一个 issue 标记为 tracked/done 时，无法直接了解"为什么这样修"和"修了什么"，需要手动去 conductor 目录找文件。

## User Story

As a support engineer, I want to see the track's problem analysis and fix plan directly in the issue detail panel so that I can understand the full context without leaving the Issues page.

## Acceptance Criteria

- [ ] 后端提供 `GET /api/issues/:id/track-spec` 端点，返回 spec.md 原始内容
- [ ] 有 trackId 的 issue 展开时，显示 "Analysis" 区域（来自 spec.md 的 Summary + Acceptance Criteria）
- [ ] 有 trackId 的 issue 展开时，显示 "Plan" 区域（来自 plan.md 的任务列表 + 完成状态）
- [ ] 无 trackId 的 issue 展开时，行为不变（仅显示 description）
- [ ] Track 数据懒加载：仅在展开时 fetch，收起时不请求
- [ ] 使用 CSS 变量适配双主题

## Dependencies

- 已有后端端点: `GET /api/issues/:id/track`, `GET /api/issues/:id/track-plan`
- 已有 conductor-reader.ts service

## Out of Scope

- 不在 Issues 页面编辑 spec/plan
- 不实现 spec/plan 的全文 markdown 渲染（只提取关键区块）
- 不修改 ImplementPanel 或 TrackProgressPanel

## Technical Notes

- plan.md 已有 API（`/track-plan`），只需新增 spec API
- 前端用 React Query 懒加载（`enabled: isExpanded && !!trackId`）
- Spec 和 Plan 内容用简单的 section 解析提取关键段落，不需要完整 markdown 渲染
