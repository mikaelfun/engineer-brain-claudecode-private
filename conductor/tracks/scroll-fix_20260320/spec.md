# Specification: Fix CaseAIPanel Double-Scroll Bug

**Track ID:** scroll-fix_20260320
**Type:** Bug
**Created:** 2026-03-20
**Status:** Draft

## Summary

Case full process 执行时，CaseAIPanel 的 auto-scroll 逻辑使用 `scrollIntoView()` 导致内外两层 scrollbar 都被滚动到底部。应仅滚动内层消息容器。

## Context

Dashboard 的 CaseDetail 页面使用正常的 document scrolling（外层），CaseAIPanel 内部有一个 `max-h-60 overflow-y-auto` 的消息列表（内层）。SSE streaming 消息到达时触发 auto-scroll。

## Problem Description

**Steps to Reproduce:**
1. 打开 Dashboard → 点击某个 Case
2. 点击 "Process" 按钮启动 full process
3. 观察 SSE 消息 streaming 过程中的滚动行为

**Expected:** 只有内层消息列表自动滚动到最新消息，页面 scrollbar 位置不变
**Actual:** 内外两层 scrollbar 都被持续拉到底部

**Root Cause:**
`CaseAIPanel.tsx` 中使用 `scrollIntoView({ behavior: 'smooth' })` —— 该 API 会沿所有可滚动祖先链向上冒泡，导致页面级 scrollbar 也被滚动。

```tsx
// 问题代码
useEffect(() => {
  chatEndRef.current?.scrollIntoView({ behavior: 'smooth' })
}, [messages.length])
```

## Acceptance Criteria

- [ ] Full process streaming 时，内层消息列表自动滚动到最新消息
- [ ] Full process streaming 时，页面（外层）scrollbar 位置保持不变
- [ ] 手动滚动内层消息列表后，新消息到达时仍能自动追尾

## Dependencies

None

## Out of Scope

- 改变 Layout 的整体滚动架构
- 修改其他组件的滚动行为
- 添加 "scroll to bottom" 按钮等额外 UX

## Technical Notes

**Fix approach:** 将 `scrollIntoView()` 替换为直接操作内层容器的 `scrollTop`：

```tsx
const messagesContainerRef = useRef<HTMLDivElement>(null)

useEffect(() => {
  const container = messagesContainerRef.current
  if (container) {
    container.scrollTop = container.scrollHeight
  }
}, [messages.length])
```

将 ref 绑定到 `<div className="max-h-60 overflow-y-auto ...">` 容器上，移除 `chatEndRef` sentinel。
