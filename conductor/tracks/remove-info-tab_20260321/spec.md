# Specification: 移除 Info Tab — 数据整合到 Header + Inspection

## Summary

移除 CaseDetail 页面的 Info tab，将独有字段分流到 Header 和 Inspection，消除数据重复。

## Context

CaseDetail 页面当前有 11 个 tab，其中 Info tab 展示的大部分字段已在 Header 和 Inspection 报告中覆盖。仅少量字段为 Info tab 独有。移除 Info tab 可减少页面复杂度，让用户聚焦于 Inspection 报告。

## Motivation

- Info tab 与 Header + Inspection 数据高度重复
- 11 个 tab 数量过多，减少一个可降低认知负担
- Inspection 是默认 tab，已作为 case 概览的主要入口

## Field Migration Plan

| 字段 | 原位置 | 目标 | 说明 |
|------|--------|------|------|
| Case Number, Title, Severity, Status | Info + Header | 保持 Header | 已覆盖 |
| SAP, Assigned To, Case Age, Created, Modified | Info + Header | 保持 Header | 已覆盖 |
| Customer | Info + Inspection | 保持 Inspection | 已覆盖 |
| **24x7** | Info only | → Header | 新增到 header badges 区域 |
| **Origin** | Info only | → Header | 新增到 header 元数据行 |
| **Timezone** | Info only | → Header | 新增到 header 元数据行 |
| **Country** | Info only | → Header | 新增到 header 元数据行 |
| **Contact (name/email/phone/preferred)** | Info only | → Inspection 模板 | 更新 inspection-writer skill 确保包含 |
| Active Hours | Info only | 删除 | 不需要 |
| Counts (email/note/phone/labor/icm/attachment) | Info only | 删除 | Inspection 叙述式已含 |
