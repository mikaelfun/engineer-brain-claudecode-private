## Overview

修复 restart-service.ts 的两个核心问题：(1) 杀进程时向上追溯进程树，确保 watcher 父进程也被杀掉；(2) spawn 命令与 package.json dev 脚本对齐，保持 --watch 模式。

## Phase 1: Fix Process Kill Logic

修复 `killSavedAndPort()` — 通过端口找到子进程后，向上追溯父进程，杀掉整个进程树根节点。

### Tasks

- [x] Task 1.1: 新增 `findProcessTreeRoot(pid)` 函数 — 给定子进程 PID，通过 `wmic process get ParentProcessId where ProcessId=X` 递归向上查找，直到父进程不是 node/cmd/pwsh（即找到进程树的根）
- [x] Task 1.2: 修改 `killSavedAndPort()` — 在杀端口占用进程之前，先调用 `findProcessTreeRoot()` 找到真正的根 PID，然后 `taskkill /F /T /PID root`，确保 watcher 和所有子进程一起被杀
- [x] Task 1.3: 增加重试验证 — 杀完后再次检查端口是否释放，如果仍被占用则重试一次（防止 watcher 来不及退出又重启了子进程）
