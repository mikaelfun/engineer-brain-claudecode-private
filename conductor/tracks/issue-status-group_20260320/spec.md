# Specification: Issues 页面按状态分组 + 折叠

**Track ID:** issue-status-group_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft
**Source:** ISS-014

## Summary

将 Issues 页面的 issue 列表按状态分为三组：Pending（待处理）、Tracked（已建 track）、Done（已完成），每组可折叠展开，Done 组默认折叠。

## Context

当前 Issues 页面所有 issue 以 flat 列表形式展示，不同状态的 issue 混在一起，不方便快速定位待处理项。需按状态分组并支持折叠，让用户聚焦于当前需要关注的 issue。

## Acceptance Criteria

- [ ] Issues 按三组显示：🟡 Pending (pending/tracking/in-progress)、🔵 Tracked (tracked)、✅ Done (done)
- [ ] 每组有标题栏 + issue 数量 + 折叠/展开按钮
- [ ] Done 组默认折叠，其余默认展开
- [ ] 折叠后只显示标题栏，展开后显示所有 issue 行
- [ ] 空组不显示
- [ ] 原有 Filter（type/status）仍然可用

## Dependencies

None

## Out of Scope

- 不改变 issue 行本身的渲染方式
- 不改变分页逻辑
- 不改变 IssueRow 组件的编辑/查看交互

## Technical Notes

- 分组在前端完成（对 API 返回的 issues 数组做 client-side grouping）
- 三组分类：pending group = pending/tracking/in-progress, tracked group = tracked, done group = done
- collapsedGroups state 用 Set<string> 管理折叠状态
