## Summary

Refactor `generate-digest.py` 的 `generate_onenote_digest()` 从单次大 prompt 架构改为 per-page parallel LLM classify + code merge，对齐 Teams digest 的 ISS-227 架构。

## Context

Teams digest 已在 ISS-227 中完成 per-chat parallel 改造：每个 chat 文件独立 LLM 调用（`score_and_extract_single_chat()`），ThreadPoolExecutor 并行，纯代码 merge（`merge_digest_results()`），写 `_relevance.json` 缓存。OneNote digest 仍为单次大 prompt 架构。

## Motivation

1. 5+ 页场景注意力分散，分类质量下降
2. 无增量缓存——即使只新增 1 页也要重跑全部
3. 单次 LLM 调用失败全挂，无容错
4. 与 Teams digest 架构不一致，维护负担
