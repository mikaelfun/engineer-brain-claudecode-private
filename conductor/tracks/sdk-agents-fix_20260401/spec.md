# Specification: SDK Session 加载自定义 Agent 定义

**Track ID:** sdk-agents-fix_20260401
**Type:** Bug
**Created:** 2026-04-01
**Status:** Draft
**Issue:** ISS-200

## Summary

WebUI 通过 Claude Agent SDK `query()` 创建的 session 无法识别 `.claude/agents/*.md` 中定义的自定义 subagent type，导致 casework 流程中 spawn `teams-search`、`onenote-case-search` 等 agent 时回退为 `general-purpose`，丢失专用 MCP、工具集和 model 配置。

## Problem Description

**根因：** `case-session-manager.ts` 使用 `settingSources: ['user']` 跳过项目级设置（避免 `.mcp.json` 300+ MCP tools 溢出 token 限制）。这同时阻止了 `.claude/agents/*.md` 的自动发现。SDK 的 `query()` 函数支持 `agents` option（`Record<string, AgentDefinition>`）来编程式注入 agent 定义，但当前代码未使用此选项。

**复现步骤：**
1. Dashboard WebUI → Case → Process → full-process
2. casework 执行到 B2 步骤，spawn `teams-search` / `onenote-case-search`
3. Agent 报错 "not available"，回退 `general-purpose`

**影响：**
- `general-purpose` agent 无专用 MCP（如 `teams-search` 需要 teams MCP）
- 回退 agent 缺少专用 prompt、工具集限制、model 配置
- CLI `/casework` 不受影响（CLI 会话启动时自动加载 `.claude/agents/`）

## Acceptance Criteria

- [ ] SDK session 启动时自动读取 `.claude/agents/*.md`，解析 YAML frontmatter，转换为 `AgentDefinition` 对象
- [ ] `query()` 调用中通过 `agents` option 注入解析后的 agent 定义
- [ ] 解析支持所有 frontmatter 字段：`name`、`description`、`tools`、`model`、`maxTurns`、`mcpServers`
- [ ] agent markdown body 作为 `AgentDefinition.prompt` 传入
- [ ] 解析失败的单个 agent 文件不阻塞整体（跳过 + 日志警告）
- [ ] WebUI full-process 执行 casework 时，`teams-search` 和 `onenote-case-search` 正确 spawn（非 fallback）

## Dependencies

- `@anthropic-ai/claude-agent-sdk` 已支持 `agents` option（已确认 `AgentDefinition` 类型存在）
- `.claude/agents/*.md` frontmatter 格式已规范化（CLAUDE.md 已定义）

## Out of Scope

- 不修改 `.claude/agents/*.md` 文件格式
- 不修改 `settingSources` 策略（保持 `['user']` 避免 MCP 溢出）
- 不处理 agent 运行时的 MCP server 可用性检查

## Technical Notes

- SDK `AgentDefinition` 类型：`{ description, prompt, tools?, model?, mcpServers?: AgentMcpServerSpec[] }`
- `AgentMcpServerSpec = string | Record<string, McpServerConfigForProcessTransport>`
- frontmatter 中 `tools` 为逗号分隔字符串，需 split 为 `string[]`
- frontmatter 中 `mcpServers` 为 YAML 列表（如 `- teams`），对应 `AgentMcpServerSpec` 的 string 形式
- 缓存解析结果避免每次 `query()` 调用重复 IO
