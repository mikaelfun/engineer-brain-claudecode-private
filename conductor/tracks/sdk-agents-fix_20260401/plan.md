## Phase 1: Agent Definition Loader

实现核心解析逻辑。

### Tasks

- [x] Task 1.1: 在 `case-session-manager.ts` 中新增 `loadAgentDefinitions()` 函数
  - 读取 `{projectRoot}/.claude/agents/*.md`
  - 解析 YAML frontmatter（`---` 包裹的块）提取 `name`、`description`、`tools`、`model`、`mcpServers`
  - 将 `---` 之后的 markdown body 作为 `prompt`
  - `tools` 字段：逗号分隔字符串 → `string[]`
  - `mcpServers` 字段：YAML 列表 → `AgentMcpServerSpec[]`（string 形式）
  - 返回 `Record<string, AgentDefinition>`，key 为 frontmatter 的 `name`
  - 加缓存（`_agentDefinitionsCache`），避免重复 IO
  - 单个文件解析失败时 console.warn + 跳过，不影响其他 agent

- [x] Task 1.2: 将 `loadAgentDefinitions()` 注入到所有 `query()` 调用
  - `processCaseSession()`：`options.agents = loadAgentDefinitions()`
  - `chatCaseSession()`：`options.agents = loadAgentDefinitions()`
  - `executeTodoAction()`：`options.agents = loadAgentDefinitions()`
  - 其他使用 `query()` 的地方按需添加
