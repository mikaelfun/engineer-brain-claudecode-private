# Specification: OneNote Inline Images + Vision Analysis

**Track ID:** onenote-vision_20260417
**Type:** Feature
**Created:** 2026-04-17
**Status:** Draft
**Issue:** ISS-224

## Summary

OneNote Export 页面中包含大量 inline 图片（Portal 截图、CLI 输出、错误信息等），当前 search-inline.py 只复制 markdown 文本，generate-digest.py 只传文本给 LLM，WebUI 只显示 digest 文本。本 track 实现三层增强：复制图片资源 → LLM vision 分析 → WebUI 内联渲染。

## Context

- 238 个 OneNote 页面包含 inline 图片，存储为 `./assets/*.png`（~100KB/张）
- 图片包含 RBAC 配置、Alert 规则、错误码等关键诊断信息
- 当前 `search-inline.py` 复制 `_page-*.md` 但不复制 `assets/`
- 当前 `generate-digest.py` 只传 text content 给 LLM API
- 当前 WebUI OnenoteTab 只渲染单个 digest markdown

## User Story

As an Azure Support Engineer, I want OneNote 页面中的截图能被 LLM 分析并在 WebUI 中直接查看，so that 我不需要手动打开 OneNote 就能获取图片中的诊断信息。

## Acceptance Criteria

- [ ] AC1: search-inline.py 复制 _page-*.md 时同步复制引用的 assets/*.png 到 `{caseDir}/onenote/assets/`
- [ ] AC2: 复制后的 _page-*.md 中的图片相对路径可正确 resolve 到 `onenote/assets/` 目录
- [ ] AC3: generate-digest.py onenote 分支支持传入图片（base64 image_url）给 LLM vision API，每页最多 3 张
- [ ] AC4: WebUI OnenoteTab 显示各 _page-*.md 内容（可折叠），图片内联渲染
- [ ] AC5: 后端 API 提供 `GET /api/cases/:id/onenote/assets/:filename` 端点服务图片文件

## Dependencies

- search-inline.py 已有页面复制逻辑（ISS-221）
- generate-digest.py 已有 onenote 分支
- NewAPI/Copilot Proxy 需支持 vision（需验证）

## Out of Scope

- 图片 OCR（依赖 LLM vision 内建能力）
- 视频文件处理
- OneNote 在线实时同步（仍依赖 onenote-export）

## Technical Notes

- OneNote Export 目录结构：`PageName.md` + `assets/` 目录（图片文件）
- 图片引用格式：`![image](./assets/xxx.png)`
- LLM API vision 格式：OpenAI 兼容 `image_url` content type（base64 data URI）
- WebUI 图片通过后端 API 代理服务（不直接暴露文件系统路径）
- 需注意图片大小控制：单张 ~100KB，base64 后 ~133KB，每页 3 张 ≈ 400KB/页
