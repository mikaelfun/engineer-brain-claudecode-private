# Specification: data-refresh 增加拉取 D365 Labor 记录

## Summary

casework data-refresh 流程缺少拉取 D365 labor 记录的步骤，导致 labor-estimate 无法判断当天是否已记录 labor，每次都给出"未记录"的错误估算。

## Context

casework pipeline 的 data-refresh 阶段负责从 D365 拉取所有 case 数据（case-info、emails、notes、attachments、ICM），但遗漏了 labor 记录。`view-labor.ps1` 脚本已存在（支持 API + UI fallback），但不接受 `-TicketNumber` / `-OutputDir` 参数，无法被 data-refresh 标准流程调用。labor-estimate skill 依赖 case 目录下的文件来判断当天活动，但无法获知已有的 labor 记录。

## Problem Description

1. `view-labor.ps1` 使用 `Get-CurrentCaseId`（浏览器上下文），不接受 `-TicketNumber` 参数
2. data-refresh SKILL.md 没有调用 labor 拉取脚本的步骤
3. labor-estimate SKILL.md 没有读取已有 labor 记录的步骤
4. 结果：labor-estimate 每次都认为当天未记录 labor，给出重复的估算建议

## Acceptance Criteria

- [ ] labor-estimate SKILL.md 读取 `labor.md`，如果当天已有记录则提示并跳过或调整估算
- [ ] case-directory schema 更新，包含 `labor.md` 文件说明
