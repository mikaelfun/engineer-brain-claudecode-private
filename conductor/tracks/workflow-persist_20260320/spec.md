# Specification: Workflow Progress Persistence

**Track ID:** workflow-persist_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft

## Summary

WebUI 页面刷新后 workflow 进度状态丢失。当前 SSE 消息可从后端恢复，但 Zustand store 的实时 workflow 进度（当前步骤、各步骤耗时、运行状态）未持久化。需要完整的 workflow state 持久化方案。

## Context

当前页面刷新后，CaseAIPanel 可以从后端恢复持久化的 SSE 消息，Timing tab 也能显示已完成步骤的时间。但实时的 workflow 进度状态（哪个步骤正在运行、整体进度百分比等）只存在于 Zustand 内存 store 中，刷新即丢失。

## Acceptance Criteria

- [ ] 页面刷新后，正在运行的 case process 的进度状态可恢复
- [ ] 当前正在执行的步骤名称可恢复显示
- [ ] 已完成步骤的状态和耗时可恢复显示
- [ ] 非运行状态的 case 不受影响

## Dependencies

- 依赖 `process-visibility` track（需要先有有意义的进度信息才值得持久化）

## Out of Scope

- 全局进度百分比计算
- 离线状态下的进度追踪

## Technical Notes

可能方案：
1. **后端维护 workflow state** — 每个步骤开始/结束时更新后端状态文件，前端刷新后从 API 恢复
2. **localStorage 缓存** — Zustand persist middleware 将 workflow 进度写入 localStorage
3. **SSE reconnect 恢复** — 重连 SSE 后从后端推送当前运行状态快照

方案 1 最可靠，因为后端已有 `activeOperations` Map。
