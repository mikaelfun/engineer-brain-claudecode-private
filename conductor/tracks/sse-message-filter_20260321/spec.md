# Specification: Agent Session SSE Message Filtering

**Track ID:** sse-message-filter_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-033

## Summary

CaseAIPanel 的实时消息流过于 verbose，tool-call/tool-result 事件显示完整内容（参数、返回值），导致信息噪音大。需要在前端 UI 层过滤/折叠非关键消息，用简洁的 "Thinking..." 动画替代，只保留关键状态流。

## Context

Dashboard 通过 SSE 接收 Claude SDK session 的实时消息，包括 `thinking`、`tool-call`、`tool-result`、`completed`、`failed`、`system` 六种类型。当前 `CaseAIPanel` 的 `MessageBubble` 组件会渲染所有类型的消息，每条 tool-call 显示工具名+参数，每条 tool-result 显示返回内容（截断到 200 字符），导致一次操作可能产生几十条 verbose 消息。

## User Story

As a support engineer, I want to see concise AI status updates instead of verbose tool details so that I can quickly understand what the AI is doing and when it's done.

## Acceptance Criteria

- [ ] tool-call 和 tool-result 消息默认折叠为一行简洁状态（如 "🔧 Running Bash..." / "✅ Bash done"）
- [ ] 连续多条 tool-call/tool-result 合并为一个 "Thinking..." 动画（带工具名轮播或计数）
- [ ] thinking 消息（assistant text）也默认折叠，只显示最新一条或 "AI is thinking..."
- [ ] 关键状态消息始终显示：system（步骤开始）、completed、failed
- [ ] 用户可以点击展开查看完整的 verbose 详情（可选，不阻挡 MVP）
- [ ] 消息面板滚动行为保持正常（自动滚到底部）

## Dependencies

- `dashboard/web/src/components/CaseAIPanel.tsx` — MessageBubble 渲染逻辑
- `dashboard/web/src/stores/caseSessionStore.ts` — 消息数据结构
- `dashboard/web/src/hooks/useSSE.ts` — SSE 事件接收

## Out of Scope

- 后端 SSE 事件格式变更（纯前端过滤）
- 其他 SSE 流（issue-track、implement）的过滤（可后续跟进）
- 消息持久化或历史查看功能

## Technical Notes

改动范围限于前端 UI 层：
1. 在 `CaseAIPanel.tsx` 的消息列表渲染层做过滤/合并
2. 可能需要在 store 层或渲染层增加消息合并逻辑
3. 不修改后端 SSE 发送逻辑，保持完整消息流（便于 debug）
