# Specification: Agents Page Unified Session View

**Track ID:** agent-session-view_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft

## Summary

Extend the Agents page (`AgentMonitor.tsx`) to display all agent session types: case sessions (full-process, steps, chat), issue implement sessions, verify sessions, and track-creation sessions — providing a unified view of all AI-driven operations.

## Context

The Dashboard's Agents page currently only shows active case sessions via `GET /api/sessions`. Issue-related sessions (implement, verify, track-creation) are tracked by separate in-memory managers (`implement-session-manager`, `verify-session-manager`, `issue-track-state`) but are never surfaced on the Agents page. Users must navigate to the Issues page to see these — and even there, only active progress is visible, not history.

## User Story

As an Azure Support Engineer, I want to see all agent sessions (case, implement, verify, track-creation) in one unified view on the Agents page, so that I can monitor all AI-driven operations and their history without switching between pages.

## Acceptance Criteria

- [ ] Agents page shows case sessions of all statuses (active, paused, completed, failed), not just active
- [ ] Agents page shows implement sessions with issue ID, track ID, and status
- [ ] Agents page shows verify sessions with issue ID and test results summary
- [ ] Agents page shows track-creation sessions with issue ID and status
- [ ] Sessions are grouped by type with clear visual distinction
- [ ] Each session shows: type badge, context (case#/issue ID), status, intent/description, timestamps
- [ ] Filter/search capability: filter by session type and status

## Dependencies

- Existing `case-session-manager.ts` (persisted, has `getAllSessions()`)
- Existing `implement-session-manager.ts` (in-memory, has `getImplementSession()`)
- Existing `verify-session-manager.ts` (in-memory, has `getVerifySession()`)
- Existing `issue-track-state.ts` (in-memory, has state per issue)

## Out of Scope

- Persisting implement/verify/track sessions to disk (separate enhancement)
- Adding resume capability for issue sessions from the Agents page
- Real-time SSE streaming of session messages on the Agents page (messages stay on Issues/CaseDetail pages)

## Technical Notes

- New backend endpoint `GET /api/sessions/all` to aggregate all session types into a unified format
- Implement/verify/track sessions are in-memory only — they will disappear on server restart (acceptable for now)
- Case sessions remain the only persisted type (via `.case-sessions.json`)
