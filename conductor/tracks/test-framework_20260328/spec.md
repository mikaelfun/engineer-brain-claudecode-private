# Specification: 自动化测试框架重构 — 文件驱动状态机 + 主-子 Agent + /loop 持续迭代

**Track ID:** test-framework_20260328
**Type:** Feature
**Created:** 2026-03-28
**Status:** Draft
**Issue:** ISS-114

## Summary

重构现有 casework-evolution-loop 为文件驱动的自动化测试框架。通过状态机 + 主-子 Agent 模式 + /loop 集成，实现持续自我迭代的测试→发现→修复→验证闭环。解决当前 context 压缩导致规则丢失、UI 测试不执行、发现问题不修复等根本性问题。

## Context

EngineerBrain 已有 21 轮 evolution loop 经验、7 个测试场景（A-G）、70+ findings、5 个已修复 bug。但现有方案把状态和规则都依赖 session memory，compaction 后丢失。新框架的核心原则：**State lives in files, not in session memory**。

## User Story

As a 项目编排者（Kun Fang），I want to 把预期的功能和效果写进需求文档后，自动化测试框架能自动发现未实现的需求、设计测试、执行验证、修复问题，so that 我不需要每一个 bug 和需求都亲力亲为，整个设计验证迭代流程自动运转。

## Technical Notes

- 所有 Bash 路径必须 POSIX 格式（`/c/Users/...`）
- Playwright 必须用 msedge 浏览器
- 变量赋值不能与 pipe 同行
- Agent spawn 不注入大段内容，让 agent 自己读文件
- 现有 evolution-loop 的 case 测试池保留：数据丰富（2603090040000814 等）、数据较少（2603250010001221 等）、无邮件（2603230010001900001）
