# Specification: Issue Implement 实时进度展示

**Track ID:** implement-progress_20260320
**Type:** Feature
**Created:** 2026-03-20
**Status:** Draft

## Summary

Issue 页面点击 Implement 按钮后，在该 issue 行下方展开内联进度面板，实时显示 Claude SDK 执行流（thinking / tool-call / tool-result），支持页面刷新后恢复进度，防止重复 implement。

## Context

Dashboard 的 casework 流程已有成熟的 SSE + 持久化 + 操作锁架构（`case-session-manager.ts` + `caseSessionStore` + `useSSE`）。Issue implement 路由已能 spawn Claude SDK 并 broadcast 4 种 SSE 事件，但前端完全没消费。需要补齐前端展示和后端鲁棒性。

## User Story

As an engineer using the Issue Tracker, I want to see real-time implementation progress after clicking "Implement", so that I know what the agent is doing, whether it's stuck, and when it's done.

## Acceptance Criteria

- [ ] 点击 Implement 按钮后，该 issue 行下方展开内联进度面板
- [ ] 进度面板实时显示 SDK 消息流：thinking（💭）、tool-call（🔧）、tool-result（✅/❌）
- [ ] 进度面板有 loading spinner 和当前状态文字（Running / Completed / Failed）
- [ ] 刷新页面后，正在进行的 implement 面板自动恢复（持久化消息）
- [ ] 同一 issue 不能重复点击 Implement（操作锁 + 按钮 disabled）
- [ ] 完成后 issue 状态自动更新、面板显示完成摘要
- [ ] 失败时面板显示错误信息

## Dependencies

- 已有 SSE 基础设施：`sse-manager.ts`、`useSSE.ts`、`events.ts`
- 已有 case session 参考实现：`case-session-manager.ts`、`caseSessionStore.ts`
- 已有 4 种 issue-implement SSE 事件类型定义
- 后端 `start-implement` 路由已有 Claude SDK spawn 逻辑

## Out of Scope

- 不做 issue implement 的 resume/chat 交互（只读进度展示）
- 不做 implement 过程中的用户干预（Cancel/Stop 可作后续 feature）
- 不复制 CaseAIPanel 全部功能，保持轻量
