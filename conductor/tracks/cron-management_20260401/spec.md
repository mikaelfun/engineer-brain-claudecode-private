# Specification: WebUI Cron Job Management

**Track ID:** cron-management_20260401
**Type:** Feature
**Created:** 2026-04-01
**Status:** Draft

## Summary

Add create, delete, and status viewing capabilities for Cron Jobs (Remote Triggers) to the WebUI Agents page, enabling users to manage scheduled tasks like periodic OneNote sync or patrol through the web interface instead of CLI.

## Context

The Agents page currently displays cron jobs in read-only mode from a local `cron-jobs.json` file. Claude Code provides a RemoteTrigger API (`/v1/code/triggers`) that supports full CRUD operations for scheduled agents. This feature connects the WebUI to that API for complete cron job lifecycle management.

## User Story

As an Azure Support Engineer, I want to create, view, and delete scheduled tasks (cron jobs) from the Dashboard so that I can set up recurring automation (e.g., OneNote sync every 3 hours, nightly patrol) without using the CLI.

## Acceptance Criteria

- [ ] User can create a new cron job via a form (name, prompt/command, cron expression, optional description)
- [ ] User can delete an existing cron job with confirmation
- [ ] Cron job list shows current status: enabled/disabled, last run time, last status, next run time, error count
- [ ] Creating a cron job calls the RemoteTrigger API and the new job appears in the list immediately
- [ ] Deleting a cron job removes it from the RemoteTrigger API and the list updates immediately
- [ ] User can manually trigger a cron job to run immediately

## Dependencies

- RemoteTrigger API (`/v1/code/triggers`) — already available as a tool in Claude Code
- Existing Agents page (`AgentMonitor.tsx`) and backend (`agents.ts`, `cron-reader.ts`)

## Out of Scope

- Execution log details (use Agent Monitor for that)
- Visual cron expression editor (calendar/time picker) — user types cron expression directly
- Editing existing cron jobs (create-only for v1; delete + recreate for changes)

## Technical Notes

- Backend: Add proxy endpoints in `agents.ts` that forward to RemoteTrigger API
- Frontend: Add create form (dialog/inline) and delete button to existing Cron Jobs section in AgentMonitor.tsx
- Data source: Switch from local `cron-jobs.json` to RemoteTrigger API as primary data source
- Provide common presets for cron expressions (every 1h, 3h, daily 9am, etc.)
