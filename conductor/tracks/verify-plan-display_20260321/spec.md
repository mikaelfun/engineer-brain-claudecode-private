# Specification: Implemented Issue 展示验证计划

**Track ID:** verify-plan-display_20260321
**Type:** Feature
**Created:** 2026-03-21

## Summary

在 Issues 页面中，implemented 状态的 issue 展开详情时，除现有的 Task 列表外，还应展示 plan.md 中的 `### Verification` 检查项和 `## Post-Implementation Checklist`，让用户清晰看到测试/验证计划的完成情况。

## Context

当前 Issues 页面展开 tracked/implemented issue 详情时，Plan 面板仅解析 `- [x] Task X.Y: ...` 格式的任务行，忽略了 plan.md 中同样重要的 `### Verification` 和 `## Post-Implementation Checklist` 部分。这些部分包含 TypeScript 编译、单元测试、手动测试等验证步骤的完成状态，对 implemented 状态的 issue 尤为关键。

## User Story

As a developer reviewing implemented issues, I want to see the verification checklist from the track plan so that I can quickly assess whether all quality gates have been passed before marking an issue as done.

## Acceptance Criteria

- [ ] Plan 面板除 Task 列表外，额外展示 Verification 检查项（来自各 Phase 的 `### Verification` 小节）
- [ ] Plan 面板展示 Post-Implementation Checklist（来自 `## Post-Implementation Checklist` 小节）
- [ ] 检查项显示 ✅/⬜ 状态（对应 `[x]` / `[ ]`）
- [ ] Verification 和 Checklist 区域视觉上与 Task 列表区分（如小标题或分隔线）
- [ ] 对无 verification 内容的 plan 保持现有展示不变

## Dependencies

## Out of Scope

- 不涉及 implement/verify 流程本身
