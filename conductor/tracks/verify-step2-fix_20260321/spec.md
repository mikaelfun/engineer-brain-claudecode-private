## Problem Description

### 现象
- Verify 第一步（`npm test -- --reporter=verbose`）通过
- Verify 第二步（`node scripts/browser-test.mjs`）失败
- Issue 状态保持 `in-progress`，无法标记为 `done`

### 已知风险点
1. **端口问题**：`browser-test.mjs` 硬编码 `http://localhost:5177`，如果前端服务运行在其他端口（如 5173），脚本直接失败
2. **服务未启动**：Verify 接口 (`POST /api/issues/:id/verify`) 直接执行 `node scripts/browser-test.mjs`，没有检查前端/后端服务是否已启动
3. **页面错误捕获**：脚本遍历 `/`, `/todo`, `/agents`, `/drafts`, `/settings`, `/issues` 六个页面，任何页面出现 `Something went wrong`、`Maximum update depth`、JS pageerror 或 console error 都会导致整体失败
4. **登录方式**：脚本用密码 `eb2026` 登录，如果登录机制有变化也会失败

### Steps to Reproduce
1. 在 Issues 页面找到状态为 `in-progress` 的 Issue
2. 点击 Verify 按钮
3. 第一步 Unit Tests 通过
4. 第二步 UI Tests 报错
