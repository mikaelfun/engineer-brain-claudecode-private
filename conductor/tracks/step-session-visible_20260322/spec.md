# Specification: Fix step session not visible in Agent Monitor

**Track ID:** step-session-visible_20260322
**Type:** Bug
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-082

## Summary

When executing fast steps (like status-judge) via Dashboard step API, the SDK session is not visible in Agent Monitor or active session lists because the session starts, runs, and ends within a single React Query polling interval.

## Context

The Agent Monitor page uses React Query polling to fetch sessions via `GET /sessions/all`. Case step sessions are created in `case-session-manager.ts` and completed in `steps.ts`. For quick steps, the session lifecycle completes between two poll intervals, making it invisible.

## Problem Description

**Steps to Reproduce:**
1. Dashboard → Case Detail → AI Panel → click "Status Judge" step
2. Step completes quickly (a few seconds)
3. Open Agent Monitor page — session never appears
4. No record of the execution in the session list

**Expected:** Session appears in Agent Monitor (even if completed).
**Actual:** Session is never visible — it completes before the next poll.

## Acceptance Criteria

- [ ] Step sessions appear in Agent Monitor page, even for fast-completing steps
- [ ] Session shows correct status (active → completed lifecycle visible)
- [ ] Agent Monitor auto-refreshes when a new session starts or completes (SSE-driven invalidation)

## Dependencies

None

## Out of Scope

- Changing the polling interval
- Real-time SSE streaming for session lists (we only need SSE-triggered React Query invalidation)

## Technical Notes

- `steps.ts` already broadcasts `case-step-started` and `case-step-completed` SSE events
- The fix is to broadcast a `sessions-changed` SSE event from `steps.ts` when sessions start/complete
- Frontend `useUnifiedSessions` hook needs to listen for `sessions-changed` SSE and invalidate React Query
- Alternative: `endSession()` in `case-session-manager.ts` could broadcast `sessions-changed`
