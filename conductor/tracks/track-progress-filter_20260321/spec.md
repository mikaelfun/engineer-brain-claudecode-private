# Specification: Track Progress Panel SSE Message Filtering

**Track ID:** track-progress-filter_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft
**Source:** ISS-035

## Summary

TrackProgressPanel 在 Issue Create Track 过程中实时显示过多 Tool: 步骤（如 Tool: Read, Tool: Write 等），信息噪音大。应复用 sse-message-filter_20260321 的消息折叠模式，将连续的 tool-call 消息合并为紧凑的状态指示器，只显示关键状态变化。

## Context

已在 CaseAIPanel 中实现了 `processMessages()` 函数和 `CollapsedGroup` 组件（track: sse-message-filter_20260321），将 verbose 的 tool-call/tool-result/thinking 消息折叠为简洁状态行。TrackProgressPanel 有相同问题但使用不同的消息类型 `IssueTrackMessage`，需要针对该类型实现类似的消息合并逻辑。

## Acceptance Criteria

- [ ] 连续的 `tool-call` 消息合并为一行，显示工具名列表和数量（如 "Read, Write... (5 tools)"）
- [ ] 连续的 `thinking` 消息只保留最后一条
- [ ] `started`, `completed`, `error`, `question` 类型始终单独显示
- [ ] 合并的消息组可点击展开查看原始详情
- [ ] TypeScript 编译通过
- [ ] 单元测试覆盖合并逻辑

## Dependencies
