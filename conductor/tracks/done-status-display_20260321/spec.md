# Specification: Done Issue 展示实现状态和验证状态

**Track ID:** done-status-display_20260321
**Type:** Feature
**Created:** 2026-03-21
**Status:** Draft

## Summary

Done 分组的 issue 需要展示完整的实现和验证状态：区分人工验证（Mark Done）和自动测试验证（Verify），自动验证时展示每步（Unit Test / UI Test）的通过/失败结果。

## Context

Dashboard Issue Tracker 已支持 Done 分组显示，`VerifyResultPanel` 组件已存在可展示自动验证结果。但当前 Done issue 缺乏以下信息：
1. 无法区分 issue 是人工 Mark Done 还是自动 Verify 通过后标记的
2. Done issue 行（collapsed 状态）没有验证方式指示器
3. 已有的 `VerifyResultPanel` 只在展开详情时显示，且仅在有 `verifyResult` 时渲染

系统现状：
- 人工 Mark Done: `updateIssue(id, { status: 'done' })`，不写 `verifyResult`
- 自动 Verify: `updateIssue(id, { status: 'done', verifyResult: { unitTest, uiTest, verifiedAt } })`
- 区分方式：`verifyResult` 字段有无（隐式）

## User Story

As a developer using the Issue Tracker, I want to see at a glance whether a Done issue was manually marked done or auto-verified by tests, so that I can quickly assess the quality assurance level of completed work.

## Acceptance Criteria

- [ ] Done issue row (collapsed) 显示验证方式徽章："Manual" 或 "Auto-Verified"
- [ ] Auto-Verified 徽章显示总体结果颜色（全通过=绿色，部分失败=红色）
- [ ] Done issue 展开详情时，对于人工 Mark Done 的 issue，显示 "Manually marked as done" 提示
- [ ] Done issue 展开详情时，对于自动验证的 issue，显示 `VerifyResultPanel`（已有组件，确保渲染正确）
- [ ] Track 完成状态（Plan tasks 进度）在 Done issue 的 collapsed 行中有简要展示（如 "5/5 tasks"）

## Dependencies

- ISS-039 `verify-results-display`: VerifyResultPanel 组件（已完成）
- ISS-034 `issue-track-detail`: Issue 详情展示 Analysis + Plan（已完成）
- 现有 `verifyResult` 字段隐式区分机制（无需新增后端字段）

## Out of Scope

- 不修改 Mark Done 或 Verify 后端逻辑
- 不新增 `doneMethod` 字段到 issue JSON（利用现有隐式区分）
- 不修改 `VerifyResultPanel` 组件内部渲染逻辑

## Technical Notes

- 纯前端变更，无需后端 API 修改
- 区分逻辑：`issue.verifyResult` 存在 → 自动验证，不存在 → 人工 Mark Done
- Track plan 任务进度可从已有的 `useTrackPlan` hook 获取
- 需要在 collapsed 行中添加小徽章，不影响现有布局
