# Specification: Filter Empty System Messages in Agent Monitor

**Track ID:** session-empty-msg_20260322
**Type:** Bug
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-059

## Summary

Agent Monitor case session 展开后，消息列表中出现大量 type 为 system 的空消息（内容为空或仅显示 'you'），需要在持久化和渲染两层过滤。

## Context

Dashboard 通过 Claude Code SDK 管理 per-case session。SDK 回调的消息中包含非展示型消息（如 system/init 元数据），这些消息没有文本内容但仍被持久化到 `.case-session-messages.json` 并在 Agent Monitor 中渲染为空白行。

## Problem Description

**根因链（已验证）：**
1. SDK 发出非展示消息（如 `system/init` 元数据），没有文本 content
2. `extractContent()` (sse-helpers.ts) 对这些消息返回空字符串 `''`
3. `getPersistedMessageType()` 将未知 SDK 类型映射为 `'system'`
4. `appendSessionMessage()` (case-session-manager.ts) 持久化时没有过滤空内容
5. Agent Monitor 恢复加载后将它们渲染为空白 "System" 行
6. `SessionMessageList.tsx` 对每条 system 消息逐条渲染，即使内容为空
7. 附带问题：`steps.ts` 使用原始 `messageType` 而非 `getPersistedMessageType()`，导致类型不一致

**复现步骤：**
1. 在 Dashboard 中对任意 case 触发 Full Process 或 Step 操作
2. 等待操作完成
3. 打开 Agent Monitor 页面
4. 展开该 case 的 session detail
5. 观察到大量空白的 "System" 消息条目

**预期行为：** 消息列表只显示有实际内容的消息
**实际行为：** 消息列表中充斥大量空白 system 消息，干扰查看

## Acceptance Criteria

- [ ] 持久化层过滤：`appendSessionMessage()` 不再写入空内容消息
- [ ] 渲染层过滤：`SessionMessageList` 不渲染空内容的 system 消息
- [ ] 类型一致性：`steps.ts` 使用 `getPersistedMessageType()` 而非原始 messageType
- [ ] 现有有内容的 system 消息仍正常显示

## Dependencies

None — 修改的文件没有外部依赖

## Out of Scope

- 清理已存在的历史空消息文件（只影响新消息）
- 修改 SSE 实时路径的消息过滤（已有 sse-message-filter track 处理）
