# Specification: Evolution 概念拆分 — Discoveries + Evolution

**Track ID:** evolution-split_20260328
**Type:** Refactor
**Created:** 2026-03-28
**Status:** Draft

## Summary

将 `/test-supervisor evolution` 命令拆分为两个独立命令：`discoveries`（测试为项目发现的 bug）和 `evolution`（测试框架自身的进化），解决当前一个命令混合两种不同关注点的问题。

## Context

当前 `/test-supervisor evolution` 视图同时展示：
1. 测试发现的项目 bug 的生命周期（results + fixes/）
2. 框架自愈记录（self-heal.md）
3. 框架架构变更（evolution.json）

这三者混在一个视图中，概念模糊。项目质量发现和框架自我进化是两个本质不同的关注点，应分别跟踪。

## Acceptance Criteria

- [ ] 新增 `/test-supervisor discoveries` 命令，展示测试为项目发现的 bug 及修复状态
- [ ] `discoveries` 视图包含 Impact Summary（found/fixed/verified/unaddressed 计数）
- [ ] `discoveries` 视图按 testId 展示完整生命周期（发现→诊断→修复→验证→回归）
- [ ] 重写 `/test-supervisor evolution` 命令，仅展示框架自身进化（evolution.json + self-heal + metrics）
- [ ] `evolution` 视图去掉原来的 per-test lifecycle 和 round discovery 部分
- [ ] `stats-reporter.sh` 在每轮结束时自动生成/更新 `tests/discoveries.json`
- [ ] `discoveries.json` 从 fixes/ 目录自动聚合，无需手动维护

## Dependencies

- `tests/evolution.json` 不变
- `tests/state.json` 结构不变
