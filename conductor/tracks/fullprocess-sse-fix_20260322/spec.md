# Specification: Fix Full Process SSE Messages Empty Content

**Track ID:** fullprocess-sse-fix_20260322
**Type:** Bug
**Created:** 2026-03-22
**Status:** Draft
**Source:** ISS-087

## Summary

Full Process 点击后 SSE 事件流中的消息内容全部为空字符串，导致 UI 只看到 "Step full-process started" 而看不到后续 agent 执行消息。

## Problem Description

### Root Cause

`case-routes.ts` 的 full-process handler 使用 `formatMessageForSSE()` → `extractContent()` 提取消息内容，但 SDK `assistant` 消息的实际结构是：

```json
{ "type": "assistant", "message": { "content": [{ "type": "tool_use", ... }, { "type": "text", ... }] } }
```

`extractContent()` 检查的是 `message.content`（顶层），而不是 `message.message.content`（SDK 嵌套结构），所以所有 assistant 消息的 content 提取结果都是空字符串。

### Why steps route works but full-process doesn't

Steps route 使用 `processStepMessages()` 函数（steps.ts:222），它正确地深度解析：
- `message.type === 'assistant'` → `message.message.content` (数组) → 遍历 content blocks
- `message.type === 'tool_result'` → `message.content` 或 `message.text`

Full-process route 直接使用 `getSSEEventType()` + `formatMessageForSSE()`，跳过了深度解析。

### Evidence

SSE 监听测试显示：
- SSE 事件确实被发送（`case-session-thinking`）
- 但所有 `content` 字段为 `""`
- `messageType` 包括 `system`、`assistant`、`user`——都无法被 `extractContent()` 正确处理

## Acceptance Criteria

- [ ] Full Process 点击后 SSE 消息流包含非空内容（tool-call 名称、thinking 文本等）
- [ ] Agent Monitor 能看到 full-process 的消息流
- [ ] full-process 和 step 的消息格式一致（都使用语义化消息：tool-call、thinking、tool-result）

## Dependencies

None — all changes are in existing backend files.

## Out of Scope

- 前端消息渲染逻辑（已经能正确处理 step 消息，格式统一后自动生效）
- 新增 SSE 事件类型

## Technical Notes

**Fix approach**: 让 full-process route 复用 `processStepMessages()` 函数（或其核心逻辑），而不是直接使用简单的 `formatMessageForSSE()`。这样 full-process 和 step 的消息处理完全一致。
