# Specification: Issue 页面增加编辑按钮（点击展开只读详情）

**Track ID:** issue-edit-btn_20260320
**Type:** Bug
**Created:** 2026-03-20
**Status:** Draft
**Source:** ISS-025

## Summary

当前 WebUI Issues 页面点击 issue 行时直接进入 inline 编辑模式，缺少只读详情状态。需改为点击展开只读详情，通过 Edit 按钮进入编辑。

## Problem Description

- 点击 issue 行直接调用 `startEdit(issue)`，立即进入编辑模式
- 用户只想查看详情时被迫进入编辑状态，容易误操作
- 缺少明确的编辑入口按钮

### Steps to Reproduce

1. 打开 Dashboard → Issues 页面
2. 点击任意 issue 行
3. 行立即切换为编辑表单

### Expected vs Actual

- **Expected:** 点击行 → 展开只读详情面板 → 面板中有 Edit 按钮 → 点击后进入编辑
- **Actual:** 点击行 → 直接进入 inline 编辑模式

## Acceptance Criteria

- [ ] 点击 issue 行展开只读详情，不直接进入编辑
- [ ] 只读详情中有明确的 Edit 按钮
- [ ] 点击 Edit 按钮进入编辑模式
- [ ] Save/Cancel 后回到只读详情
- [ ] 再次点击行标题/头部可以折叠详情

## Affected Areas

- `dashboard/web/src/pages/Issues.tsx` — IssueRow 组件

## Dependencies

None

## Technical Notes

- 新增 `expandedId` state 管理展开/折叠
- `editingId` 保持不变，用于编辑状态
- 交互流：collapsed → expanded (readonly) → editing → expanded (readonly)
