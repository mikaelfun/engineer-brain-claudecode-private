# Specification: Cron Job Prompt Arguments Not Passed to Skill

**Track ID:** cron-args-bug_20260402
**Type:** Bug
**Created:** 2026-04-02
**Status:** Draft
**Issue:** ISS-207

## Summary

WebUI Agents 页面创建的 cron job（如 `/onenote-export sync`）在执行时，`sync` 参数未传递给 skill，导致 skill 提示"需要参数才能执行"。

## Problem Description

### Steps to Reproduce

1. 在 Dashboard → Agents 页面点击 "New Cron Job"
2. Prompt 选择 preset "OneNote Sync (incremental)"，值为 `/onenote-export sync`
3. 保存后等待定时执行，或手动点击 Run

### Expected Behavior

skill `/onenote-export` 收到 `args: "sync"`，执行增量同步。

### Actual Behavior

skill `/onenote-export` 收到空 args，回复用法说明而不执行。`lastOutput` 确认：

> `/onenote-export` 需要参数才能执行。以下是支持的用法：...

### Root Cause

`cron-manager.ts` 的 `executeCronPrompt()` 通过 `spawn('claude', ['-p', prompt])` 执行。在 `claude -p` 模式下，Claude 将 `/onenote-export sync` 中的 `/onenote-export` 识别为 skill 并调用 Skill 工具，但 `sync` 未被解析为 `args` 参数。

这是 `claude -p` 模式的已知限制——它作为自然语言 prompt 处理，不像交互式模式那样解析 slash command 的 arguments。

## Acceptance Criteria

- [ ] 通过 cron job 执行 `/onenote-export sync` 时，sync 参数正确传递给 skill
- [ ] 所有带参数的 prompt preset（如 `/patrol`、`/rag-sync`）也能正确工作
- [ ] 现有已创建的 cron job 无需用户重新配置即可生效

## Dependencies

- `dashboard/src/services/cron-manager.ts` — 执行引擎
- `claude -p` 模式的 slash command 参数处理行为

## Out of Scope

- 修改 Claude CLI 本身的 `-p` 模式行为
- 修改前端 CreateTriggerDialog 的 UI 交互
- 新增 cron 表达式格式支持

## Technical Notes

可行方案：修改 `executeCronPrompt()` 中的 prompt 格式，使 Claude 在 `-p` 模式下能正确传递 args。例如将 `/onenote-export sync` 改为显式指令 `请执行 /onenote-export，参数为 sync` 或直接在 prompt 中告知 Claude 使用 Skill 工具时带上 args。
