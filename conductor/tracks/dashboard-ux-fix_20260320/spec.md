# Specification: Dashboard UX Fix

**Track ID:** dashboard-ux-fix_20260320
**Type:** Bug
**Created:** 2026-03-20
**Status:** Draft

## Summary

修复 EngineerBrain Dashboard 的 24 个已知 UX 问题，涵盖重复 Agent spawn、进度丢失、Todo 加载失败、Workspace Not Found、Analysis 误报空等关键缺陷，全面提升 Dashboard 的可用性和可靠性。

## Context

EngineerBrain Dashboard 是一个全栈 Web 应用（React + Hono），用于管理 Azure 支持案例的 AI 工作流。当前 Dashboard 架构已基本完整，但在实际使用中暴露了多个 UX 问题，记录在 Finetune.txt 和 .learnings/ 中。这些问题严重影响日常工作效率，尤其是 Critical 级别的 4 个问题会导致工作流中断或数据丢失。

## Problem Description

### Critical (必须修复，影响核心工作流)
1. **重复 Agent/Session spawn** — 按钮响应慢、无即时状态反馈，用户多次点击生成多个 Agent
2. **无实时进度反馈** — 关键操作（Full Process、Troubleshoot、Draft Email）缺乏实时步骤级状态展示
3. **刷新丢失进度** — 页面 refresh 后工作流进度消失，完成后也无持久化状态/细节
4. **Todo 加载失败** — per-case todo 和 legacy 全局 todo 架构混乱，导致数据无法加载

### High (严重影响可用性)
5. **Workspace Not Found** — Dashboard 顶部出现误报的 workspace not found 横幅
6. **Analysis tab 误报空** — 文件夹有内容但 UI 显示 "No analysis report"
7. **Case 标题显示不完整** — 长标题被截断，无 tooltip 或展开方式
8. **Patrol 不并行** — WebUI 触发的 patrol 实际串行执行
9. **Draft Email 缺少 AI 自动识别类型** — 需手动选择邮件类型，缺少 AI 默认识别
10. **步骤级日志缺失** — casework 各步骤无对应日志写入

### Medium (UX 摩擦)
11. Agent Monitor 刷新行为 — 自动刷新间隔和手动刷新体验
12. Drafts 页面格式/空行 — 邮件格式化不佳
13. Drafts 可折叠 — 草稿卡片需可折叠（可能已修复）
14. Todo 解析不准确 — 基于 emoji regex 的统计不可靠
15. Legacy vs per-case todo 混合 — 过渡期架构导致不一致
16. Session 面板与 AI 面板功能重叠 — 控件分离不清晰
17. Patrol 进度不准确 — 进度百分比和状态展示不完整
18. Tab 数据延迟加载 — 标签页 badge 在未打开前不可见

### Low (打磨)
19. Case 列表排序仅按 Severity — 缺少状态/SLA/Todo 维度排序
20. Teams/Logs tab 原始展示 — 缺少结构化格式
21. Settings 手动路径输入 — 无浏览器/选择器

## Acceptance Criteria

- [ ] 每个 case 的每种操作类型最多一个活跃 session，UI 即时禁用已触发按钮
- [ ] 操作触发后有实时步骤级进度显示（SSE 驱动）
- [ ] 页面刷新后能从后端恢复活跃 session 状态和历史消息
- [ ] Todo 页面可正常加载 per-case todo 数据，移除 legacy 代码
- [ ] Workspace Not Found 仅在真正无 workspace 时显示
- [ ] Analysis tab 能正确检测并显示 analysis/ 目录下所有内容
- [ ] Case 标题完整显示或有 tooltip
- [ ] 所有 Critical 和 High 问题修复后通过手动 smoke test
