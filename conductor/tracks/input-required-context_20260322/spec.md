# Specification: INPUT REQUIRED 卡片应显示 specification 上下文

**Track ID:** input-required-context_20260322
**Type:** Bug
**Created:** 2026-03-22

**Source:** ISS-072

## Summary

Create Track 时 INPUT REQUIRED 卡片只显示问题文本（如 "Is this specification correct?"），specification 内容仅出现在上方 SSE 流中。根因是 `contextContent` 提取逻辑在遇到 `tool-call` 消息时中断，而 thinking → tool-call(AskUserQuestion) → question 的消息序列导致 spec 内容永远无法被捕获到卡片内。

## Context

Dashboard WebUI 的 Issues 页面支持 Create Track 功能，通过 SSE 流式展示 Claude agent 的创建过程。当 agent 调用 AskUserQuestion 时，前端 TrackProgressPanel 渲染一个 INPUT REQUIRED 卡片，允许用户直接在页面内回答。

## Problem Description

**消息序列：**
1. `thinking`: "Pre-flight checks passed... Here is the generated specification: ..."
2. `tool-call`: "Tool: AskUserQuestion" ← contextContent 在这里 break
3. `question`: "Is this specification correct?"

**根因（QuestionForm line 210-226）：**
```typescript
// 当前逻辑只 skip 'question'，遇到 'tool-call' 就 break
if (m.kind === 'thinking' && m.content) {
  thinkingTexts.unshift(m.content)
} else if (m.kind === 'question') {
  continue  // 只跳过 question
} else {
  break     // tool-call 在这里被 break 了！
}
```
