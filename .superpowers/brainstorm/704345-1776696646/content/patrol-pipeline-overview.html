Summary
设计要点： 左侧 Patrol Pipeline 从 6 个 flat 阶段重构为 3 个父阶段 + 子步骤。
• Initialize = SDK Session + Config/Skill 加载 + D365 查询 + 筛选/归档 + Token 预热
• Process = 每个 spawn 的 subagent 是独立子步骤，按 spawn 顺序排列，含 SSE 流式消息
• Finalize（替代 Aggregate）= Orphan 清理 + 结果聚合 + State 写入 + Lock 释放
右侧 Casework Pipeline 保持现有设计不变（水平 4 步 + 3-zone case row），作为参考对照。
