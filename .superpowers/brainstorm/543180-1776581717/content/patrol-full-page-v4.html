信息层次设计

🔭 全局 Pipeline（6 列卡片）
  • Start: SDK 启动耗时 + session ID
  • Discover: 查询了什么 + 找到多少 case
  • Filter: 多少有变化 / 多少跳过
  • Warmup: 哪些 token 预热了
  • Process: 进度条 + done/total
  • Aggregate: todo 汇总结果
📋 Case 行（4 列网格）
  • Refresh: 5 个子任务各自状态 + 耗时
  • Assess: 结果标签 + 推理摘要
  • Act: agent 卡片 + 当前操作描述
  • Summary: todo 更新 / 状态变更
  • Completed 行自动收紧，Active 行展开
  • Queued 只显示一行 header

需要后端配合的数据扩展

当前 patrol-progress.json 和 state.json 缺少部分字段。实现时需要：
  • 全局: patrol skill 在 discover/filter/warmup 阶段写入结果数据（totalFound、changedCount、warmupTokens）
  • Assess: casework skill 写入 result（已有）+ reasoning 摘要（新增）
  • Act: 每个 agent action 写入 detail 描述当前操作（新增）
  • Summary: 写入 result 描述生成的 todo/状态变更（新增）
